# 主要思路

ZEUS方法的提出源于一个关键观察：模型对不同问题的不确定性程度与其推理能力密切相关。这种不确定性可以作为选择高质量示例的重要指标，从而指导示例的选择和生成。

其适用三种扰动方式, 对同一个问题多次生成答案, 评估答案的不确定性. 然后选择不确定性在一定策略范围内的问题-答案对, 作为few-shot. 

论文认为 用不确定性筛选后的问题-答案对, 有助于大模型提高推理能力

# 其他信息

ZEUS 的目标是提高零样本思维链 (CoT) prompting 的性能，它通过从无标签数据集中选择合适的样本（问题）来构建高质量的 few-shot demonstrations。 它的整体流程如下：

  1. **准备阶段:**

     - 拥有一个无标签问题数据集 Q，这些问题将作为潜在的 demonstration 样本。
     - 确定目标 LLM 和 CoT prompting 的基本设置（例如，推理时使用的提示词和温度）。
  2. **不确定性估计:** 对于数据集 Q 中的每个问题 qj，进行以下步骤：

     - **多重扰动:** 使用多种扰动方法生成多个答案。这些扰动方法包括：
       - 温度扰动：使用不同的温度值进行解码。
       - 提示词扰动：使用不同的提示词（例如，“Let's think step by step”, "Let's solve this problem" 等）。
       - 问题改写：将问题改写成不同的表达方式。
     - **计算不确定性:** 根据生成的多个答案，计算问题 qj 的不确定性 uj。论文中使用预测熵 (Predictive Entropy) 来衡量不确定性，越高表示越不确定。
  3. **样本选择:**

     - **计算统计量:** 计算所有问题不确定性的均值 (µ) 和标准差 (σ)。
     - **选择策略:** 根据 µ 和 σ，选择一个预定义的策略来确定选择哪些样本。策略定义了选择样本的不确定性范围 (umin, umax)。例如，“选择中等不确定性的问题”策略会选择 uj 接近 µ 的问题。
     - **筛选样本:** 根据选择的策略，从数据集 Q 中筛选出符合条件的问题，构成子集 Qs。
  4. **构建 Demonstrations:**

     - **聚类:** 将 Qs 中的问题进行聚类（例如，使用 k-Means++ 算法）。
     - **选择代表:** 从每个聚类中选择一个代表性问题（例如，最接近聚类中心的问题）。
     - **生成推理和答案:** 使用零样本 CoT 方法为每个代表性问题生成推理步骤 rj 和答案 aj。
     - **组装 demonstrations:** 将选择的代表性问题、生成的推理步骤和答案组装成 few-shot demonstrations D = {(qj, rj, aj)}。
  5. **推理:**

     - 对于新的测试问题 q，将其添加到 demonstrations D 之前，构成新的输入。
     - 使用目标 LLM 和预设的 CoT prompting 设置（例如，固定的提示词和温度 0）对新的输入进行推理，得到最终答案。

总而言之，ZEUS 的核心在于通过多重扰动估计问题的不确定性，并基于不确定性选择合适的样本构建 demonstrations，从而提高 LLM 在零样本 CoT prompting 下的推理性能。 重要的是，扰动只是用于样本选择，最终推理过程并不涉及扰动。
