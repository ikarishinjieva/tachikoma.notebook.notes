# 概述

文档提出了一种方法, 自动构建CoT的样例, 用作few-shot

方法细节: 

```
Auto-CoT 是一种自动构建思维链 (Chain-of-Thought, CoT) 演示的方法，旨在解决人工构建 CoT 演示 (Manual-CoT) 费时费力且不灵活的问题。其核心思想是利用 LLM 自身的推理能力，自动生成高质量的 CoT 演示，并结合一些策略来降低错误演示带来的负面影响。

Auto-CoT 的方案主要包含以下两个阶段：

**1. 问题聚类 (Question Clustering):**

   * 目的：将语义相似的问集合类在一起，确保选择的演示具有多样性，避免相似问题带来的错误放大。
   * 步骤：
      1. 使用 Sentence-BERT 模型对所有问题进行编码，得到每个问题的向量表示。
      2. 利用 k-means 聚类算法将问题划分到 k 个不同的集群中。
      3. 在每个集群内，根据问题与集群中心的距离对问题进行排序，距离越近，代表性越强。

**2. 演示采样 (Demonstration Sampling):**

   * 目的：从每个集群中选择一个代表性问题，并使用 Zero-Shot-CoT 生成其推理链，最终构建 k 个演示。
   * 步骤：
      1. 对于每个集群 i：
         a. 遍历集群 i 中的排序后的问题列表。
         b. 对当前问题 q，使用 Zero-Shot-CoT (即在问题后添加 "Let's think step by step")，让 LLM 生成推理链 r 和答案 a。
         c. 判断问题 q 和推理链 r 是否满足预设的选择标准 (例如：问题长度、推理步骤数量等)。
         d. 如果满足，则将 (q, r, a) 作为一个演示加入演示集，并进入下一个集群的处理。
         e. 如果不满足，则继续处理集群 i 中的下一个问题。
      2. 最终得到包含 k 个演示的演示集。

简单来说，Auto-CoT 首先将问题进行聚类，确保选择的演示来自不同的问题类别，具有多样性。然后，从每个类别中选择一个代表性问题，利用 LLM 生成其推理链，并通过一些选择标准来过滤掉质量较低的推理链，最终构建出 k 个演示，用于后续的 CoT 推理任务。

**Auto-CoT 的优势：**

* 自动化：无需人工构建 CoT 演示，节省了大量时间和人力成本。
* 任务自适应：可以根据不同的数据集自动构建相应的演示，灵活性更高。
* 性能好：在多个基准数据集上的实验结果表明，Auto-CoT 的性能与 Manual-CoT 相当，甚至更好。

总而言之，Auto-CoT 提供了一种高效、灵活、通用的 CoT 演示构建方案，有效解决了 Manual-CoT 的局限性，推动了 CoT 技术的进一步发展。
``` 

算法简述: 

1\. 所有问题 在 Sentence-BERT embedding 的空间内, 进行k聚类  
2\. 聚类划分后就不再调整  
3\. 从每个聚类中, 进行zero-shot, 选择满足要求的, 作为CoT的样例

# 算法缺陷

在Sentence-BERT embedding 的空间内进行聚类, 比如说其中有问题Q1和Q2, 但不代表Q1和Q2的推理链的推理深度是类似的.

这个算法的前提 "聚类空间 == 推理深度" (等于或有助于), 这个前提并不成立
