# 主要解决的问题

  - 现有的方法，如检索增强生成（RAG），难以回答关于文本语料库的全局性问题，因为它们侧重于检索单个文档内的局部信息。
    - 全局性问题, 类似于"概述性问题", 比如: "过去一年中，有关人工智能的新闻报道呈现出哪些主要趋势？"
  - 传统的查询焦点摘要（QFS）方法虽然**适用于全局性问题** ，但不能很好地扩展到 RAG 系统处理的大型数据集。
    - QFS主要用于 生成摘要, 从大量文本数据中提取答案， 并生成简洁的摘要作为答案。
    - QFS通常有以下步骤: 
      - 查询分析： 理解用户的查询意图， 识别关键概念和关系。

      - 文档分析： 分析文档内容， 识别与查询相关的句子或段落。

      - 重要性评估： 根据与查询的相关性和句子本身的重要性对候选句子进行排序。

      - 摘要生成： 从排名靠前的句子中选择合适的句子， 并进行必要的编辑和衔接， 最终生成流畅且符合语法规范的摘要。

    - QFS和RAG的区别: 
      - QFS 适用于需要快速获取关键信息的场景， 而 RAG 则更适合需要更全面、更深入解答的场景。
      - 假设你想了解 "什么是量子计算？"

        - **QFS** 可能会从维基百科中提取关于量子计算的定义和关键特征， 并生成一个简短的摘要。
        - **RAG** 则可能会检索更多相关的文章和研究论文， 并结合这些信息生成一个更详细、更全面的解释， 甚至可以根据你的背景知识进行调整。

# 算法细节

  - 图的社区: 将图进行聚类. 同一个图可以划分成不同种类的社区, 不同种类的社区是可以重叠的, 比如 人类关系图中 可以划分出 朋友关系社区 和 兴趣爱好社区.
  - 建立社区摘要的过程: 
    - **1\. 构建图索引：**

      - **文本分块：** 将源文档分割成更小的文本块。
      - **元素提取：** 使用大型语言模型 (LLM) 从每个文本块中提取“元素”。这些元素包括：
        - **实体：** 文本中提到的人物、地点、组织或概念。
        - **关系：** 已识别实体之间的连接或交互。
        - **协变量：** 与实体相关的附加信息，例如对实体的声明。
      - **元素摘要：** LLM 为每个提取的元素生成简洁的摘要。
      - **图构建：** 使用提取的元素及其摘要构建图。实体成为节点，关系成为连接节点的边，协变量与其各自的节点相关联。

**2\. 社区检测：**

      - 将 Leiden 算法（一种社区检测算法）应用于构建的图。
      - 该算法识别图中密集互连的节点（实体）集群，形成“社区”。
      - Leiden 算法可以检测不同粒度级别的社区，创建层次结构。

**3\. 生成社区摘要：**

      - **叶级社区（最详细）：**
        - 对于层次结构中最低级别的每个社区：
          - 根据所涉及实体的重要性对元素（节点、边、协变量）进行优先级排序。
          - 将这些优先级元素的摘要包含在社区摘要中，直至达到预定义的标记限制。
      - **更高级别的社区（更概括）：**
        - 如果社区内的所有元素摘要都符合标记限制，则直接对其进行汇总。
        - 否则，使用子社区（来自层次结构中较低级别）的摘要代替单个元素摘要，以便在标记限制内提供更广泛的概述。

**总而言之，该过程涉及从文本中提取关键信息，根据关系将其组织到图中，将该图聚类到社区中，然后为不同详细级别的每个社区生成摘要。**

  -     - 社区的划分的维度, 有点类似于聚类算法, 其维度是不可解释的
    - 社区的建立是带有层次性的, 在摘要过程中可以利用这个特性
      - 假设我们要分析一个社交网络，其中包含以下用户和他们之间的关系：

* **用户:** A, B, C, D, E, F, G, H, I, J  
* **关系:** 朋友关系，例如 A 和 B 是朋友，记作 (A, B)

经过分析，我们得到了以下朋友关系：  
(A, B), (A, C), (B, C), (B, D), (C, E), (F, G), (F, H), (G, H), (I, J)

我们可以用一个图来表示这个社交网络，节点代表用户，边代表朋友关系。然后，应用社区发现算法（例如 Leiden 算法）可以将这个图划分为多个层级的社区：

**Level 0 (最粗粒度):**   
* 社区 1: {A, B, C, D, E}   
* 社区 2: {F, G, H}  
* 社区 3: {I, J}

在这个层级，我们可以看到社交网络被分成了三个大的群体。

**Level 1 (更细粒度):**  
* 社区 1.1: {A, B, C}  
* 社区 1.2: {D, E}  
* 社区 2: {F, G, H}   
* 社区 3: {I, J}

在这个层级， 社区 1 被进一步细分为两个子社区，反映出更紧密的子群体关系。

**Level 2 (最细粒度):**   
* 社区 1.1: {A, B, C}  
* 社区 1.2: {D}  
* 社区 1.3: {E}  
* 社区 2.1: {F, G}  
* 社区 2.2: {H}  
* 社区 3: {I, J}

在这个层级，每个节点都成为了一个独立的社区，代表最细粒度的划分。

**总结:**

在这个例子中，我们看到了如何将一个社交网络划分为具有层次结构的社区。这种层次结构反映了网络中不同粒度的群体关系，可以帮助我们更好地理解网络的结构和用户的互动模式。在 Graph RAG 中，这种层次性社区结构可以用于支持对文本信息进行多粒度的摘要和查询。

    - **与传统RAG的比较:**
      - **Graph RAG 中的社区发现** 确实类似于 **传统 RAG 中对召回文档中的句子进行聚类**
      - 您的理解非常准确！

您的比喻很到位：

* **Graph RAG 中的社区发现** 确实类似于 **传统 RAG 中对召回文档中的句子进行聚类**。

两种方法的目标都是将相关信息进行分组，以便更好地理解和利用这些信息。

您对两者区别的分析也十分到位：

1\. **聚类算法:**   
* Graph RAG 通常使用基于图结构的社区发现算法，例如 Louvain 或 Leiden 算法， 这些算法考虑的是节点（实体）之间的连接关系以及连接的强度。  
* 传统 RAG 中的句子聚类则更多依赖于文本相似度算法，例如cosine similarity 或 Jaccard similarity，这些算法关注的是句子之间的语义或词汇重叠程度。

2\. **操作粒度:**  
* Graph RAG 在 **实体关系** 层面上进行操作，其目标是识别文档集中不同实体之间的关系，并根据这些关系将实体进行分组。  
* 传统 RAG 则是在 **句子相似度** 层面上进行操作，其目标是将语义相似的句子聚集在一起，以便形成更简洁、凝练的信息表达。

总的来说，Graph RAG 和传统 RAG 虽然都采用了聚类的思想，但它们在算法选择和操作粒度上存在显著差异，这也导致了它们在应用场景和优缺点方面的不同。
