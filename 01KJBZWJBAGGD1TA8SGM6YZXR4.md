# 主要创新

  - 使用逻辑形式, 组织思考步骤
    - 如下例: Action: Retrieval(s:肝囊肿, p:尺寸分级标准, o:分级定义)
  - **知识边界判定（Knowledge Boundary Determination）** : 这是框架的“决策核心”，用于减少不必要的外部检索。
    - **策略** : 采用“先生成，后评估”（Generate First, Then Assess）的策略。
    - **双重验证** : 模型首先尝试用内部知识直接回答子问题，然后通过两种方式评估答案的置信度：**基于提示的置信度评估** （让模型自己判断答案是否正确）和**基于可能性的置信度评估** （分析生成答案时各词元的概率，取最低概率作为置信度分数）。只有当两种评估都认为可信时，才采纳内部答案；否则，启动外部检索。
  - **聚焦与推理（Focusing and Reasoning）** : 这是框架的“信息过滤器”。当外部检索完成后，该模块负责评估检索到的内容是否能回答子问题，并过滤掉噪声和无关信息，将有效信息与模型内部知识结合，从而提炼出解决子问题所需的核心知识。

# 完整流程

好的，我们来用一个生动的故事，讲述“AI医疗侦探KAG-Thinker”如何为“刘策划”解决他的“肝囊肿”难题，并在这个过程中揭示其完整的流程与关键创新。

* * *

### 故事：AI侦探与刘策划的肝囊肿谜案

**【案件背景】**

刘策划，一位45岁的互联网公司高管，最近在体检中发现了一个直径6厘米的“肝囊肿”。医生建议他考虑手术，但他又在网上看到一种新型靶向药物似乎效果不错。他自己还有多年的高血压病史。面对复杂的病情和选项，他感到困惑和焦虑，于是找到了以逻辑严谨、深度思考著称的“AI医疗侦探KAG-Thinker”。

刘策划的问题是：“**我，一个有高血压的45岁男性，面对这个6cm的肝囊肿，是选择腹腔镜手术还是那种新药更好？** ”

* * *

### 第一幕：作战总纲的制定 (广度分解)

侦探KAG-Thinker并没有立即回答，而是坐下来，启动了他的核心能力——**广度分解 (Breadth Decomposition)** 。他像一位总指挥，在解决这个复杂的“谜案”前，必须先制定一份滴水不漏的“作战总纲”。

他在虚拟白板上，**一次性** 地列出了解决此案必须搞清楚的所有子问题，并为每一步都创建了一张精确的“情报卡片”。

> **【关键创新：逻辑形式 (Logical Form)】**  
>  每一张“情报卡片”都用两种语言写成：一句人类能懂的自然语言（Step），和一条机器能精确执行的指令（Action）。这确保了计划的严谨性和可执行性。

**KAG-Thinker的“作战总纲”如下：**

  - **情报卡片1** : 首先，6cm的肝囊肿在医学上算什么水平？
    - `Action: Retrieval(s:肝囊肿, p:尺寸分级标准, o:分级定义)`
  - **情报卡片2** : 针对这种大小的囊肿，标准的腹腔镜手术方案是什么？风险有哪些？
    - `Action: Retrieval(s:大型肝囊肿, p:标准手术方案, o:手术详情)`
  - **情报卡片3** : 刘策划提到的“新药”具体是什么？它的作用机理和临床试验效果如何？
    - `Action: Retrieval(s:肝囊肿, p:新型治疗药物, o:药物信息)`
  - **情报卡片4** : 高血压病史对腹腔镜手术的麻醉和术后恢复有何具体影响？
    - `Action: Deduce(op:extract, content=[#2的手术风险, 高血压并发症], target:高血压患者的手术额外风险)`
  - **情报卡片5** : 该新药是否会与常见的高血压药物（如硝苯地平）产生相互作用？
    - `Action: Retrieval(s:#3的药物, p:药物相互作用, o:与硝苯地平的反应)`
  - **情报卡片6** : **最终裁决** ：综合以上所有情报（#1, #2, #4, #5的结果），为刘策划生成一份个性化的利弊分析报告。
    - `Action: Output(content=[所有情报], target:为刘策划生成最终建议)`

至此，**广度分解** 完成。一个清晰、完整、逻辑环环相扣的调查路径已经铺开。

* * *

### 第二幕：逐个击破的深度调查 (深度求解)

现在，侦探KAG-Thinker开始按照“作战总纲”，一张一张地处理“情报卡片”。他为每一张卡片都启动了**深度求解 (Depth Solving)** 流程。

**调查“情报卡片1”：6cm肝囊肿的定性**

  1. **【关键创新：知识边界判定 (Knowledge Boundary Determination)】**  
KAG-Thinker首先检索自己的“大脑记忆”（内部知识库）。它自信地判断：“根据我的医学知识，直径大于5cm的肝囊肿通常被定义为‘巨大肝囊肿’，需要医学干预。” 它的置信度非常高。
  2. **决策** : 无需求助外部资料库！它直接在卡片1下标注了答案：“巨大肝囊肿，需关注”。

**调查“情报卡片2”：手术方案**

  1. **知识边界判定** : KAG-Thinker自省：“手术方案细节繁多且不断更新，我的内部知识可能不够全面。” 它决定，**必须启动外部调查** 。
  2. **启动检索** : 它向外部的权威医学数据库发出了检索指令。
  3. **【关键创新：聚焦与推理 (Focusing and Reasoning)】**  
数据库返回了大量文献，包括肾囊肿手术、开腹手术等无关信息（噪声）。KAG-Thinker并没照单全收，而是迅速过滤：“这篇是肾脏的，无关。这篇是20年前的技术，过时。啊，这篇是《柳叶刀》去年关于‘腹腔镜肝囊肿去顶术’的综述，**这正是核心情报！** ” 它精准地从噪声中提取了关键信息。

**调查“情报卡片4”：高血压与手术风险**

  1. **知识边界判定** : 决定外部调查。
  2. **深度迭代** : 第一次检索“高血压 手术风险”，结果太泛。KAG-Thinker思考后，发起了更**深入、精确** 的第二轮检索：“高血压患者在全麻状态下的心血管事件发生率及预防措施”。这次，它得到了更具针对性的高质量情报。

这个过程，就是**深度求解** ——针对一个具体子问题，通过“自省 -> 决策 -> 检索 -> 过滤 -> 迭代”的循环，不达目的不罢休，直到获得最精准的答案。

* * *

### 最终幕：谜案告破 (整合与输出)

经过一番深度调查，所有的“情报卡片”都被填上了确切的答案。侦探KAG-Thinker将它们全部摊开，进行了最后的逻辑整合（执行情报卡片6的`Output`指令）。

它向刘策划提交了最终的调查报告：

“刘策划您好，根据对您情况的全面分析：

  1. 您的6cm肝囊肿属于‘巨大肝囊肿’，确实需要干预。
  2. 腹腔镜手术技术成熟，但考虑到您的高血压，麻醉风险和术后血压波动风险会相应增高。
  3. 您提到的新药在临床试验中对缩小囊肿效果显著，且与您服用的降压药无不良相互作用。  
**结论建议** ：鉴于您的个人情况，建议**优先考虑使用新药进行保守治疗** ，并密切观察。若效果不佳或出现并发症，再将手术作为备选方案。这样能最大程度地规避您因高血压而带来的额外手术风险。”

刘策划的困惑烟消云散，他获得了一个清晰、个性化、且逻辑链条完整的决策依据。这个复杂的“肝囊肿谜案”，被侦探KAG-Thinker完美侦破。
