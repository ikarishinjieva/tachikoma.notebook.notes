# 论文要点

在CoT推理中, 论文将第一轮推理的结果 (LLM的最后一层隐藏状态向量), 直接作为了第二轮推理的输入 (用隐藏状态向量 直接 替换了 词嵌入向量).

其核心思想是: 隐藏状态向量, 比起文字来说, 能承载更多地信息. 在CoT中, 用隐藏状态变量承载第一步推理的结果, 比起文字来说信息更丰富. 

其中的问题是: 隐藏状态向量 并不是 词嵌入空间 中的一个有效概念, 方案从逻辑上是有点拧巴, 也就是说训练的路径从 "词嵌入空间 -> 模型" 变成了 "某种逻辑空间 -> 模型"

# 训练方式

![image2025-1-6 16:55:41.png](/assets/01KJBZPTXXKTXVDR009ADNWMAK/image2025-1-6%2016%3A55%3A41.png)

逐步进行遮蔽

通用模型的处理逻辑: 输入文字 -> token ID -> 词嵌入向量 -> (多层神经网络) -> 最后一层隐藏状态向量 -> 线性变换 -> 词表概率向量 -> (解码策略) -> 输出词

Coconut的处理逻辑: 

  - 第一轮: 输入文字 -> token ID -> 词嵌入向量 -> (多层神经网络) -> 最后一层隐藏状态向量X
  - 第二轮: 最后一层隐藏状态向量X -> (多层神经网络) -> 最后一层隐藏状态向量 -> 线性变换 -> 词表概率向量 -> (解码策略) -> 输出词

从理解上, (多层神经网络) 需要同时能处理 词嵌入空间 和 隐藏状态向量空间 作为输入空间, 不确定是否可行.
