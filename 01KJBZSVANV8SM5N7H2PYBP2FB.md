安全地从子查询（包括派生表、CTE等）的SELECT列表中移除未被外部查询或后续查询阶段引用的字段，以优化查询性能。

    //   遍历主查询（或引用该CTE的后续查询部分）的以下子句：

        // 只要该字段不被外部查询引用（已由步骤1检查），它就可以从EXISTS子查询的SELECT列表中移除。
        // 优化器通常可以将这类子查询的SELECT列表替换为 `SELECT 1` 或 `SELECT NULL`。
        返回 "可删除 (EXISTS/NOT EXISTS context) - 字段在SELECT列表，可被移除或替换为常量"
        // 注意: 此判断具有较高优先级。如果子查询用于EXISTS，此结论通常适用。

    // 步骤3: 检查字段对子查询自身结果集完整性的影响 (当子查询非EXISTS/NOT EXISTS用途时)

    // 如果字段未在步骤1中被发现外部引用，
    // 并且不属于步骤2 (EXISTS已处理并返回) 或步骤3中必须保留的情况，

安全地从子查询（包括派生表、CTE等）的SELECT列表中移除未被外部查询或后续查询阶段引用的字段，以优化查询性能。

### **逻辑伪代码**
函数 判断子查询字段是否可移除(主查询结构, 子查询实例, 待检查字段名):
    /*
    功能: 分析SQL子查询中的字段是否可以安全移除
    参数说明:
      - 主查询结构: 整个SQL查询的结构
      - 子查询实例: 主查询中的特定子查询(如派生表或CTE)
      - 待检查字段名: 子查询输出的某个字段名称
    */
    
    // 步骤0: 检查子查询内部的依赖关系（新增9）
    如果 子查询内的其他计算字段或表达式中引用了待检查字段名:
        返回 "不可删除 - 字段在子查询内部的其他计算字段中被引用"
    
    // 步骤1: 逐步检查字段是否被外部引用
    // 1.1: 检查SELECT列表
    如果 外部查询的SELECT列表直接引用该字段或在表达式中使用该字段:
        返回 "不可删除 - 字段在外部查询的SELECT列表中被引用"
    
    // 1.2: 检查WHERE子句
    如果 外部查询的WHERE子句条件中引用该字段:
        // 检查CASE表达式和复杂表达式（新增4）
        如果 该引用嵌套在CASE表达式或其他复杂表达式内:
            返回 "不可删除 - 字段在外部查询的WHERE条件的复杂表达式中被引用"
        返回 "不可删除 - 字段在外部查询的WHERE条件中被引用"
    
    // 1.3: 检查JOIN条件
    如果 外部查询的JOIN子句的ON或USING条件中引用该字段:
        返回 "不可删除 - 字段在外部查询的JOIN条件中被引用"
    
    // 1.4: 检查GROUP BY子句
    如果 外部查询的GROUP BY子句中引用该字段:
        返回 "不可删除 - 字段在外部查询的GROUP BY中被引用"
    
    // 1.5: 检查HAVING子句
    如果 外部查询的HAVING子句中引用该字段:
        返回 "不可删除 - 字段在外部查询的HAVING条件中被引用"
    
    // 1.6: 检查ORDER BY子句
    如果 外部查询的ORDER BY子句中引用该字段:
        返回 "不可删除 - 字段在外部查询的ORDER BY中被引用"
    
    // 1.7: 检查窗口函数
    如果 外部查询的窗口函数的PARTITION BY或ORDER BY子句中引用该字段:
        返回 "不可删除 - 字段在外部查询的窗口函数中被引用"
    
    // 1.8: 检查聚合函数和分析函数（新增10）
    如果 外部查询中的聚合函数或分析函数使用该字段作为参数:
        返回 "不可删除 - 字段被外部查询的聚合或分析函数使用"
    
    // 1.9: 检查CTE在查询中的多次引用（新增2）
    如果 子查询是CTE且在查询中被多次引用:
        对于 每个CTE引用点:
            执行 步骤1.1到1.8的检查
            如果 任何一处引用点使用了待检查字段:
                返回 "不可删除 - 字段在CTE的多处引用点中被使用"
    
    // 步骤2: EXISTS/NOT EXISTS子查询的特殊处理
    如果 子查询用于EXISTS或NOT EXISTS判断:
        返回 "可删除 - EXISTS/NOT EXISTS子查询中的字段可替换为常量"
    
    // 步骤3: 检查字段是否影响子查询结果
    // 3.1: DISTINCT和UNION去重逻辑
    如果 子查询包含DISTINCT或作为UNION(非UNION ALL)的一部分:
        返回 "不可删除 - 字段参与去重逻辑，移除会改变结果集"
    
    // 3.2: 带行数限制的排序
    如果 子查询同时满足以下条件:
        - 包含ORDER BY子句
        - ORDER BY中引用了待检查字段
        - 使用了LIMIT/OFFSET/TOP等限制行数:
            返回 "不可删除 - 字段影响有限制行数的排序结果"
    
    // 3.3: IN/NOT IN子查询
    如果 子查询用于IN/NOT IN条件:
        如果 待检查字段是比较目标列:
            返回 "不可删除 - 字段是IN/NOT IN比较的目标列"
        否则:
            返回 "可删除 - 非IN/NOT IN比较目标且未被外部引用"
    
    // 3.4: INTERSECT和EXCEPT操作（新增6）
    如果 子查询是INTERSECT或EXCEPT操作的一部分:
        返回 "不可删除 - 字段参与INTERSECT或EXCEPT操作，移除会改变列对应关系"

    // 注意：步骤1中已有对各子句中CASE表达式的检查，此处是额外的全面检查
