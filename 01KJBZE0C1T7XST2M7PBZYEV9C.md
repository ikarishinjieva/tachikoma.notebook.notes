# 主要创新

使用 合成的边界数据 来提升 提示词的生成质量

主要步骤: 

```
基于意图的提示校准 (IPC) 的具体步骤如下：

**1. 初始化:**

   * **用户提供初始信息:** 包括任务描述、初始提示以及可选的少量样本（少量样本学习）。
   * **定义目标:**  明确任务类型（例如分类或生成），并确定评估指标（例如准确率或排名得分）。

**2. 迭代优化循环:**

   * **(a) 生成合成数据：**
      * 利用样本生成器，根据当前提示和任务描述，生成一组多样化且具有挑战性的边界案例样本。
      * 在初始迭代中，生成器侧重于类别均衡的对抗性样本。
      * 在后续迭代中，生成器还会参考历史提示、已识别出的错误案例以及真实数据集中的样本，以生成更具代表性的样本。
   * **(b) 评估提示性能：**
      * 使用当前提示，在生成的合成数据集上进行预测。
      * 根据预先定义的评估指标，计算当前提示的性能得分。
      *  使用分析器组件分析错误案例，总结主要错误类型和原因。
   * **(c) 生成新的提示：**
      *  提示生成器接收当前提示的性能得分、分析器生成的错误分析报告以及历史提示信息。
      *  根据这些信息，生成器尝试生成一个预期性能更高的新的提示。
   * **(d) 判断是否终止循环：**
       *  检查是否满足终止条件，例如达到最大迭代次数或性能提升低于预设阈值。
       *  如果满足条件，则终止循环；否则，返回步骤 (a)，继续迭代优化。

**3. 输出最终提示：**

   * 当循环终止时，系统输出最后一次迭代中性能最佳的提示作为最终校准后的提示。

**对于生成任务，IPC 还包括一个额外的步骤：**

   * 在进行上述迭代优化循环之前，首先将原始生成任务转换为一个排名任务。
   *  使用上述步骤对排名任务的提示进行校准，得到一个用于评估生成结果质量的排名器。
   *  然后，利用已校准的排名器作为评估指标，对原始生成任务的提示进行优化。

总而言之，IPC 通过迭代生成合成数据、评估提示性能、分析错误案例以及生成新的提示，逐步优化提示，使其更符合用户意图，并最终提高 LLM 在目标任务上的性能。
``` 

步骤与AutoPrompt项目的流程一致

# 主要问题

使用LLM, 根据任务目标和一些样例提示词, 合成边界数据时, LLM是否有足够的能力

```
你说的没错，IPC 中的边界案例确实是使用 LLM 生成的。 

**至于 LLM 对边界案例的理解是否正确，这是一个比较复杂的问题。**

一方面，LLM 在生成边界案例时，是基于其对任务描述、初始提示以及已有样本的理解进行生成的。如果 LLM 对这些信息的理解存在偏差，那么生成的边界案例也可能存在问题。

另一方面，即使 LLM 对任务的理解是正确的，它也可能难以穷尽所有可能的边界情况。这是因为边界案例的定义本身就具有一定的主观性和模糊性，不同的用户对于什么是边界案例可能有不同的理解。

为了尽可能保证生成的边界案例的质量，IPC 系统采取了以下措施：

1. **使用高质量的 LLM:**  使用性能更强、理解能力更强的 LLM 作为样本生成器，可以提高生成的边界案例的质量。
2. **提供丰富的上下文信息:** 在生成边界案例时，为 LLM 提供尽可能丰富的上下文信息，例如任务描述、初始提示、少量样本以及历史提示等，可以帮助 LLM 更好地理解任务需求，生成更合理的边界案例。
3. **迭代优化:** IPC 系统采用迭代优化的方式，在每一轮迭代中，都会根据模型在边界案例上的表现对提示词进行调整，从而逐步提高生成的边界案例的质量。

总而言之，LLM 对边界案例的理解不一定完全正确，但通过采取上述措施，可以尽可能提高生成的边界案例的质量，使其能够有效地指导提示词的优化。

**此外，IPC 系统还允许用户参与到边界案例的生成和评估过程中，通过人工的方式对生成的样本进行筛选和修正，进一步提高边界案例的质量。**
```
