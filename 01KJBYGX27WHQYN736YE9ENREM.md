# 论文1

[硕士研究生学位论文_v1.6.pdf](/assets/01KJBYGX27WHQYN736YE9ENREM/%E7%A1%95%E5%A3%AB%E7%A0%94%E7%A9%B6%E7%94%9F%E5%AD%A6%E4%BD%8D%E8%AE%BA%E6%96%87_v1.6.pdf)

  - 图像的视觉特性
    - 尺度不变的 SIFT 局部特征
    - 快速特征提取的 SURF 特征
    - 描述图像梯度信息的 GIST 全局特征
  - 对实值向量进行二值化
    - 本文提出了名为“一元汉明嵌入”的哈希二值化算法

      - 通过一元编码将实值向量嵌入到汉明空间，用汉明向量之间的距离近似表达实向量之间的距离

      - 在汉明嵌入的过程中，本文通过联合最小化哈希过程中产生的量化误差和距离差，让哈希后的汉明向量最大化保留原始实值向量之间的相似性。

  - 对二值化索引进行优化 (本文提出了名为“带权组件哈希”的哈希索引算法)

    - 索引操作分为两个关键环节: “哈希值生成”以及“索引项检索”

    - 哈希值生成
      - 选取重要比特进行哈希, 而不是使用全部比特
      - (本文提出了“基于后验概率与互信息的比特选择方法”选择聚合二值向量中信息熵大的重要比特作为哈希值)
    - 索引项检索
      - (本文提出了“基于频率统计的自适应加权方法” 利用索引项中的统计信息自适应地给哈希表中的冲突项打分)

      - 为什么要打分?
  - 最近领算法历史:

    - 传统的准确的最近领算法

      - 维度灾难: 高维向量对于传统最近领算法, 复杂度会大幅上升, 接近于暴力搜索

    - 概率性的最近领算法: LSH
    - 大数据和短编码, 取代高维向量: 哈希二值算法

  - 哈希二值算法分类: 将向量变成二值向量
    - 数据非相关方法
      - 不需要使用任何训练数据，因而与数据独立

      - 局部敏感哈希 (LSH): 其基本的想法是将相似的数据项以很高的概率哈希成同样的哈希值
        - 局部敏感哈希算法已在理论上给出证明，当二值码的长度趋近无穷大时，二值码之间的汉明距离会与原始向量之间的距离无限地接近。

      - 转移不变核哈希 (SIKH)
        - 采用随机投影矩阵将实值向量投影到低维空间。不同的是，SIKH 用选择不变的余弦函数[43]去生成哈希值

    - 数据相关方法
      - 从训练数据中学习哈希函数，相比数据非相关方法能获得更紧凑更具有区分力的二值码

      - 通过定义优化目标, 直接生成投影矩阵
        - 谱哈希
        - 重构二值哈希
        - 最小损失哈希
          - 将实值向量对分为两类，一类是相邻的点对，另一类是非相邻的点对

          - 优化目标: 让相邻的点对获得较小的汉明距离，让非相邻的点对或者较大的汉明距离
      - 迭代量化(ITQ)算法

        - 先给出一个初始化的投影矩阵，然后通过一个正交化的旋转矩阵来进一步优化投影矩阵使投影向量与二值向量之间的量化误差最小

  - 哈希索引算法: 建立哈希表,并进行查找
    - 要解决的问题: 随着向量的维度升高, 索引算法的复杂度急速升高
    - 多表索引哈希(MIH): 
      - 使用分治方法, 分别解决: 查询半径k 和 二维向量维度m 的升高问题
      - 解决查询半径k增大的问题: 比如: 将向量分成k段, 只需检索每段k=1的近邻
      - 解决二维向量维度m 增大的问题: 将索引表改为排序的数组, 使用二分法检索, 而不使用空间换时间
    - 错误带权哈希 (EWH):
      - 错误带权哈希采用了与局部敏感哈希相同的方式，随机的选取二值向量的若干个维度构成一个哈希值

      - 在搜索过程中从错误地址中寻找冲突项并给寻找到的冲突项按照自定义的方式加权打分

      - 为搜索过程应该考虑到其它相差不明显的索引地址，并从中寻找查询向量的冲突项

      - 要点: 在哈希搜索到的目标位置和附近位置(错误位置)进行查找, 对结果进行打分评级
  - 典型的基于聚类的量化算法
    - 讨论的意义: 与哈希索引算法形成对比  

    - 二值空间的边界稠密性, (在汉明空间，无法找出一个能够明显地区分任意两点的界限), 因此基于聚类的算法不适合二值向量
    - 典型算法: 
      - 乘积量化
      - 最优化乘积量化
      - 局部最优化的乘积量化
  - 哈希二值方法 (通常向量的索引处理??)
    - 分为两阶段: 投影和量化
    - 问题: 普遍用的量化函数为 单比特符号函数, 会导致大量信息丢失
    - 解决: 本文设计更合理的量化方法(一元汉明嵌入哈希二值算法)，将投影后的每个实数值量化成多个元素，并用一元编码将其嵌入到汉明空间以保证距离结构的一致性
    - 目标函数: 采用两个因素: 
      - 量化误差: 量化前和量化后的数值误差
        - 所有向量, 在量化前后每个维度值的平方差
      - 原始空间的近邻结构: 需要保留原始空间的近邻性质  

        - 所有向量对, 在量化前后的空间里的距离的平方差
    - 通过动态规划, 将目标函数的求解降低复杂度 (论文中带有动态规划式的证明)
  - 聚合二值向量 (本章节讨论针对特定种类的向量的索引处理)
    - 聚合二值向量 是视觉领域的一种视觉特征, 以 二值化 Fisher Vector 为例

      - 聚合向量是由多个高斯函数 (多组件) 聚合而成
    - 本文提出算法: 带权多组件哈希索引算法
      - 如何生成哈希值
        - 首先选择后验概率大的高斯函数 (后验概率大的函数, 在统计上更有影响力)

        - 然后从被选的哈希函数中选择总信息熵最大的比特 (信息熵最大的比特, 包含更多的信息)

      - 如何检索索引项
        - 本文提出: 基于频率统计的自适应加权方法
        - 将哈希搜索的范围 扩展到所有汉明距离与查询向量的哈希值在 r 以内的地址
        - 给每个冲突槽赋予权重, 累加权重作为冲突向量的权重

# 论文2

[毕业论文_黄祎程_20180502 (2).pdf](/assets/01KJBYGX27WHQYN736YE9ENREM/%E6%AF%95%E4%B8%9A%E8%AE%BA%E6%96%87_%E9%BB%84%E7%A5%8E%E7%A8%8B_20180502%20%282%29.pdf)

  - 本文针对于小目标搜索的场景
    - 小目标搜索: 在数据库中找到所有包含这个物体的图像及物体在图像中相应的位置, 而不是 传统的基于全图特征表达的搜索技术
      - AI算法如何解决小目标搜索和的问题? 
    - 使用 分块多叉字典树 (MBNT) 加速汉明空间中精确r近邻搜索
      - 在汉明空间内, 常用的全局视觉特征（例如 Fisher Vector 和 VLAD）通常维度很高, 适合使用近似搜索

      - 近年来出现了少量针对短特征（≤ 256 比特）的精确搜索算法

    - 在 MBNT 结构基础上加入了属性向量并设计带过滤器的搜索算法

    - 比 现有最好的精确搜索算法 Multi-Index Hashing (MIH) 快 2 至 4 倍
  - 在紧凑向量的情况下, 精确搜索 v.s. 近似搜索  

    - 近似搜索的算法, 对向量进行随机分隔. 精确搜索的算法, 对向量进行完全分隔采样
    - 对于紧凑向量, 精确搜索的算法, 只需要少数几次采样, 就可以覆盖紧凑向量; 而近似搜索的算法需要更多的采样次数  
  

  - ![image2022-2-8 23:50:59.png](/assets/01KJBYGX27WHQYN736YE9ENREM/image2022-2-8%2023%3A50%3A59.png)
  - 分块多叉字典树 (MBNT)
    - 采用 object proposal 算法来产生潜在目标局域，并利用哈希算法来生成紧凑二值特征. 将每一个 (这步骤未包含在论文中)

    - 将特征划分为m个子串, 每子串构成一个MBNT
    - 将子串的前b位放在字典树中(字典树每节点包括c位 (用来缩减树的高度)), 其余位在叶子节点用哈希表或者线性查询  

      - ![image2022-2-13 16:55:50.png](/assets/01KJBYGX27WHQYN736YE9ENREM/image2022-2-13%2016%3A55%3A50.png)
    - 搜索r近邻, 在每个子串的MBNT里搜索 r/m 近邻
    - 优化细节: 
      - 我们使用静态结构（即满 2c 叉树）来实现 MBNT。字典树节点在内存中连续存储。叶子节点的容器同样使用静态数组实现。

      - 一个 1 比特的二值标记位用于标记每一个节点是否真实存在。当 c = 4 时，C++中 16 比特的 short int 类型恰好可以存储一个节点的 16 个儿子的信息。

      - 我们使用广度优先搜索，而不是深度优先搜索来实现在线搜索时的遍历。配合静态结构的实现，广度优先搜索的节点访问顺序与节点在内存中的存储顺序相一致，从而最大化地利用了空间局部性。

  - MBNT中, 增加标量条件过滤器
    - 三种过滤器
      - 类别
      - 尺度
      - 位置
    - 三种过滤器的条件都是有界的, 所以可以表达成二进制串 (可穷举所有条件)
    - MBNT的中间节点, 增加条件过滤器, 是其子节点的过滤器的并集
