# 总体结构

  - 启动入口是main.py
    - 配置初始化: configure.env.init_app_env
    - 启动服务: controller.webserver.start
      - API服务的router: controller.chat.routers
        - 路径"/api/chat": controller.chat.chat
  - 服务入口: controller.chat.chat  

    - 消息类型 == Diagnosis, 并且没有历史, service.chat.ChatService.async_query_method (调用Plan)
      - service.agent.plan.PlanAgent.async_query_with_method
    - 消息类型 == Diagnosis, 并且有历史, service.chat.ChatService.async_query_method_continue (调用Method)
      - service.agent.method.MethodAgent.async_query_question
    - 其他类型 (消息类型==Reason), service.chat.ChatService.async_query_reason (调用Reason)
      - service.agent.reason.ReasonAgent.async_query_question
  - 调用顺序: Reason -> Plan -> Method

# Reason

  - 通用Retriever: 
    - 包括 mcontriever_embeddings 和 openai_embeddings 两个embedding构成的retriever
    - 对于召回的文档, 使用RRF (Reciprocal Rank Fusion algorithm) 进行重排序
  - Reason: ReasonAgent.async_query_question

  -     - 召回文档的提示词: "`我需要了解这个问题出现的原因: `" + question + reason
    - 召回文档: 通用Retriever + RagasRerank 进行重排序, 使用ContextualCompressionRetrieverWrapper将每5个召回文档合并成一个
    - 对于每一个召回的文档, 调用reason_chain, 评估每个文档的原因
    - 将各个reason拼接起来 (format_and_contact_reason_list), 交给大模型进行总结 (总结用的提示词)

# Plan

  - Plan: ReasonAgent.async_query_with_method

  -     - 召回文档的提示词: "我需要一个用于诊断这个问题的排查计划: " + question + reason
    - 召回文档: 通用Retriever + RagasRerank 进行重排序, 使用ContextualCompressionRetrieverWrapper将每5个召回文档合并成一个
    - 通过大模型, 将召回的文档 组织成 排查树

生成排查树的提示词: 

```
外部文档:
###
{context}
###
----------------
目前我遇到了一个问题，并且给出了一个可能导致该问题的原因。
您的任务是根据我提供的可能原因，帮我生成一个高效的排查计划。
答案计划用Mermaid格式表示。 答案除了Mermaid 不要包含其他内容。
答案必须只包含一个subgraph图. 否则答案将会出错.
排查计划一定要详细给出每一个步骤包含要检查的信息。
我有一个mermaid的样例,样例中缺少了内容但请参考例子中的结构,输出的是一个子图不要生成多个子图,不要存在环:

graph TD \n subgraph 排查xx\n A --> B[xx]\n B --> |x| C[xxx]\n C --> |x| D[结束排查]\n end

{question}

``` 

# Method

  - 使用原始问题进行文档召回, 然后用Cohere rerank
  - 使用Method提示词, 给出诊断过程

```
外部文档:
###
{context}
###
----------------
您是一名资深的DBA，您的任务是根据给定的排查计划 "逐步的" 和我进行 "多轮对话" 来 "引导我" 解决我的问题。
我输入的排查计划是一张图。
图的名字代表一个场景(例如: subgraph 图的名字)。
图中每个节点代表一个排查方法(例如: x --> x[排查方法]) 。
请严格按照我计划图的每一个节点进行排查。
请注意，不要一次性将所有节点代表的排查方法和相关指令放入答案中。
需要强调的，您每次只能提出一个节点代表的排查方法。请逐步的向我进行引导。
排查指令中，根据当前一个节点代表的排查方法，请输出完整的指令来告诉我应该怎么操作。
最重要的，排查指令中，一定不能涉及到排查计划图中其他节点的相关内容。输出一次下述答案样例，立即停止答案生成。
答案的最后不要出现 "注意" 的字样。
输出形式样例:
    排查场景:图的名字\n
    排查方法:[排查方法]\n
    排查指令:__________\n
``` 
