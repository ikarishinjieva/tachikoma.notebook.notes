要解决的问题: 

  - 缺乏一个通用的框架来整合语言反馈和非语言奖励信号，以训练能够在不同任务和环境中适应的 LLM agents。

```
语言反馈 (Linguistic Feedback) 指的是用自然语言形式提供的反馈信息，例如：
指令 (Instructions): 告诉模型要做什么，例如 "写一篇关于夏天的短文"。
演示 (Demonstrations): 提供完成任务的示例，例如给出几篇关于夏天的短文作为参考。
评价 (Evaluations): 对模型的表现进行评价，例如 "这篇文章写得很好" 或者 "这篇文章还需要改进"。

非语言奖励信号 (Non-linguistic Reward Signals) 指的是不用自然语言形式提供的反馈信息，而是一些数值化的信号，例如：
分数 (Scores): 例如在问答任务中，模型回答正确可以得到 +1 分，回答错误可以得到 -1 分。
胜负 (Win/Loss): 例如在游戏中，模型获胜可以得到 +1 的奖励，失败可以得到 -1 的奖励。
成功率 (Success Rate): 例如在机器人控制任务中，模型成功完成任务的次数越多，奖励越高。
```

    - 语言反馈: 类似于策略生成, 通过微调后的LLM可以生成
    - 非语言奖励信号: 类似于策略选择, 通过学习算法来进行
  - 本文提出的LTC框架, 将 策略生成 和 策略选择 合并进行训练

LTC框架: 

  - 类似于启发搜索 + 奖励函数
    - 启发搜索阶段: 通过LLM进行策略生成, 将策略路径 保存到 "通用缓存区", 并选择策略+获取反馈
    - 学习阶段: 对"通用缓存区"进行学习和决策, 使用奖励函数, 更新参数
      - 奖励函数需要兼容两个目标: 
        - 损失函数通常包含两个部分：一部分是语言模型的损失函数，用于保证生成文本的流畅性和一致性；另一部分是强化学习的损失函数，用于最大化 agent 获得的累积奖励。
      - 更新模型参数: 利用梯度下降等优化算法更新 LLM 的模型参数。

**2\. 迭代训练:**

  - **(1) 探索阶段:**
    - **观察状态:** Agent 观察当前的环境状态。
    - **生成动作:** Agent 利用 LLM 根据当前状态和目标生成一系列可能的动作。
    - **选择动作:** Agent 根据一定的策略（例如贪婪策略、ε-greedy 策略等）从生成的候选动作中选择一个执行。
    - **执行动作:** Agent 在环境中执行选择的动作。
    - **接收反馈:** Agent 接收环境返回的反馈信息，包括新的状态、奖励以及可能的语言反馈。
    - **存储数据:** 将本次交互过程产生的数据存储到通用缓冲区中。
  - **(2) 更新阶段:**
    - **采样数据:** 从通用缓冲区中随机采样一批数据。
    - **计算损失函数:** 根据采样数据和选择的强化学习算法（例如 PPO、A2C 等）计算损失函数。损失函数通常包含两个部分：一部分是语言模型的损失函数，用于保证生成文本的流畅性和一致性；另一部分是强化学习的损失函数，用于最大化 agent 获得的累积奖励。
    - **更新模型参数:** 利用梯度下降等优化算法更新 LLM 的模型参数。
