# 主要思路

  1. 使用五种人类思维动作, 作为蒙特卡洛树搜索 (MCTS) 中的节点扩展方法
     1. 动作 1：提议一步思路。针对给定问题，该动作会让 LLM 基于已有的推理步骤生成接下来的一步思路。  
动作 2：提议余下的思路步骤。该动作与标准 CoT 一样，能实现「快速思考」，从而解决只需少量步骤的简单问题。给定已经生成的推理步骤，它会让 LLM 直接生成剩余步骤，直到得到最终答案。  
动作 3：提议下一个子问题及其答案。  
动作 4：再次回答这个子问题。考虑到动作 3 有可能无法正确回答对应的子问题，因此这个动作的作用是再次回答它。  
动作 5：重新表述问题 / 子问题。这个新动作是以更简单的方式重新表述该问题。具体来说，这里是让 LLM 清晰列出问题陈述中的所有条件。
  2. 用另一个小模型, 来验证MCTS中某一条推理路径是否正确

# 其他信息

rStar 方法的具体步骤如下：

**第一阶段：推理轨迹的自我生成 (Self-Generating Reasoning Trajectory)**

  1. **问题输入:** 将待解决的推理问题 x 输入给目标小型语言模型 (SLM1)。

  2. **MCTS 增强 SLM1:** 利用蒙特卡洛树搜索 (MCTS) 算法增强 SLM1，使其能够探索多种推理路径。 MCTS 构建一个搜索树，树的根节点代表问题 x。

  3. **执行多种类人推理动作:** rStar 定义了一个丰富的类人推理动作集 A = {A1, A2, A3, A4, A5}，在 MCTS 的每一步中，SLM1 从 A 中选择一个动作 `a_i` 执行。

     - A1: 提出一步思考。
     - A2: 提出剩余的思考步骤。
     - A3: 提出下一个子问题并给出答案。
     - A4: 再次回答子问题 (使用 Few-shot CoT)。
     - A5: 重新表述问题/子问题。
  4. **生成推理步骤:** 根据选择的动作 `a_i` 和当前推理状态（之前的推理轨迹 x⊕s1⊕s2⊕...⊕s_(i-1)），SLM1 生成下一个推理步骤 `s_i`。 搜索树中的边代表动作，节点代表推理步骤。

  5. **构建推理轨迹:** 从根节点到叶节点（终止节点 `s_d`）的路径构成一个候选推理轨迹 t = x⊕s1⊕s2⊕...⊕s_d。

  6. **MCTS 迭代:** MCTS 通过选择、扩展、模拟和反向传播四个步骤进行多次迭代，生成多条候选推理轨迹 T = {t1, t2, ..., tn}。

     - **选择:** 使用 UCT (Upper Confidence Bounds applied to Trees) 算法选择节点，平衡探索和利用。UCT公式为：UCT(s, a) = Q(s, a) / N(s, a) + c * sqrt(ln Nparent(s) / N(s, a)) 。其中，Q(s, a) 为动作 a 在状态 s 下的奖励值，N(s, a) 为访问次数，Nparent(s) 为父节点访问次数，c 为平衡探索和利用的常数。
     - **扩展:** 选择一个未探索的节点进行扩展。
     - **模拟:** 使用默认的 rollout 策略进行模拟，到达终止节点。
     - **反向传播:** 根据终止节点的奖励值更新路径上所有节点的 Q 值。终止节点的奖励值基于自一致性多数投票的可能性（置信度）。中间节点的奖励值累加终止节点的奖励值。

**第二阶段：推理轨迹的选择 (Reasoning Trajectory Selection)**

  1. **引入判别器 SLM2:** 引入另一个 SLM (SLM2) 作为判别器，对第一阶段生成的候选推理轨迹进行验证。

  2. **部分轨迹提示:** 对于每个轨迹 t = x⊕s1⊕s2⊕...⊕s_d，随机选择一个步骤 i (i < d)，将轨迹的前 i-1 步 (x⊕s1⊕s2⊕...⊕s_(i-1)) 作为提示输入给 SLM2。

  3. **SLM2 补全轨迹:** SLM2 根据提示补全剩余的推理步骤，并生成最终答案。

  4. **相互一致性验证:** 比较 SLM2 补全的答案与 SLM1 生成的原始轨迹 t 的最终答案是否一致。如果一致，则认为轨迹 t 是有效的。

  5. **SLM1 最终选择:** SLM1 从所有有效的轨迹中选择最终轨迹。每个轨迹的最终得分是其累积奖励值与终止节点置信度的乘积。选择得分最高的轨迹作为最终解决方案。

**重要知识:**

  - **蒙特卡洛树搜索 (MCTS):** 一种强大的搜索算法，用于在具有高分支因子的搜索空间中找到最佳决策。
  - **UCT (Upper Confidence Bounds applied to Trees):** 一种用于 MCTS 的策略，平衡探索新节点和利用已知高奖励节点。
  - **自一致性 (Self-Consistency):** 一种通过多次采样推理路径并选择多数答案来提高推理准确性的方法。
  - **Few-shot CoT (Chain-of-Thought):** 一种通过提供少量示例来引导 LLM 生成推理步骤的方法。
  - **类人推理动作:** rStar 的核心创新之一，通过模拟人类的推理行为，例如分解问题、提出子问题、重新表述问题等，来增强 SLM 的推理能力。
  - **相互一致性 (Mutual Consistency):** rStar 的另一个核心创新，通过引入判别器 SLM 进行交叉验证，提高了选择正确推理轨迹的准确性。

rStar 的关键在于其丰富的类人推理动作集和相互一致性验证机制，这两个创新点有效地解决了 SLM 在推理过程中遇到的探索解空间和评估推理质量的难题。
