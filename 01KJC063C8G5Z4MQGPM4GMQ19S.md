# 实验设计

### **模仿学习实验核心步骤清单**

**任务：** 机械臂末端移动到固定目标点。

  1. **环境搭建 (Setup)**

     - 在 Isaac Sim 中创建场景。
     - 放入 Franka 机械臂。
     - 在场景中放置一个可见的物体（如小球）作为固定的**目标点** 。
  2. **专家算法 (Expert)**

     - 编写一个Python脚本。
     - 该脚本能获取机械臂**当前状态** （关节角度、末端位置等）。
     - 使用**逆向运动学 (IK)** 计算出能让机械臂朝向目标点移动的**目标关节角度 (专家动作)** 。
  3. **数据收集 (Data Collection)**

     - 编写一个循环脚本，用于生成训练数据。
     - 在循环开始时，**随机化** 机械臂的初始姿态。
     - 在循环的每一步：
       - 获取当前**状态 (State)** 。
       - 调用专家算法得到**动作 (Action)** 。
       - 将 `(State, Action)` 这对数据保存下来。
       - 执行这个动作，进入下一步。
     - 将成千上万个 `(State, Action)` 数据对保存成一个或多个文件（如 `.hdf5` 或 `.npz`）。
  4. **模型训练 (Training)**

     - 设计一个简单的神经网络模型（例如，一个多层感知机 MLP）。
     - 模型的**输入** 维度与 `State` 向量一致，**输出** 维度与 `Action` 向量一致。
     - 加载第3步收集的数据集，将其分为训练集和验证集。
     - 使用**均方误差 (MSE)** 作为损失函数，进行监督学习训练。
     - 保存训练好的模型权重。
  5. **评估与部署 (Evaluation)**

     - 编写一个新的评估脚本。
     - 加载第4步训练好的模型。
     - **不使用专家算法** ，在循环的每一步：
       - 获取当前**状态 (State)** 。
       - 将 `State` 输入模型，得到**预测动作 (Predicted Action)** 。
       - 执行这个 `Predicted Action`。
     - 从不同的随机初始位置开始测试，观察机械臂能否独立完成移动到目标点的任务。

# 结论

  - 当前代码给出了一个简单的Demo, 如何从Isaac Sim中获取训练数据, 使用数据训练模型, 并回到Isaac Sim中回放
  - 其没有解决的问题: 
    - 从Isaac Sim中, 只获取了简单的规划数据, 并没有进行实际的运行
      - 这也就出现了轨迹0这种 实际上无法执行的规划
    - 训练数据, 只对固定环境有效, 只采用了机械臂的状态值, 并不具备环境感知能力
    - 训练模型是最简单的模型, 而且并没有使用大规模数据进行训练 (只验证了通过训练数据可以调度机械臂)
  - 一些重要的概念问题需要讨论
    - 目前使用的采集数据的方式, 是在规划路径上, 每0.1秒进行差值
    - 在回放时, 无法按照每0.1秒进行回放, 而是要按照每1秒进行回放, 否则机械臂无法到位
    - 即使按照每1秒进行回放, 整个轨迹也很卡顿, 不像是一个连续的轨迹
    - 如果改变回放形式, 那么就需要改变模型的输入和输出形式

在代码中, 增加了README和机器人学习控制_核心问题总结.md, 对当前的一些问题进行了讨论

# 下一步

### **推荐的融合学习路径**

#### **第一阶段：在仿真中成为“任务专家” (Master the Task in Sim)**

**目标** ：利用传统工具，在仿真中完美解决“拿取和倾倒”任务。这是后续所有工作的基础。

**核心任务** ：执行你的**想法2** 。

  1. **搭建完整仿真场景** ：在Isaac Sim中创建包含烧杯、液体、目标容器的场景。
  2. **任务分解与脚本化** ：编写Python脚本，定义“抓取-移动-倾倒-复位”的关键路径点(Waypoints)。倾倒过程可能需要定义一个姿态变化的轨迹。
  3. **生成并验证专家轨迹** ：使用cuMotion连接这些路径点，生成平滑的完整轨迹。利用Isaac Sim的流体仿真功能，反复调试倾倒轨迹，直到液体能被平稳、准确地倒出。
  4. **产出** ：
     - 一个能够稳定运行“拿取-倾倒”任务的仿真环境。
     - 一个调用cuMotion完成此任务的、逻辑清晰的Python脚本。
     - **(重要副产品)** 在这个过程中，你可以记录下大量的**专家演示数据（状态-动作序列）** ，这将是你未来训练高级学习模型（路径A）的宝贵财富。

**为什么先做这个？**

  - **低风险，高回报** ：在进入高风险的真实世界前，你在一个安全的环境里，已经对任务的完整逻辑和几何约束了如指掌。
  - **定义问题** ：你清楚地知道了“成功”是什么样的。这为你后续评估学习模型提供了基准。

#### **第二阶段：攻克“虚实鸿沟” (Bridge the Sim-to-Real Gap)**

**目标** ：将一个**简单的、确定性的** 任务，成功从仿真迁移到真实世界。

**核心任务** ：执行你的**想法3** ，但要带着系统性的方法。

  1. **搭建数字孪生 (Digital Twin)** ：精确测量真实世界中机械臂、工作台、相机（如果使用）的位置，并在Isaac Sim中复刻一个一模一样的场景。这是成功迁移的第一步。
  2. **硬件接口与安全** ：
     - 在你的电脑上配置好真实机械臂的ROS2驱动。
     - 编写一个最简单的脚本，比如让机械臂的某个关节来回摆动，以验证通信链路是通畅的。
     - 在你的控制代码中加入**严格的速度限制和急停逻辑** 。
  3. **标定！标定！标定！** ：这是本阶段**知识密度和风险最高** 的部分。学习并实践：
     - **眼在手上 (Eye-in-Hand)** 或 **眼在手外 (Eye-to-Hand)** 的相机-机械臂标定。
     - 如果你暂时不用相机，至少要完成**工作空间标定** ：在真实世界中定义一个坐标系原点，并让机器人知道这个原点在它的基坐标系下的位置。
  4. **执行简单任务** ：使用cuMotion，在你的**数字孪生环境** 中规划一个简单的Pick and Place轨迹。然后，将这条轨迹发送给**真实机械臂** 去执行。观察并调试，直到真实动作与仿真基本一致。

**为什么第二步做这个？**

  - **隔离变量** ：你使用的是可靠的cuMotion，而不是不完美的学习模型。这样，如果真实世界执行失败，你可以把问题范围缩小到**硬件、通信或标定** 上，而不是怀疑规划算法本身。这极大地降低了调试难度。
  - **直面最大风险** ：你正在正面解决Sim-to-Real中最困难、最容易出错的部分。一旦你掌握了标定和系统集成的技能，后续部署任何算法都会变得更加顺利。

#### **第三阶段：用“实战经验”反哺“模型学习” (Reinforce Learning with Real-world Insights)**

**目标** ：回到你的学习模型，利用前两阶段的经验和数据，解决其根本性缺陷。

**核心任务** ：执行你的**想法1** ，但现在你有了更强的武器。

  1. **升级控制框架** ：根据你的总结文档，将控制方式从位置控制切换到**速度控制** ，并实现**滚动窗口(MPC)**的执行逻辑。这是让学习模型变得平滑和鲁棒的关键一步。
  2. **训练高级模型** ：
     - 使用你在**第一阶段** 收集的“倾倒任务”专家数据。
     - 放弃简单的MLP，实现一个**Transformer序列模型** ，让它学习预测未来一段时间的动作序列。
  3. **在仿真中验证** ：将你新训练的Transformer模型，部署在新的MPC+速度控制框架下，在**第一阶段** 的仿真环境中进行测试。目标是让学习模型能够独立复现接近cuMotion专家水平的平滑倾倒动作。

**为什么现在做这个？**

  - **目标明确** ：你已经知道一个“好”的倾倒策略是什么样的（来自第一阶段）。
  - **数据充足** ：你拥有了高质量的训练数据。
  - **基础扎实** ：你即将掌握真实世界的部署经验，这会让你在设计学习系统时，更多地考虑现实约束。

#### **最终阶段：王者降临**

当你的学习模型在仿真中表现优异后，你就可以尝试用它替换cuMotion，在你的真实机械臂平台上，去执行真正的、由AI驱动的“拿取-倾倒”任务了。因为你已经在第二阶段解决了底层的硬件和标定问题，所以这一次你可以更专注于算法本身的表现。

先用传统工具, 完成仿真环境中的液体倾倒, 并且考虑使用跟实际相同型号的机械臂
