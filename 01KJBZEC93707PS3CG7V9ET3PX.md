# 主要思路

项目: <https://github.com/keirp/automatic_prompt_engineer>

生成候选提示词, 并进行beam search找到最优提示词. 

主要创新在生成候选提示词的方法: 

```
APE 生成候选指令的过程可以分为两个阶段：**初始候选指令生成** 和 **迭代优化过程中的候选指令生成**。

**阶段一：初始候选指令生成**

这一阶段的目标是生成一组多样化的初始候选指令，为后续的搜索提供起点。

1. **选择生成模式:** APE 提供两种主要的生成模式：

   * **正向生成 (Forward Generation):** 将准备好的少量样本和一些引导性文字拼接成一个完整的 prompt，引导 LLM 从头开始生成指令。
     * **示例 Prompt:**  "以下是一些任务示例：{演示样本}。请描述这个任务的目标："
   * **反向生成 (Reverse Generation):** 在 prompt 中预留一个空位，并将演示样本放置在空位前后，引导 LLM 填补空位，生成指令。
     * **示例 Prompt:** "请阅读以下示例，并补全指令：{演示样本的一部分} [指令] {演示样本的另一部分}"

2. **构建 Prompt:**  根据选择的生成模式，将准备好的少量样本填充到相应的 prompt 模板中。

3. **调用 LLM 生成指令:** 将构建好的 prompt 输入预训练的 LLM  (通常与目标 LLM 相同或更大)，让其生成多个候选指令。

4. **增强多样性 (可选):**  可以通过以下方式增加候选指令的多样性：
    * **多次采样:**  使用相同的 prompt，多次调用 LLM 生成指令。
    * **调整 LLM 生成参数:**  例如，可以调整 temperature 参数来控制生成结果的随机性，或调整 top_k 参数来控制 LLM 只从概率最高的 k 个词中选择下一个词。

**阶段二：迭代优化过程中的候选指令生成**

这一阶段的目标是利用已有的评分信息，生成更优秀的候选指令。

1. **选择需要优化的指令:**  根据评分函数，选择当前得分较高，并且有优化潜力的指令进行下一步优化。

2. **生成新的候选指令:**  APE 主要使用以下方法生成新的候选指令：

   * **基于语义相似度重采样 (Semantic Similarity-based Resampling):** 
      * 将选中的指令输入 LLM，并引导其生成语义相似的变体指令。
      * **示例 Prompt:** "生成一个与以下指令意思相近，但表述不同的指令：{需要优化的指令}"

3. **其他变体生成方法 (可选):**  除了语义相似度重采样，还可以使用其他方法生成新的候选指令，例如：
    * **随机替换词语:**  随机选择指令中的一个或多个词语，将其替换为语义相似的其他词语。
    * **增加或删除词语:**  在不改变指令核心语义的情况下，尝试增加或删除一些修饰性的词语。

需要注意的是，APE 在实际应用中，通常会根据具体任务和数据集的特点，选择合适的生成模式、prompt 模板以及变体生成方法，以达到最佳的指令生成效果。
``` 

对于以上两点的质疑: 

```
你的这两个问题非常 insightful，直击 APE 算法设计的核心！

**1. 为什么要同时使用正向生成和反向生成？**

   * **原因在于不同的 LLM 模型能力不同，以及适应不同的任务需求。**

   * **正向生成** 更加依赖于 LLM 从零开始组织语言的能力，适合于那些 LLM 已经具备较强语言生成能力的任务，例如文本摘要、故事生成等。这种方法的优点是可以生成更富有创造性的指令，但缺点是生成的指令质量可能不够稳定，而且容易偏离目标任务。

   * **反向生成**  则更像是在做完形填空，LLM 只需要填补空缺的部分，生成难度相对较低。这种方法更适合于那些需要 LLM 理解特定上下文语义的任务，例如问答、代码生成等。其优点是生成的指令更加简洁、准确，但缺点是缺乏一定的创造性。

   * **APE 同时提供两种生成模式，是为了更好地适应不同 LLM 的能力特点，以及不同任务的需求。**  在实际应用中，可以根据具体情况选择合适的生成模式，或者将两种模式结合使用，以获得最佳的指令生成效果。

**2. 基于语义相似度重采样，会生成语义相似的 prompt，那不会产生大的 prompt 变更，会陷入局部最优解吧？**

   * **你担心的这个问题确实存在，但 APE 也采取了一些措施来缓解这个问题。**

   * **首先，语义相似度重采样只是 APE 生成新候选指令的一种方式，而不是唯一的方式。**  如前所述，APE 还可以使用其他变体生成方法，例如随机替换词语、增加或删除词语等，这些方法可以生成语义差异更大的候选指令。

   * **其次，APE 在进行语义相似度重采样时，会控制相似度的大小。**  如果相似度设置过高，则生成的候选指令会过于相似，容易陷入局部最优；如果相似度设置过低，则生成的候选指令可能与原始指令的语义相差太大，失去指导意义。APE 会根据经验选择合适的相似度阈值，以平衡探索效率和探索范围之间的关系。

   * **最后，APE 会结合全局评分信息来指导搜索方向。**  即使在局部搜索中，APE 也会优先选择那些得分更高的候选指令进行下一步优化，从而避免陷入局部最优解。

   * **总而言之，基于语义相似度重采样是 APE 利用 LLM 生成能力的一种有效方式，但也存在一定的局限性。** APE 通过结合其他变体生成方法、控制相似度大小以及利用全局评分信息等措施，来缓解局部最优解的问题，提高搜索效率。
```
