# 20260316 与 苏州 的沟通, 获得的反馈

- 比起上一个版本, 更像人类专家思路

  - 专家认为这套系统的最大帮助是 广度覆盖

- 在 异构烷烃 组

  - run10, 并没有将异构烷烃组 单独列出来 (之前的 run 有的是单独列出来的)
  - AI提到了 isopar 等商业产品, 专家想知道 AI是怎么获得的这部分知识 (还提到了 “日本初光” )
  - 对于异构烷烃的选择, 专家的观点是: 同样分子量下，闪点比正烷烃 高一点，体积大，溶胀低. 因此 用高分子异构烷，替换低分子正烷. (这种专家思路, 是从本质论, 获取一种人类对趋势的思考)

- 对于 酯类的选择

  - 专家的观点:
    - 极性酯 因为其 具有极性, 会降低击穿. 所以优先考虑非极性酯
    - 非极性酯, 除了链式对称, 也可以考虑四角对称, 季戊四酯

- 给了一份对现有溶剂的反向工程的报告: [972c63c6_1.pdf](/assets/01KKX5Q3YNGMB8W04Y6MADZFFN/972c63c6_1.pdf)

- 对于KPI:

  - 是否将指标拆开, 拆成: 施工现场 有感知的指标, 以及 施工现场 没有感知的指标. 这个拆分主要是考虑 现有产品是否是在实验中作弊, 那么我们在做材料研发时, 就不知道哪个指标是可以降级的
  - 是否要考虑: 蒸发性
  - 对于闪点, 专家提示: 溶液闪点, 是由成分中最低的闪点 来做主要决定的 (?)
  - 对于 运动粘度的计算, 各方法偏差很大, 需要讨论测量方法 (这一部分 专家对运动粘度的计算 是找了一些额外文献的?)

- 专家对他的探索范围做了总结 (从闪点/粘度/BDV/溶胀/蒸发/对环境(水汽)的敏感度):

  - 14-15 正烷+对称酯
  - 异构 C20 (提闪点) ，纯品或者配酯
  - 14-15 正烷 + 季戊四酯
  - 异构 C20 + 季戊四酯

# 思考

最大的收获是: 从物化本质的角度, 对可能的路线进行规划:

- 正烷烃 v.s. 异构烷烃
- 极性酯 v.s. 非极性酯

这都是基于人类 “概念” 的方向性推理

# 对反向工程报告的解读

这里是最大的收获, AI分析后认为:

```markdown
### 报告核心要点速览：
1. **主要内容**：该竞品配方主要由 重质双萜二醇（约32%）、宽馏分正构烷烃（C11/C12/C15，约27%）、乙酸丁酯（约14%）、芳香烃（二甲苯/乙苯，约12%）以及橡胶增塑剂（DOTP/DEHP，约6%）组成。
2. **对 run10 的修正**：竞品大量使用了轻质烷烃（C11/C12）和低闪点的乙酸丁酯，这说明实际产品可能并未执着于极高的闭杯闪点（>110°C），而是更看重清洗力和挥发速度的平衡。
3. **run10 未发现的类型**：
   - **橡胶增塑剂（DOTP等）**：这是最大的盲点突破。竞品通过添加增塑剂来“反向补偿”强溶剂对 EPDM 橡胶的抽提破坏，而不是像 run10 那样一味寻找不伤橡胶的温和溶剂。
   - **芳香烃类**：为了强效去污，竞品依然保留了12%的二甲苯类芳烃。
   - **重质树脂/成膜剂**：占比高达32%的双萜类物质，可能用于形成绝缘保护膜。
4. **后续 run 的参考意义**：建议后续探索从“寻找单一完美溶剂”转向“复合体系设计”，特别是引入 **“强效清洗剂 + 橡胶增塑剂补偿”** 的新范式，并重新向需求方确认闪点红线的实际工业宽容度。
```

最大的问题是: 之前的流程 偏重于 主体的选择 (这不是提示词的导向, 但流程设计让模型偏向于这个方向)

预期将流程改往 “复合体系设计”的方向

机制, 每个机制按文件整理如下:

```

02_新架构实现计划.md
03_runtime_SOP与广深算子扩展讨论.md

05_runtime_SOP设计.md
06_广深算子设计.md
07_路线推演设计.md
08_审查员体系设计.md
09_Predictor设计.md

11_界面投影协议.md
12_价值观分层与应用原则.md

```

其中:

1. 线性处理样例, 是将前12项组装起来, 查看线性过程是否正确
2. GPT-5会问一些设计倾向的问题, 结果记录在 v2最终确认清单

然后让 gemini-cli 进行生成, 将生成结果送给GPT-5, 进行多轮迭代, 重点是让GPT-5 根据形容词和run10的差异, 将v1的提示词的各个方面, 逐步迁移到v2中

v2版本的重点变化:

- 广深算子: 将广度和深度搜索, 更多用于各个环节, 而不只是作为一个特殊阶段
- 路线推演: 模仿专家, 先进行趋势推演, 再由数值求证
- 整体阶段, 从 寻找主材 切换到 根据缺陷打补丁

碰到的问题:

- research\_[tool.py](http://tool.py) 的limit很小, 3或5, 需要更大
  - 已修改
- 以强调使用开杯闪点, 但仍需要强调是室内场景
- research_tool 关键字用的都是中文

放弃v2版本, 经过多轮测试, 其搜索深度完全比不上v1, 其根本原因是: 

- v2的处理流程, 已经脱离了探索树的结构, 所以在线性流程上, 会过早收敛 (树形结构是先出生, 后深度研究, 就会好一些)

从v1版本上, 重新逐个问题解决, 放到v3中: 

- 搜索宽度: 
  - 使用现有清洗液的反向工程结果 去比较, 发现当前流程的问题: 集中于主材, 考虑的方向也是找到温和的主材, 而完全不碰 激烈+修复 的方法.
  - 想了一些办法让其脱离只研究主材, 强制搜索一些辅助材料思路: 但其对于KPI的理解是 每一种成分的KPI都要达标或接近, 在搜索阶段完全就不会去碰 KPI偏离过大的成分
  - 改为完全的BFS, 但凡研究过程中碰到的思路, 都记下来, 然后强制做 本质+辅助材料的研究, 并且不允许淘汰路线
    - 测试结果, 对于每一个可能, 都能进行完整搜索, 但能扩展出的思路, 还是 主材+辅助材料, 而不能做出复杂配方
