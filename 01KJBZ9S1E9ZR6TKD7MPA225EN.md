# 主要思路

  - 目标: 优化LLM生成代码的可靠性
  - 两个关键组件
    - 目标相似性调整 (TST): 通过结构相似性 (而不是词句相似性), 找到相似代码的片段
    - 受约束语义解码 (CSD): 通过定义一些约束, 让代码遵循这些约束, 消除代码实现的错误
      - 引入概念"Brzozowski 导数", 就是在一定代码前缀+约束下, 后面还能使用的代码token的集合
      - "Brzozowski 导数"举例

好的，我们来具体看一下在 CSD 中，如何结合 Brzozowski 导数来实现代码约束。

## 例子：生成简单的算术表达式

假设我们要生成简单的算术表达式，它由数字、加号 (+) 和乘号 (*) 组成，并且满足以下约束：

* 表达式必须以数字开头和结尾。  
* 运算符 (+, *) 不能连续出现。

我们可以用以下正则表达式来表示这个语言 L：

```  
L =  ( '+'  | '*'  )*   
```

其中：

* `` 表示任意数字。  
* `( ... )*` 表示括号内的模式可以重复零次或多次。  
* `|` 表示 "或" 的关系。

### 场景 1：程序前缀为空字符串

此时，我们需要计算 L 对空字符串的导数，也就是 L 本身。根据正则表达式，合法的起始符号只能是 ``，也就是任意数字。

### 场景 2：程序前缀为 "1"

现在，我们需要计算 D("1", L)。根据 Brzozowski 导数的定义，我们从 L 中移除 "1" 前缀，得到：

```  
D("1", L) = ( '+'  | '*'  )*   
```

此时，合法的后续符号可以是 '+' 或 '*'。

### 场景 3：程序前缀为 "1 +"

继续计算 D("1 +", L)，移除前缀 "1 +" 后，得到：

```  
D("1 +", L) =  ( '+'  | '*'  )*   
```

此时，合法的后续符号只能是 ``，因为运算符不能连续出现。

### 场景 4：程序前缀为 "1 + 2"

计算 D("1 + 2", L)，得到：

```  
D("1 + 2", L) = ( '+'  | '*'  )*   
```

此时，合法的后续符号可以是 '+' 或 '*'。

### 场景 5：程序前缀为 "1 + 2 *"

计算 D("1 + 2 *", L)，得到：

```  
D("1 + 2 *", L) =  ( '+'  | '*'  )*   
```

此时，合法的后续符号只能是 ``。

## 总结

通过 Brzozowski 导数，CSD 可以在每一步生成代码时，根据当前前缀和预设的语法/语义约束，动态地确定合法的后续符号集，从而保证最终生成的代码始终满足规范。

  
## 注意

上述例子为了方便理解，对 Brzozowski 导数的应用做了简化。实际应用中，CSD 会使用更复杂的状态机或语法分析树来表示程序语言的语法和语义规则，并在此基础上计算 Brzozowski 导数。

      - 约束举例: 

```
提供的文本讨论了约束语义解码 (CSD) ，它是一种提高大型语言模型生成代码可靠性的方法。 虽然它没有直接列出具体的代码约束，但它在三种编程语言（SQL、Vega-Lite 和 SMCalFlow）的上下文中提供了一些示例。

以下是为每种语言提供的约束示例：

**SQL：**

* **语法有效性：**确保生成的代码符合 SQL 语法规则。 例如，在 “AS” 关键字之后，代码必须跟随着一个有效的标识符。
* **模式遵守：**保证对表和列的引用存在于被查询的数据库模式中。 例如，在 “Flights.AirportCode = Airports.SourceAirport” 中，CSD 将验证 “AirportCode” 是 “Flights” 表中的有效列，而 “SourceAirport” 是 “Airports” 表中的有效列。
* **别名跟踪：**跟踪表别名以防止错误引用。 在示例中，CSD 确保 “T1” 被正确识别为 “Flights” 表的别名。

**Vega-Lite：**

* **数据字段验证：**确保可视化中使用的数据字段是数据集模式的一部分。
* **类型兼容性：**验证数据字段是否与与其数据类型兼容的可视化元素一起使用。 例如，将 “temporal” 字段与条形图（专为名义或定量数据设计）一起使用将被标记。
* **渲染约束：**实现约束以防止出现无意义或难以渲染的可视化。 例如，可以防止在具有太多不同值的字段上进行分面。

**SMCalFlow：**

* **API 参数类型检查：**确保传递给 API 函数的参数具有正确的类型。 例如，将字符串传递给需要数字温度的函数会被捕获。
* **变量声明和范围：**跟踪声明的变量及其类型，确保在变量的范围内使用变量，并使用与其类型兼容的操作。

**关键要点：**使用 CSD 实现的特定约束高度依赖于目标编程语言和所需的行为。 该论文主张使用完成引擎 (CE) 作为在代码生成过程中定义和实施这些约束的灵活机制。
```

      - 看上去, "Brzozowski 导数"只是一种形式化概念, 对于约束的使用没有太大帮助, 通过其他手段也可以向解码器来声明约束
