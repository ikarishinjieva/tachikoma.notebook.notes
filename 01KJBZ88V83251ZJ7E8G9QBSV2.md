# RAG的分类

```
在这篇论文中，Retrieval-Augmented Generation (RAG) 被细致地分类为四种基础形式，每一种都描述了检索者如何增强生成器的不同方式。下面是这四种RAG基础分类的详细描述：

### 1. 基于查询的 RAG (Query-based RAG)
基于查询的 RAG 从提示增强的概念出发，将用户的查询与检索到的信息无缝集成，直接输入到语言模型的初始阶段。在检索之后，获取的内容与用户原始查询合并，形成一个复合输入序列，然后由生成器处理以创建响应。这种方法在RAG应用中非常普遍。例如，可以将检索到的文档或数据段与查询一起输入到生成模型中，以增强生成的内容的相关性和准确性。

### 2. 基于表征的 RAG (Latent Representation-based RAG)
在表征基础的 RAG 框架中，检索到的对象以潜在表征的形式整合到生成模型中。这样做可以增强模型的理解能力，从而提高生成内容的质量。例如，FiD (Fusion-in-Decoder) 是一种典型的实施方式，其中检索到的每个段落通过独立的编码器处理，然后将结果融合在一起，由单一解码器解码以产生最终输出。

### 3. 基于逻辑的 RAG (Logit-based RAG)
在逻辑基础的 RAG 中，生成模型在解码过程中通过逻辑集成检索信息。通常，逻辑通过简单的求和或模型计算来结合，以计算逐步生成的概率。例如，kNN-LM (k-Nearest Neighbors Language Model) 在每个解码步骤结合语言模型生成的概率和从类似前缀的检索距离得出的概率，从而增强生成的文本的相关性和准确性。

### 4. 推测性 RAG (Speculative RAG)
推测性 RAG 寻求在可能的情况下使用检索代替纯生成，目的是节省资源并加快响应速度。例如，REST 使用推测解码替换小模型中的生成步骤，通过检索生成草稿。另一个例子是 GPTCache，它为了解决使用LLM API时的高延迟问题，构建了一个语义缓存来存储LLM的响应。

这四种RAG基础分类不仅说明了检索与生成过程的不同集成方式，而且展示了RAG技术在多样化应用场景中的灵活性和潜力。每种分类的方法都针对特定的需求和场景优化，为研究人员和开发者提供了广泛的选择来实现更高效和精确的生成任务。
``` 

        - 举例: 

```
假设查询是 "猫"，有两个文档：

文档 A: "猫是一种可爱的动物。" (reranking 得分高)
文档 B: "太阳是一颗恒星。" (reranking 得分低)
软标签会告诉 embedding 模型：

查询 "猫" 的向量表示应该与文档 A 的向量表示相似。
查询 "猫" 的向量表示应该与文档 B 的向量表示不相似。
通过这种方式，embedding 模型可以学习到一种更准确地反映查询和文档语义相似度的向量表示方法。

所以，软标签的作用是引导 embedding 模型学习，使其能够生成更有效的向量表示，从而提高检索的准确性。
``` 
        - 调整后的embedding模型, 不能取代reranking+embedding的组合: 
        - 理论上来说，如果 embedding 模型能够完全学习到 reranking 模型的知识，那么的确可以只使用 embedding 模型进行检索，而不再需要 reranking 阶段。

但是，在实际应用中，这样做可能会遇到一些挑战：  
信息损失: reranking 模型通常会考虑更复杂的因素，例如文档的长度、结构、权威性等等。这些信息可能无法完全体现在 embedding 模型的向量表示中。  
训练难度: 要让 embedding 模型完全学习到 reranking 模型的知识，需要大量的训练数据和精细的调参。  
效率: reranking 模型通常只对少数候选文档进行排序，而 embedding 模型需要对所有文档进行排序，效率可能较低。

      - 主要思路: 解决问题 (比如一个年份问题) 时, 先推导一个更抽象的层次 (先得到历史时间线), 根据抽象层次的答案, 再进行思考. (防止大模型陷入一个细节)
    - Active prompting: [20240610 - 阅读论文: Active Prompt]
      - 主要思路: 在CoT中, 选择 不确定性最高的 问题 (形成问答) 作为few-shot的样例
    - CoT: [20240210 - 阅读论文: Chain-of-Thought Prompting Elicits Reasoning in Large Language Models]
      - 主要思路: 主要通过样例, 教会LLM (few-shot) 给出逻辑的推断过程.
    - LLMLingua: (不详细分析)  

      - 主要思路: 通过微调一个小模型, 对提示词进行压缩. 压缩过程中, 需要解决两个问题: 
        - 压缩用的小模型, 和推断用的大模型 的能力之间是有差异的, 需要让小模型的预测结果尽量靠近大模型
        - 压缩过程中, 不要独立地估算单个token的概率, 而是需要保持压缩对象之间的关系 (比如转折词, "但是""仍然"..., 都包括了token之间的关系)
    - Lost in the Middle: [20240610 - 阅读论文: Lost in the Middle: How Language Models Use Long Contexts]
      - 主要思路: 通过评估, 得到结论: 对于LLM上下文的 中间部分 的信息, 模型性能会下降
    - ReMoDiffuse: [20240610 - 阅读论文: ReMoDiffuse: Retrieval-Augmented Motion Diffusion Model]
      - 主要思路: 
        - ReMoDiffuse 将 语义相似性和运动学相似性两个方向结合起来 检索 (单一方向的结果不好)
        - ReMoDiffuse 用一个微调的Transformer, 对 召回的信息 进行调整(选择有用部分 ,...), 使得其信息与问题一致
        - ReMoDiffuse 用 不同的权重参数, 在 结果太像样例 和 结果太不像样例 中取得平衡
      - (与综述论文中对ReMoDiffused的描述不符)
    - ASAP: (不详细分析)
      - 主要思路: 在代码摘要任务中, 将代码分析工具的结果, 连同代码一起给到LLM, 会增强LLM对代码的理解. 
    - CEDAR: (不详细分析)
      - 主要思路: 在代码生成的任务中, 用RAG技术召回相似代码片段, 放在提示词中进行推理, 会大幅提高正确率
    - XRICL: [20240520 - 阅读论文: XRICL: Cross-lingual Retrieval-Augmented In-Context Learning for Cross-lingual Text-to-SQL Semantic Parsing]
      - 主要思路: 将 翻译任务和生成任务 放在同一个提示词中, 模仿CoT思维链, 增强推理任务的效果
    - ACTIVERAG: [20240611 - 阅读论文: ACTIVERAG: Revealing the Treasures of Knowledge via Active Learning]
      - 主要思路: 使用四个思考代理, 从四个角度来梳理文档: 
        - 语义关联代理: (抽取模型中已有的知识) 将新信息与 LLM 已有的知识网络建立联系
        - 认知锚定代理: (抽取模型中未知的知识) 帮助 LLM 理解和学习未知的概念和信息
        - 逻辑推理代理: (抽取文档中的强逻辑) 利用结构化信息进行逻辑推理
        - 认知校准代理: (事实性检查) 识别并解决新信息与 LLM 现有知识之间可能存在的矛盾，提高答案的准确性
    - Make-An-Audio: 不详细分析, 涉及多模态输入  

  - 解码器优化
    - InferFix:  

      - 主要思路: 用RAG召回代码片段进行代码缺陷修复, 比较标准的流程

# RAG流程增强

召回不总是有助于推理的, 需要决定召回的必要性

  - 基于规则判断必要性
    - FLAR: [20240613 - 阅读论文: FLAR: Active Retrieval Augmented Generation]  

      - 主要思路: 
        - 思路1: 让大模型生成搜索指令
        - 思路2: 对于LLM生成的低置信概率的token所在的句子, 进行重新检索和生成
    - [20240613 - 阅读论文: Efficient Nearest Neighbor Language Models]
      - 与本节相关的内容: 在提高kNNLM的效率方法中, 可以通过自适应检索 (由一个小模型决定什么时候进行主动检索), 来提高效率
    - [20240613 - 阅读论文: When Not to Trust Language Models: Investigating Effectiveness of Parametric and Non-Parametric Memories]
      - 主要思路: LLM的记忆能力与话题流行度成正比, 论文使用话题流行度来判断 LLM对该话题的记忆 是否需要进行额外的补全 (通过召回的文档进行补全)
    - [20240613 - 阅读论文: How can we know When language models know? on the calibration of language models for question answering]
      - 与本节相关的内容: 使用 特征决策树 (抽取一些答案的特征 训练一棵决策树), 来判断LLM的答案是否可靠
    - Large Language Models Struggle to Learn Long-Tail Knowledge: (不详细分析)

      - 主要结论: LLM的 长尾知识 (训练数据中出现次数较少的知识) 的准确性, 与训练数据中相关文档的数量 成正比

  - 基于模型判断必要性
    - SELF-RAG: [20240205 - 阅读论文: SELF-RAG: LEARNING TO RETRIEVE, GENERATE, AND CRITIQUE THROUGH SELF-REFLECTION]
      - 与本节相关的内容: SELF-RAG使用一个单独的模型, 评估问答中是否需要更多的证据
    - [20240613 - 阅读论文: Investigating the Factual Knowledge Boundary of Large Language Models with Retrieval Augmentation]  

      - 与本节相关的内容: 文中用了先验指标和后验指标 (两组指标都是通过LLM提示词获得) 来评估大模型的认知边界. 综述论文认为这两组提示词可以用于评估 LLM 是否应当召回外部知识文档
    - SKR: [20240614 - 阅读论文: Self-Knowledge Guided Retrieval Augmentation for Large Language Models]
      - 主要思路: 
        - 将训练问题 在有/无RAG的情况下, 进行答案效果对比, 确定哪些问题是LLM的自有知识
        - 将知识边界形成可使用的模型
        - 使用知识边界模型, 判断新的问题是否用LLM的自有知识能解答, 从而判断是否需要召回文档
    - Rowen: [20240614 - 阅读论文*: Retrieve only when it needs: Adaptive retrieval augmentation for hallucination mitigation in large language models]
      - 主要思路: 
        - 通过 相同的问题 在不同语言间结果的不一致, 来诊断LLM产生了幻觉
    - Adaptive-RAG: [20240614 - 阅读论文*: Adaptive-RAG: Learning to Adapt Retrieval-Augmented Large Language Models through Question Complexity]
      - 主要思路: 
        - 通过对问题复杂度的分类器, 使用不同的策略来回答问题 (直接回答不检索/单步检索/多步检索)

迭代式RAG:

  - RepoCoder: [20240614 - 阅读论文: RepoCoder: Repository-Level Code Completion Through Iterative Retrieval and Generation]
    - 主要思路: RepoCoder没有太大的创新, 只是进行代码切片并迭代处理, 下一迭代可以用上一迭代的结果作为已知片段, 进行继续的优化
  - ITER-RETGEN: [20240209 - 阅读论文: ITER-RETGEN: Enhancing retrieval-augmented large language models with iterative retrieval-generation synergy]
    - 主要思路: 将上一迭代的RAG回答的结果 作为 下一迭代RAG过程的输入, 不断进行迭代
  - SelfMemory: [20240204 - 阅读论文: Self-mem: Retrieval-augmented text generation with self memory.]
    - 主要思路: 每轮迭代 进行 记忆增值 + 记忆筛选 的流程, 不断迭代直到收敛
  - RAT: [20240614 - 阅读论文*: RAT: Retrieval Augmented Thoughts Elicit Context-Aware Reasoning in Long-Horizon Generation]
    - 主要思路: 对CoT进行多轮迭代, 每轮迭代修订 T1...Ti 步骤
