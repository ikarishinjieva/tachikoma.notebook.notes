# 目标

整理与我相关的三个过程域: MPM；PCM；PAD

所属团队是EPG: (Engineering Process Group. EPG的主要角色是提供组织内部的过程定义、维护和改进)

MPM - 度量

PCM - 过程和产品质量保证

PAD - 过程资产开发

# MPM - 度量

  - 度量哪些数据: 
    - 维度1: 研发管理和质量管理
    - 维度2: 进度, 偏差, 进度分布, 偏差分布
  - 度量的意义: 进度分析和偏差分析
  - 度量的来源: 
    - 商业目标拆解 (顶向下) + 调研 (底向上)
    - 与商业目标的对齐: <度量指示器>有拆解
  - 定义文件: <度量指示器>, 包括: 定义, 采集方法, 存储方法, 分析方法, 汇报线
    - 级别: 
      - 项目级别
      - 公司级别: 各项目的情况汇总
    - 存储: 
      - 项目相关的数据: <项目度量表>
      - 公司相关的数据: <公司度量库>
    - 汇报线: 公司领导和项目相关人员
  - 具体的度量指标 - 工作量:
    - 采集方法: 填写统计表
    - 分析方法: 进行偏差分析, 以及偏差分布
  - 具体的度量指标 - 测试缺陷密度:
    - 采集方法: jira跟踪
    - 分析方法: 缺陷在各维度的分布 (严重程度, 模块, 原因, 阶段等)
  - 具体的度量指标 - 规模:
    - 采集方法: 代码行数统计
    - 分析方法: 偏差分析 (与预估行数的偏差)
  - 度量的使用: 
    - 数据分析: 
      - 数据是否准确: 人员审核 + 工具采集
      - 公司级别数据: 在公司经营会议上分析
    - 偏差处理: 
      - 项目级别的数据: 减少需求, 申请资源, 加长工作时间, 优化效率
      - 公司级别的数据: 根因分析 + 管理优化
        - 分析结果通报: 高层 + 全员

# PCM - 过程和产品质量保证

EPG的情况: 

  - 1人专职 + 各业务领域的负责人
  - 七八年前成立
  - 职责: 推动研发流程改进, 降本增效  
  

过程改进:

  - 需求来源: 
    - 顶向下: 公司目标的拆解
    - 底向上: 日常工作中的发现和反馈
    - 外生: 客户的要求
    - 内生: 复盘发现的问题
  - 对需求的记录: 记录在《过程改进建议清单》
  - 对需求的排期/优先级: 投入产出比 (对公司目标的贡献度, 紧急程度和影响, 资源要求, 时间要求)
  - 计划: 
    - 年度: 《年度过程改进计划书》
      - 其中需要将 改进 和 目标 进行关联, 确保目标一致
      - 包括: 商业目标、改进目标、重点改进过程、改进策略、改进资源、预算、各改进点负责人、培训计划、工具计划、效果验证方法
    - 拆解: 《过程改进日程表》
  - 实施:
    - 与日常工作的资源冲突: 由EPG专员进行沟通, 找平衡
    - 任务分配: 按业务领域进行拆解
    - 跟踪进度: 由EPG专员进行监控, 会议, 汇报
    - 召集会议: 评审, 月度例会, 重大事项讨论, 年度会议
    - 工具: 
      - 是什么: excel
      - 如何选择工具: 按功能打分
      - 试点: 选择项目, 在开始时实施, 结束时观察效果
  - 流程体系发布:
    - 定期发布/紧急发布: 发布申请 -> 公司审批 -> 全员邮件
    - 发布后的保障: 
      - QA负责监控
      - 每年会进行年审
      - 年审时会对效果进行分析
  - 报告: 
    - 月度改进报告
    - 年度总结报告

# PAD - 过程资产开发

  - 流程体系 (产品开发流程)
    - 计划 -> 需求 -> 概要设计 -> 详细设计 -> 编码 -> 单元测试 -> 集成测试 -> 系统测试
    - 角色: 项目经理, 需求分析人员, 设计人员(架构师), 开发人员, 测试人员, QA, EPG
    - 主要流程: 过程改进流程、项目管理流程、需求开发和管理流程、设计和开发流程、测试流程、评审流程、配置管理流程、质量保证流程  

      - 文件类型: 过程文档, 指导指南, 模板表单, 检查单
    - 过程裁剪: <裁剪指南> \+ <裁剪表> \+ EPG审批
    - 更新: 评审 -> <发布申请> -> 审批 -> 全员邮件
    - 偏差修正: 
      - 要求项目组整改
      - 修正流程规范
      - 允许特例存在
  - 过程资产
    - 包括: 
      - 过程标准库: 全套流程体系
      - 公共模块: 
        - 包括: 代码, 设计文件, 说明文件等
        - 更新: 由专人进行开发和管理, 包括在总结时进行提取
      - 公共风险库?
      - 公共测试用例: 测试进行总结
      - 培训库: 存放在公共平台
      - 工具库: 存放在公共平台
      - 度量库: 项目经理填写《项目度量表》
      - 样例库
    - 主要存放在SVN
    - 分析: 偏差分析
    - 宣讲: 邮件通知 + 培训
    - 监控: QA负责监控
    - 工作环境的统一: 《公司环境规范》
