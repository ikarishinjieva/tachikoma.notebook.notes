# 主要思路

  1. 观察: 模型对于一个问题的多个变种的答案中, 一致性越高的部分, 其正确率越高
  2. 论文基于以上观察, 让模型多次回答相似的问题, 获得一致性高的答案和一致性低的答案, 形成DPO用的训练对
  3. 其中有用的知识: 在DPO训练过程中, 损失函数综合了DPO和NLL（Negative Log-Likelihood，负对数似然）, 其作用在单独章节描述

# 使用NLL的原因

NLL可以用于训练过程中保持 "自然语言"的流畅性. 其机制是 计算 答案中词序 在 LLM内的语言概率表的大小, 来判断 词序是否符合语言规律

好的，我用一个例子来说明 NLL (Negative Log-Likelihood) 在结合 DPO 训练中的作用。

假设我们正在训练一个模型来总结文章。我们有一些偏好数据，其中包含对不同总结的偏好比较，但没有标注哪个总结是“正确”的。

**场景 1：只使用 DPO 训练**

模型的目标是学习偏好顺序。它可能会过度拟合偏好数据，学习到一些奇怪的模式。例如，如果偏好数据中较短的总结总是被认为更好，那么模型可能会倾向于生成非常短的、信息量不足的总结，即使这些总结并不能很好地概括文章内容。 它只关注 *哪个* 总结更好，而不关注总结本身 *好不好* 。

**场景 2：结合 DPO 和 NLL 训练**

加入 NLL 后，情况就不同了。NLL 会鼓励模型生成更符合自然语言规律的总结。 它会惩罚那些语法错误、语义不通顺的总结，即使这些总结在偏好数据中排名较高。

**具体例子：**

假设有两篇待总结的文章，以及模型生成的几个总结：

**文章 1:** "小猫在阳光下玩耍，它追逐着蝴蝶，玩得很开心。"

  - 总结 A (短): "猫玩。"
  - 总结 B (完整): "小猫在阳光下追逐蝴蝶，玩得很开心。"

**文章 2:** "科学家们发现了一种新的治疗癌症的方法，这项研究发表在顶级期刊上。"

  - 总结 C (短): "癌症治。"
  - 总结 D (完整): "科学家们发现了一种新的癌症治疗方法，该研究已发表。"

假设偏好数据显示：总结 A > 总结 B，总结 C > 总结 D (即较短的总结更好)。

如果只用 DPO 训练，模型可能会学习到“总结越短越好”的规律，并倾向于生成类似“猫玩”和“癌症治”这样信息量不足的总结。

但是，如果加入 NLL，模型会发现“猫玩”和“癌症治”这样的总结在语法和语义上都不完整，因此会受到 NLL 的惩罚。 NLL 会鼓励模型生成更完整的总结，例如“小猫在阳光下追逐蝴蝶，玩得很开心”和“科学家们发现了一种新的癌症治疗方法，该研究已发表”。

最终，结合 DPO 和 NLL 训练的模型会尝试找到一个平衡点：既要满足偏好数据的排序 (A > B, C > D)，又要生成语法正确、语义通顺的总结。 这样，即使偏好数据存在一些偏差，NLL 也能起到一定的纠正作用，防止模型生成质量太差的文本。 它相当于告诉模型，除了满足偏好，总结本身也要说得通，说得完整。

NLL（Negative Log-Likelihood，负对数似然）通过以下方式鼓励模型生成更符合自然语言规律的答案：

  1. **概率建模:** 语言模型本质上是在学习一个概率分布，即给定上下文，预测下一个词出现的概率。NLL 损失函数的目标是最小化负对数似然，这等价于最大化训练数据中观测到的文本序列的概率。换句话说，模型会尝试学习如何生成更可能出现在真实世界中的文本序列。

  2. **训练数据的影响:** 训练数据通常包含大量的自然语言文本，这些文本隐含了丰富的语言规律，例如语法规则、词语搭配、语义关系等。通过最小化 NLL 损失，模型会学习到这些规律，并在生成文本时尽可能遵循这些规律。

  3. **具体计算过程:** NLL 的计算过程涉及到将文本序列分解成一系列条件概率的乘积，然后取对数并取负值。例如，对于一个句子 "The cat sat on the mat"，它的 NLL 可以表示为：
         
         -log P("The") - log P("cat"|"The") - log P("sat"|"The cat") - ... - log P("mat"|"The cat sat on the")
         

在这个公式中，每个条件概率都反映了给定上下文，下一个词出现的可能性。如果模型生成的文本序列不符合自然语言规律，例如语法错误或语义不通顺，那么对应的条件概率就会很低，导致 NLL 损失值很高。 反之，如果模型生成的文本序列符合自然语言规律，那么对应的条件概率就会较高，导致 NLL 损失值较低。

  4. **NLL 的作用类似于“语法纠错”和“语义通顺性检查”。** 它会惩罚那些不符合自然语言规律的文本序列，并鼓励模型生成更符合语法和语义的文本。

**举例说明：**

假设模型要生成一个接在 "The cat is" 后面的词。

  - 如果模型生成 "sleeping"，这个词符合语法和语义，在训练数据中也经常出现，那么 P("sleeping"|"The cat is") 的概率就会比较高，NLL 损失值就会比较低。
  - 如果模型生成 "eatting"，虽然语义上勉强说得通（猫会吃东西），但语法错误（应该是 eating），在训练数据中出现的频率也较低，那么 P("eatting"|"The cat is") 的概率就会比较低，NLL 损失值就会比较高。
  - 如果模型生成 "purple"，这个词在语法上可以接受（可以做形容词），但在语义上不太合理（猫不太可能是紫色的），在训练数据中这种搭配也比较少见，那么 P("purple"|"The cat is") 的概率就会很低，NLL 损失值就会很高。

通过最小化 NLL 损失，模型会学习到 "sleeping" 比 "eatting" 和 "purple" 更好，从而生成更符合自然语言规律的文本。 NLL 就像一个内置的“语言老师”，不断引导模型生成更合理的文本。
