# 代码

com.oceanbase.oms.connector.coordinator.BootstrapV2#bootStrap0

  - taskManagerAllocator, 负责构建 Task Manager
  - Task Manager 分类
    - SourceTaskManager: 为源端创建任务线程
    - BridgeTaskManager: 为处理链创建任务线程
    - SinkTaskManager: 为目标端创建任务线程

## SourceTaskManager

  - 初始化: 通过 createConnectorTask 创建 Source源端连接 和 Connector工作线程
    - 源端连接的作用: 连接源数据库, 订阅变更数据
      - StoreSource, 对应的工作线程类型为FORWARD

        - 有单独的进程 store, 从MySQL订阅变更
        - StoreSource 是通过HTTP接口, 从store获取数据
        - 猜测: FORWARD的意思 是 从store直接获取数据, 而不再次加工
    - 工作线程的作用, 从 源端连接 获取数据, 发送给 下游
    - 工作线程分为: (com.oceanbase.oms.connector.coordinator.threadmanager.sourcetask.SourceTaskManager#createConnectorTask)
      - ClassicSourceConnectorTask (普通Connector) 
        - source.poll()
        - batchProcessor 整形 ?? (举例: ETLBatchProcessor)
        - 限流器整流 (FrameControl.inControl)
        - this.batchSink.offer
      - ConditionedSourceConnectorTask
        - 未看到有效的使用
      - 特殊机制: forwardSources (FORWARD)
        - 限流器整流 (FrameControl.inControl)
  - 下游
    - 初始化位置: com.oceanbase.oms.connector.coordinator.threadmanager.TaskManagerAllocator#buildTaskManagers
    - 下游为 taskManagerAllocator 传入的处理链的第一环
  - 问题

## BridgeTaskManager

  - 作用: 为 处理链 中的各环节创建 工作线程
  - 处理链 (在com.oceanbase.oms.connector.coordinator.BootstrapV2#bootStrap0进行初始化)
    - etlBatchProcessor: 
      - ddlTransformerChain
        - DDLTypeDbTableFilterTransformer
          - 根据黑白名单配置, 剔除不需要迁移的DDL
        - DDLTransformer
          - 将 源数据库 的DDL, 转换成 目标数据库的DDL (使用中间的DDL解析结构 进行 中介)
      - dmlTransformerChain
        - DMLTypeFilterTransformer
          - 根据黑白名单配置, 剔除不需要迁移的DML
        - RowFilterTransformer
          - 根据黑白名单配置, 剔除不需要迁移的 行
        - ColumnFilterTransformer
          - 根据黑白名单配置, 剔除不需要迁移的 列
    - transactionScheduler, 梳理事务顺序. (DefaultTransactionScheduler)有三个作用: 
      - 限流阀: 
        - 阀内能容下 maxRecordCapacity 个行, 超过则等待消费
      - (checkpointManager) 生成checkpoint: 
        - 当阀内的 事务 完全消费掉, 生成一个checkpoint
      - (conflictBroker) 依赖管理:
        - 根据DML事务的行的唯一键值, 计算事务的依赖图 (拓扑顺序)
    - 经过 com.oceanbase.oms.connector.coordinator.threadmanager.TaskManagerAllocator#tryRebuildTaskChain 整形
      - 在必要的环节, 增加Queue过渡 (BatchProcessor 在BridgeTaskManager 中处理, 需要Queue作为 BatchProcessor 的代位, 这是一段诡异的逻辑处理)
      - 不允许两个相邻的阶段 同时 run exclusive
      - 不允许两个相邻的阶段 同时为 BatchProcessor
  - 工作线程分为:   

    - 多线程模式 (ParallelProcessConnectorTask, 针对于BatchProcessor环节)
      - 该线程 输入为 BatchProcessor前一环节, 输出为 BatchProcessor后一环节
      - BatchProcessor只作用于处理过程中
    - 单线程模式 (SerialBridgeConnectorTask)
      - 该线程 输入为 当前环节的前一环节, 输出为 当前环节
      - source.poll()
      - 处理batchProcessor
      - sink.offer()

## SinkTaskManager

  - 初始化: 通过 createConnectorTask 创建工作线程
    - 工作线程的作用: 向目标端进行回放
    - 工作线程分为: (com.oceanbase.oms.connector.coordinator.threadmanager.sinktask.SinkTaskManager#createConnectorTask)
      - SyncSinkConnectorTask: 同步回放 (OBMySQLJDBCSink)
        - source.poll()
        - sink.offer()
        - 限流器整流 (FrameControl.outControl)
      - ASyncSinkConnectorTask: 异步回放, 带重试和错误处理机制  

        - 将同步回放的逻辑 包装 成任务, 多线程执行
  - 上游
    - 上游为 taskManagerAllocator 传入的处理链的最后一环
