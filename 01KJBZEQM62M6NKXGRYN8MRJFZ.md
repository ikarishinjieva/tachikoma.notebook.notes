本文提出了prompt tuning, 也就是将soft tuning应用于提示词的方法:

这篇论文中提到的方案是 **Prompt Tuning**，它是一种参数高效的预训练语言模型微调方法。其核心思想是：

**冻结预训练模型的所有参数，只训练一小段可学习的“Prompt”。**

具体来说，Prompt Tuning 的流程如下：

1\. **添加 Prompt**: 在输入文本的开头插入一段特殊的 token 序列，称为 Prompt。这段 Prompt 并不来自于词表，而是拥有独立的可学习嵌入向量。  
2\. **编码**: 将拼接后的文本（Prompt + 输入文本）输入预训练语言模型的编码器，得到每个 token 的向量表示。  
3\. **解码**: 将编码后的向量序列输入预训练语言模型的解码器，进行下游任务（例如文本分类、问答等）。  
4\. **训练**: 在训练过程中，只更新 Prompt 的嵌入向量，而保持预训练模型的所有参数不变。

**论文中提到的设计决策包括:**

* **Prompt 初始化**:   
* 随机初始化：最简单的方式，从均匀分布中随机采样初始化 Prompt 嵌入向量。  
* 词汇表采样初始化：从预训练模型的词表中随机选择一些 token 的嵌入向量来初始化 Prompt。  
* 类别标签初始化：对于分类任务，可以使用类别标签的嵌入向量来初始化 Prompt。  
* **Prompt 长度**: Prompt 的长度是一个超参数，需要根据具体任务和模型大小进行调整。  
* **预训练目标**:   
* 传统的语言模型预训练目标（例如自回归语言建模）更适合于 Prompt Tuning。  
* 基于掩码语言建模的预训练目标（例如 T5 使用的 Span Corruption）效果不佳，需要进行额外的语言模型适应训练。

**Prompt Tuning 的优势:**

* **参数效率**: 只需要训练一小部分参数，大大降低了存储和计算成本。  
* **可复用性**: 训练好的 Prompt 可以用于不同的下游任务，提高了模型的泛化能力。  
* **鲁棒性**: 冻结预训练模型的参数可以避免过拟合，提高模型在领域迁移任务上的性能。

总而言之，Prompt Tuning 是一种简单有效且参数高效的预训练语言模型微调方法，在大规模语言模型上展现出巨大的潜力。
