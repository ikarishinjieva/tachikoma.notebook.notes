# 视觉部分

### 臂上单目摄像头 (CNY 60 - 200), 识别AprilTag

  -     - 体积要小, 负载要轻, 否则限制机械臂的活动
      - ![image](https://static.elephantrobotics.com/wp-content/uploads/2022/10/myCobot-%E6%91%84%E5%83%8F%E6%A8%A1%E7%BB%84-1.png)
      - ![image2026-1-8 13:16:1.png](/assets/01KJC09P0B1PQVA3T49QEWQ6RF/image2026-1-8%2013%3A16%3A1.png)
    - 晚上需要补光
    - 焦距清晰范围短, 30-50cm
    - 对Apriltag识别的要求: 
      - Apriltag大小不能太小 (大小 20mm 以下, 距离 40cm以上 稳定性会变差)
      - Apriltag必须平整, 否则对于求倾斜位姿识别差很多
        - 用 双Apriltag 去修正倾斜位姿
      - 以上要求意味着: 在正常大小的移液器上 (100-1000ul), 在其最大侧面也贴不下平整的Apriltag, 只能弯曲失真
    - 臂上摄像头会比前部末端遮挡, 视野只有一部分, 且可能看不到 "未来"的轨迹上的点 (比如上图, 机械臂如果向夹爪上侧运动, 摄像头是无法感知)

    - 使用(大模型标记 + 深度), 进行目标的位姿计算:

      - 其他做法: 
        - 传统的位姿估计, 需要 3D纹理展开图
        - 模式匹配, 对噪音(背景纯净度)要求比较高
        - 轮廓估计, 在3D场景下 适用度低

      - 相比之下, (大模型标记 + 深度) 近乎零成本, 利用通识模型, 泛化性好, 不做额外训练
    - 发现问题: 
      - 用纯的可见光图像做深度估计, 光照影响大, 深度的稳定性很差
      - 对纯色部分, 深度估计会对中间部分失效
        - 通过增加斜向色块规避

### 经验

  - 固定摄像头, 对机械臂的微观操作不友好:
    - 容易被遮挡
    - 摄像头不能部署在 机械臂的臂展范围内, 否则影响动作
      - 水平界面需要离目标最大两个臂展, 其定位精度影响大
      - 垂直界面需要离目标一个臂展, 只能俯视
    - 需要反复进行手眼标定
  - 臂上摄像头, 只能看到局部, 没有全局观

### 经验 (演示视频)

  - 拇指内收, 才有可能在握持姿态, 按到移液器顶部
  - 与人手的不同, 造成了决策差异
    - 手部不能形成与大小刚好的空腔, 抓握后, 移液器的稳定是纯靠摩擦力
    - 机械手 比 一般人 手要大, 且关节位置比例不对 (蜘蛛样指)
      - ![image2026-1-8 16:37:20.png](/assets/01KJC09P0B1PQVA3T49QEWQ6RF/image2026-1-8%2016%3A37%3A20.png)![image2026-1-8 16:37:51.png](/assets/01KJC09P0B1PQVA3T49QEWQ6RF/image2026-1-8%2016%3A37%3A51.png)
      - 抓握姿势 和人不同
    - 拇指的平移轨迹固定, 且是人手达不到的 (7自由度能解决一部分)
    - 拇指的按压轨迹, 只有单自由度, 导致必须抓到指定位置, 才能按到按钮 (这一点, 对动作的预测性有很高要求)
      - 7自由度, 可以让拇指后撤一点, 但仍然是单一轨迹
      - 按压轨迹上, 双自由度, 能提供更大调整空间
    - 手的大小和按压轨迹限制, 导致只能以 后三指 握住移液枪, 且随着移液枪的大小不同, 使用的指数不同. 如果指数少了, 紧靠单个手指的摩擦力, 固定不住移液枪的轴向
    - 6轴机械臂, 末端是 Z轴旋转单元, 不像人是 X轴的折叠单元, 折叠的活动度小, 那么相同的折叠动作, 手部+臂部 扫过的面积 比人类更大 (对当前实验没影响, 做后面的避障动作序列的考虑)
    - 因为需要接线和散热, 手腕僵直的部分, 比人类大很多, 伸出机械臂的力矩要大很多, 导致机械臂稳定性下降
  - 对视频中的动作解释: 与手部相关的动作
    - 先移动到大概位置
    - 拇指侧移到低位, 与下三指形成一个宽松的环. 目的是 捏住时, 保持移液枪轴向不要偏离太大
      - 摄像头看不到环内的状态
    - 环上提, 让手指接触移液枪的手柄, 增加稳定性
    - 握紧拇指, 保证下一步下三指握紧时, 不会把移液枪挤出来
    - 握紧下三指, 拿起移液枪, 且能保持轴向
    - 复位拇指
    - 拇指按压
  - 目前整个动作是通过程序硬编码, 与人类的步骤不同. 
  - 对动作的偏差性分析 (现在成功率为 20%)
    - 视觉识别出的坐标轴向偏差
      - 视频中用了简化后 的蓝色区域识别, 其长边和短边决定接近轴向, 因为蓝色区域小 (距离60cm+), 所以轴向有偏差. 这一点是纯算法问题, 可以改进. 
        - 但这里是 固定视觉 的弊端, 最好是 臂上视觉, 能近距离观察
    - 臂部的精度有偏差, 官方偏差为 5mm
      - 其主要影响是 抓握后的 按压轨迹, 差5mm抓握距离, 那么按压轨迹侧向偏差 会 导致按不到按钮
    - 碰撞  

      - 轻微碰撞后的姿态随机: 即使放慢动作, 刚接触时的碰撞后的方向 微观上 不可控, mm级的偏差是存在的
      - 笛卡尔运动的运动过程中, 会改变末端姿态 (插值平滑), 其有可能产生碰撞
        - 在运动规划时 可以避免, 代价是对于 异型器皿, 其碰撞模型比实际模型更大
      - 近距离微移, 机械臂末端会抖动, 其有可能产生碰撞
        - 在运动规划时 不可避免
  - 为什么不用移液架
    - 没有好的算法, 完成在手上进行姿态调整的动作

# 机械臂

先后使用了两款6轴机械臂 (RMB 6000+), 主要区别: 

  - 末端负载: 完全伸直时的有效负载
  - 控制精度/重复定位精度: 我去哪
  - 反馈精度: 我在哪

小象机械臂 280: 

  - ![image2026-1-8 18:39:57.png](/assets/01KJC09P0B1PQVA3T49QEWQ6RF/image2026-1-8%2018%3A39%3A57.png)
  - 末端负载: 250g
  - 控制精度/重复定位精度:  ± 0.5mm
  - 反馈精度: 很高 (3mm)
  - 反馈延迟: 很高 (且接口不能并发)
  - 寿命时长: 500 h
  - 工作半径: 280mm

  

玄雅机械臂:

  - 末端负载: 1kg
  - 控制精度/重复定位精度: ± 2mm
  - 控制精度/位置精度: ±5~8mm
  - 反馈精度: 低 (手眼标定误差 50mm)
  - 反馈延迟: 低
  - 寿命时长: (保修6个月)
  - 工作半径: 650mm  
  

**UR3/UR3e协作机械臂：**

  - 末端负载：3kg
  - 重复定位精度：±0.1mm (UR3)，±0.03mm (UR3e)
  - 工作半径：500mm[[1]](<https://www.universal-robots.com/media/240787/ur3_us.pdf>)
  - 控制精度：具有6轴旋转关节，能实现360度旋转[[1]](<https://www.universal-robots.com/media/240787/ur3_us.pdf>)
  - 安全功能：17个可配置安全功能，符合EN ISO 13849-1和EN ISO 10218-1标准[[4]](<https://www.universal-robots.com/media/1807464/ur3e_e-series_datasheets_web.pdf>)
  - 价格：约25,000-30,000美元（约180,000-215,000人民币）

  

经验: 

  - 末端负载是一个很严重的指标, 灵巧手的重量 和 力矩长度, 会占用掉很大一部分负载, 导致成本上升
  - 桌面级的寿命都不长
  - 6轴的姿态规划要求更高: 
    - 7轴可以在保持末端位姿时, 调整姿态, (可能)解开奇异点限制: (17000 RMB)
      - ![image](https://static.elephantrobotics.com/wp-content/uploads/2023/08/8%E6%9C%883%E6%97%A5-1.gif)
      - [20251104 - 六轴机械臂和七轴机械臂的区别]
    - 路径规划需要考虑: 
      - 避开奇异点
      - 移动时, 器皿的姿态 (不能翻覆)
      - 器皿与臂上零件的碰撞 (根据器皿不同, 进行建模/划定不同的碰撞体, 碰撞体太大, 移动就很有限, 碰撞体太小, 建模要求太高)
      - 下一个动作 (倾倒), 当前动作应留有余地
      - 器皿的稳定性支点 (倾倒动作, 假设一定能夹紧, 还是假设需要支点?)
      - 更大的未来自由度
      - 稳定性
        - 对于轻微的移动, 动离末端比较近的关节, 会更稳定
  - 路径规划目前比较可行的方式 (人类实现抽象的阶段)  

    - 粗看场景, 大致接近一个好的观测位置
    - 进行观测, (带着范围限制) 决定下一步的接近位姿, 进行路径规划(避障)
      - 对未来动作进行验算, 决定最好的规划 (包括围观动作, 以及下一个整体动作)
    - 进行靠近后的微观 观察, 确定可以无障碍直线进行, 进行多次直线动作, 完成动作
      - 对观察能力的挑战: 焦距很短, 只能看到失真的局部
    - 路径规划需要对路径上的障碍物有实时的描述
  - 即使在最好的规划条件下, 有些动作仍然 有概率 不能一次完成 (拿起杯子, 倒水). 比如: 为了避开奇异点, 占用了下一动作的余地, 或者反之. (两个原则冲突)
    - 人类的做法: 需要让机械臂把杯子移动到另一个更宽敞的地方, 放下, 以新姿态重新拿起 (自由暂存区)
    - 双手协同? 
    - 对于多种路径规划原则, 人类程序的做法是: 在场景中以一种固定的模式协调, 不行就进行异常处理

# 对成本的过程回顾

  - 仿真环节: 
    - 环境搭建复杂度高 (整个行业对云服务设施不友好, 只针对 本地固定型号的设备集群 比较友好)
    - 对移动和场景的多样性很有帮助
    - Isaac sim在物理崩溃时给出的信息不足, 诊断需要很长时间
    - Isaac sim 的操作和能力 在不同版本间差异很大, 查找到的历史的经验文档不适用
  - 机械臂 + 夹爪: 
    - 机械臂的控制串口, 产生了并发延迟
    - 相机的 安装坐标系和视觉坐标系, 和仿真标准不同, 调试了很长时间
    - 碰撞体计算 (DAE的碰撞体体积过大, 换用STL, 但坐标系有差异)
    - 机械臂的奇异点, 在仿真中是不存在的, 而现实上过不去
  - 机械臂 + 灵巧手: 
    - 仿真方案, 尝试模仿灵巧手的复杂构型, 物理参数模拟, 过于复杂
    - 末端负载超过机械臂的有效范围, 导致出现运动偏差
    - 关节扭矩热失效
    - 灵巧手 马达连接性 会失效
  - 更换高负载机械臂 + 灵巧手
    - 灵巧手连接件的设计+3D打印
    - 高负载机械臂 的手眼标定精度 降低, 位置反馈单元不准
    - 双眼摄像头的深度计算 偏差 研究
  - 更换深度摄像头
    - 因为 USB线质量问题, 产生了 D2C 对齐偏差的问题, 研究花了几天
    - 安装坐标系和光学坐标系 再次变化, 重新适配
  - 高负载机械臂 J2 关节过热, 齿轮打碎, 诊断+返修

总结:

  - 仿真: 还没想出办法降低成本
  - 各硬件的"电气标准"只是看起来有标准, 实际精度和参数差很多. 
  - 各硬件都带有不同的驱动 (有的需要手工计算坐标系/标定, 有的自动发布坐标系/标定, 但方案都不规范)
  - 重点问题: 
    - "看 -> 动作" 这个需求, 中间的要求是: "看 -> 准确的看坐标 -> 准确的动作坐标 -> 动作策略"
      - 我们大部分的成本, 都花在如何在一个较远的位置 "看", 来预测几秒钟之后那个动作到"哪里". 这就要求"看"要准
      - 为了"准确的坐标", 就需要"绝对值准": 坐标系准, 深度准
      - 为了 "看"到"动作"准, 就要求 手眼标定准
    - 硬件的寿命成本, 之前是没考虑的

# 现在的 单眼+单臂+单手 的方案, 工程化落地的成本考量

不考虑VLA模型训练, 不考虑过大的泛化性, 只针对一个实验室的新的固定场景:

  - 对各个实验器皿进行建模, 目的是 较精确的位姿识别
    - 透明器皿需要贴辅助标
  - 眼睛必须放在手臂上 (或者 多眼协同)
    - 因为上述精度偏差, 必须在近姿态下进行微调
    - ( 局部视觉 -> 微调动作 ) 需要单独训练策略
  - 对实验器皿的动作进行定义 (抓取/倾倒/...), 对每一种动作, 写出抓取的动作序列和限制 (手臂+灵巧手, 复杂构型)
  - 更换机器人零件时, 需要再来一次(比如灵巧手大小不同, 自由度不同, 抓取的方式就会变)
  - 泛化性: 依靠仿真获取训练数据 + 人工做数据, 进行泛化

#   
新的想法: 人类示教+逻辑模仿+AI监督下的AI模仿和适配+AI自迭代优化

  - 已知: 
    - 对于复杂构型, 很难通过统一的策略来完成. 也不能完全模仿人 (因为机械和人的构型不能完全重合). 需要"仿人+定制"的策略
    - 传统的路径规划/避障等算法已经非常成熟
  - 思考:
    - 训练 "图像 -> 姿态/轨迹" 这个模型的成本太高, 需要将传统算法的成熟的逻辑一并拧进去
      - 训练 "图像 -> 策略 -> 调用传统算法" 这样的模型, 能降低成本
  - 已知:  

    - 不可避免的随机误差, 会导致预定姿态的变化. 
    - 臂上视觉 是 不能实时获取完整姿态 (手臂遮挡), 只能获取局部姿态.
  - 思考:
    - 由于随机误差和实时遮挡存在, 近距离下 进行 臂上视觉的观察和 近距离微调 是必然的步骤
  - 已知: 
    - 针对不同的器皿, 不同的尺寸, 不同的调度目的, 需要 不同的调度策略. 
      - 通过人类穷举策略, 或者 模式匹配+泛化推广, 需要大量精力来训练
  - 思考:
    - 让AI教师来训练AI学生
      - AI学生 参考策略的描述, 根据 臂上视觉 (局部视觉), 调用传统算法来进行动作
  - 已知: 
    - 现在的大模型已经可以很好的判断2D图片的状况
      - <https://poe.com/s/mSol7NdwRhuY1nWEc60L>
    - 仿真最大的好处是: 不断重置场景, 不断变化场景, 不断引入波动
  - 思考  

    - AI教师通过另一个可移动的视角进行观察 (模仿人眼)
    - AI教师可以阻止危险动作, 减少场景重置次数
    - AI教师可以重置场景 (难度高)

### 想象的方案 - (单臂): 

  -     - 由人类通过遥操:
      - 用一个机械臂+仿生手+臂上相机, 一个特定的器皿完成人类动作
      - 用另一个机械臂+臂上相机, 跟踪人的视觉位置, 拍摄人类视角上的过程图片
      - 记录 一些人类遥操时做决策的逻辑判断 ("用大拇指按住移液器, 以防握紧时出现偏移", 不求全, 只给AI做参考)
    - 将这些信息给大模型做参考信息
    - 现在交给AI调度, 要求目标动作是一样的, 目标器皿可以更换不同型号/尺寸
      - 要求 学生AI: 用一个机械臂+仿生手+臂上相机, 完成动作. 但动作要非常缓慢
      - 要求 监督AI: 用另一个机械臂+臂上相机, 不断变化位置, 从多角度观察学生AI的动作, 如果学生的非常缓慢的动作即将触发危险, 那么给学生AI停止信号, 要求其重新思考
        - 减少场景重置的次数
      - 学生AI要学习之前整理的参考(逻辑等)信息
        - 用传统调度算法写代码, 要求只能使用它本体的信息, 来进行调度
      - 监督AI会多角度 观察 当前时间下的 危险信号, 给学生AI以反馈, 供学生AI来调整策略代码
    - 如何达成泛化
      - 对策略代码的生成 AI 提出要求, 比如 "距离值和角度值都需要通过视觉信号计算得来"
      - 更换起始姿态
      - 增加禁区
      - 更换随机场景 (期望是更换次数很少)
    - 如何进行优化
      - 逐步加快调度速度, 简化调度步骤 (不确定是否能稳定判断 安全动作, 可能增加场景重置的次数)

核心思路: 

  - 在人的示教过程中, 教师AI站在人的位置, 理解人的决策逻辑
  - 在人的示教过程中, 学生AI站在臂上
  - 训练过程中, 教师辅助学生, 学习: 如何能独立(使用自己的视角)完成的事项
  - 用"逻辑"作为调度的单位, 来增加泛化性

# 目标demo场景

  - 一个机械车有两个机械臂, 机械臂都是 "机械臂+相机+灵巧手"结构, 构成一个"机器人", 站在化学实验室门口
  - "机器人"进入化学实验室, 靠近试验台, 对化学试验台上的器皿, 用右手进行掂量和抓取尝试, 与此同时, 左手通过不同角度进行观察. 如果右手动作有风险, 那么及时要求调整策略. 这是机器人的自主学习阶段
  - 经过一段时间后, "机器人"已经掌握了试验台上各器皿的抓取方法和重量, 可以自如操作这些器皿, 而不用经过学习阶段, 完成基本实验
