最终结果, 见提示词: prompts/v5/pipeline

# 迭代过程

1. 跟AI讨论大的阶段, 每个阶段的目标是什么, 分成了4个Milestone: 
   1. KPI理解

      ```markdown
      > 10个问题，覆盖KPI的物化机制、阈值依据、测试方法、耦合关系、预测方法、感知度等维度。
      
      Q5(隐性KPI) ──→ Q1(机制) ──→ Q4(耦合) ──→ Q6(影响与应对) ──→ Q8(感知度)
           │              │              │              │
           │              ├──→ Q2(阈值)  ├──→ Q9(解耦)  └──→ (收尾)
           │              │              │
           │              └──→ Q3(测试)  └──→ Q7(预测)
           │                     │
           └─────────────────→ Q10(需求审视，依赖Q2+Q3的上下文)
      ```
   2. 行业方案搜索

      ```markdown
      Q1(直接竞品) ──→ Q2(技术路线全景) ──→ Q3(专利七维穷举) ──→ Q5(范式归纳)
                         │                                          ↑
                         └──→ Q6(竞品深度) ──→ Q7(专利全文) ────────┘
                                                ↑
                                Q4(相邻行业) ───┘
      ```
   3. 候选物探索

      ```markdown
      Q1（组分角色图谱）
        └── Q2（架构推导：通用→具体应用）
              └── Q3（维度穷举：候选物搜索）
                    └── Q4（KPI冲突解析：物化查缺补漏）
      ```
   4. 配方组装

      ```markdown
      Phase A（修补循环）→ Phase B（增强插件）→ Phase C（最终评估）
      
      ## 核心方法论
      
      - **基底穷举（产品级）**：N个具体基底产品，每个独立建树
      - **问题驱动修补**：诊断KPI缺口 → 找关联角色 → 展开角色内候选物 → 修补
      - **修补也穷举**：每个缺口的所有候选修补方案都试，形成分支
      - **Gemini REST API + 检索工具**：AI通过 `research_tool.py` 实时查物化参数，不靠记忆猜
      - **只标记不淘汰**：法规/成本在最终评估中标记，不在树中剪枝
      ```

重点: 

- 迭代中, 与run12进行信息对比, 迭代中缺少哪些信息, 由人工模仿化学专家, 从本质上进行提问, 诱导模型查找到缺失的信息, 从而来构建提问集合
- 需要穷举时, 可以通过代码形式来完成 完全穷举
- 用cc去调用gemini cli, 其工作区为单独的目录, 防止信息污染
