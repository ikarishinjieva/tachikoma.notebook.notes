# 主要思路

论文首先分析了LLM对于知识的记忆能力: 

论文中分析 LLM 的记忆能力，得到了以下四个主要结论，现对其进行详细解释：

**1. 模型的整体性能:**

* **结论**: 即使没有使用上下文示例，较大的 LLM 也表现出合理的性能，这表明它们在一定程度上记住了事实性知识。  
* **详细解释**: 实验中，研究人员测试了不同规模的 LLM 在零样本或少样本设置下的问答能力。 结果显示，即使没有提供任何相关上下文信息，大型 LLM (例如 GPT-3) 也能正确回答一部分问题。 这表明，LLM 在预训练过程中，已经将大量的知识编码进了自身的参数中，形成了“参数化记忆”，能够在一定程度上直接回答关于这些知识的问题。

**2. 主题实体的流行度可以预测记忆:**

* **结论**: 主题实体的流行度与其在模型中的准确率之间存在正相关关系，这支持了论文的假设，即主题实体的流行度可以作为 LLM 事实性知识记忆的可靠指标。  
* **详细解释**: 实验结果表明，对于大多数关系类型，LLM 在回答关于高流行度主题实体的问题时，准确率更高。 这是因为高流行度的实体在训练数据中出现的频率更高，LLM 更有可能学习到关于它们的知识。 因此，主题实体的流行度可以作为 LLM 是否记住了相关知识的一个有效指标。

**3. 关系类型影响记忆:**

* **结论**: 模型对某些关系类型的平均性能高于其他关系类型，这表明某些关系类型的事实性知识比其他关系类型更容易被记住。  
* **详细解释**: 实验发现，即使对于相同流行度的主题实体，LLM 在回答涉及不同关系类型的问题时，准确率也可能存在差异。 例如，对于“出生日期”这类常见的关系类型，LLM 的回答准确率通常较高；而对于一些较为冷门的关系类型，LLM 则更容易出错。 这可能是因为不同关系类型在训练数据中的出现频率不同，导致 LLM 对它们的学习程度存在差异。

**4. 扩大规模可能无助于尾部知识:**

* **结论**: 虽然扩大模型规模通常可以提高性能，但在处理低流行度问题时，这种改进并不明显。  
* **详细解释**: 通常情况下，更大的 LLM 意味着更强的性能。 然而，实验结果表明，仅仅依靠扩大模型规模，并不能有效解决 LLM 在处理低流行度问题时的困境。 这是因为低流行度实体在训练数据中出现的频率很低，即使模型规模再大，也很难学习到关于它们的足够信息。 因此，对于这类问题，需要探索更有效的解决方案，例如结合外部知识库等方法。

  
总的来说，这些结论表明 LLM 的记忆能力与其预训练数据密切相关，并且表现出明显的“长尾现象”，即对于高流行度的知识，LLM 能够较好地记住；而对于低流行度的知识，LLM 的记忆能力则十分有限。

在以上结论的基础上, 使用召回的文档对LLM的记忆进行补全就是必要的.

论文中提出了一种方法, 来判断LLM的记忆什么时候需要额外补全: 通过对流行度的判断, 当流行度低于某个阈值, 则认为LLM的记忆可能是有问题的:

## 自适应检索的步骤

自适应检索的核心思想是：**根据主题实体的流行度，自适应地决定是否使用检索系统来增强 LLM 的输入。**

具体步骤如下：

1\. **确定阈值**: 首先，需要确定一个主题实体流行度的阈值。 该阈值用于区分 LLM 能够记住的“高流行度”实体和需要检索系统辅助的“低流行度”实体。 论文中使用了一种启发式的方法来确定阈值，但更理想的方法是根据具体的 LLM 和数据集进行调整。

2\. **判断实体流行度**: 对于一个给定的问题，首先识别出问题中的主题实体。 然后，根据预先收集的实体流行度数据（例如，维基百科页面浏览量），判断该实体是属于“高流行度”还是“低流行度”。

3\. **选择性检索**:   
* 如果主题实体的流行度高于阈值（“高流行度”），则认为 LLM 能够记住相关知识，因此直接将问题输入 LLM，不使用检索系统。  
* 如果主题实体的流行度低于阈值（“低流行度”），则认为 LLM 可能无法仅凭参数化记忆回答问题，因此需要使用检索系统从外部知识库中检索相关信息，并将检索到的信息作为附加上下文与问题一起输入 LLM。

4\. **LLM 生成答案**: 最后，LLM 基于输入的问题和（可选的）检索到的上下文信息生成最终答案。

**简而言之，自适应检索方法根据主题实体的流行度，在使用 LLM 的参数化记忆和外部检索系统之间进行切换，从而在保证答案准确性的同时，尽可能地减少检索系统的使用，提高效率并降低成本。**
