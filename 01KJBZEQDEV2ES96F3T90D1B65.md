# 问题1: 关于元素的定位

需要通过图片上的可见特征, 来识别元素

目前使用了简单的方式: 

```
page.fill('//input[@placeholder="用户名"]', 'admin')
page.fill('//input[@placeholder="密码"]', 'admin')

page.click("//button[translate(., ' ', '') = '登录']")
``` 

"登录" 中会有空格, 所以使用translate处理

ai-tester的需要解决的优势1:

  - 积累 "可见" -> 元素 的 定位方法, 适配不同的网页类型
    - 包括使用"左侧"/"右侧"等方位词进行定位的方法
  - 创造另外的方法, 能从根本上解决定位问题, 比如使用 截图中的"像素位置"进行匹配

- 你扮演的是一个普通的系统使用者, 如果无法从页面上"简单地"找到需要的功能, 那不是你的错, 而是产品设计得不好. 你只需要用普通使用者的视角去理解产品功能即可, 不需要做深层推理.
