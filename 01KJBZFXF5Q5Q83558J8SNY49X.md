# 论文的目标

CoT的推理过程, 结论和推理过程可能有冲突. 比如: 

```
问题： 所有鸟都会飞。麻雀是鸟。那么麻雀会飞吗？
CoT (不忠实的推理): 麻雀很小，翅膀力量不足，需要经常休息，所以麻雀不能长时间飞行。因此，麻雀不会飞。
``` 

为了解决这个问题, 尝试用逻辑的形式化来解决这种不忠实: 

  - 用LLM将文本转化为逻辑表达式
  - 对逻辑表达式进行演算
  - 用演算结果, 指导LLM进行逻辑推理

但又会产生"隐含事实丢失"的问题, 举例: 

```
论文中给出的具体例子如下：

* **上下文：**
    1. 书籍包含大量知识。
    2. 当一个人读一本书时，这个人会获得知识。
    3. 如果一个人获得知识，他们会变得更聪明。
    4. 哈利读了亨利·梭罗的《瓦尔登湖》。

* **推理目标：** 哈利比以前更聪明。
``` 

在提取逻辑表达式时, 会忽略“哈利是一个人” 和 “《瓦尔登湖》是一本书” 这两个隐含的事实, 导致逻辑推理有误. 

LoT就是为了解决这个问题提出的方法

# LoT方法步骤

  - 首先利用 LLM 从原文提取逻辑关系
  - 然后使用 Python 程序扩展这些逻辑关系
  - 扩展后的逻辑关系会被翻译回自然语言，作为补充信息添加到原始上下文中
  - 最后，LLM 基于包含隐含事实的原始上下文和这些补充的自然语言逻辑描述进行推理，得出最终结论。

核心思路: 对于能推导的逻辑进行推导, 将结果转成自然语言放回到LLM的问答中, 又LLM结合"隐含的知识" (也就是LLM内置的知识和上下文提供的无法整理成逻辑的知识) 进行最终推理
