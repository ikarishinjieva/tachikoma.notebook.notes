# 主要内容

针对的问题: 尽管LoRA很成功，但它与全量微调（Full Fine-tuning, FT）之间仍然存在着明显的性能差距。学术界普遍将此归因于LoRA可训练参数较少，但缺乏更深层次的根本原因分析。

论文使用的研究方法: 将一个权重矩阵 (Weight) 分解为两个独立的组成部分：

  - 大小 (Magnitude)：一个向量，决定了权重矩阵每一列的“长度”或“强度”。
  - 方向 (Direction)：一个单位矩阵，决定了每一列的“朝向”。

发现: 

  - 全量微调 (FT): 其“大小”和“方向”的更新呈现出一种复杂的、通常是负相关的模式。这意味着模型可以独立地、精细地调整权重的大小或方向（例如，大幅改变方向的同时，只微调大小）。

  - LoRA: 其“大小”和“方向”的更新表现出强烈的正相关。这意味着两者倾向于同增同减，缺乏FT那种解耦的、精细调整的能力。

Dora的创新: 

  - 在训练的反向传播时, 将大小和方向作为两个维度进行训练, 解开两者的相关关系  
  

### 1\. 回顾LoRA的反向传播（实际过程）

在LoRA中，原始的权重矩阵 **W₀** 是**被冻结的，不参与训练** 。  
训练时，模型学习的是两个小矩阵 **B** 和 **A** 。

  - **前向传播** : `W_final = W₀ (冻结) + B @ A (可训练)`
  - **反向传播** :
    1. 计算出损失函数对最终权重 `W_final` 的梯度 `dL/dW_final`。
    2. 因为 `W₀` 是一个常数（冻结），它的梯度为零。
    3. 所以，`dL/dW_final` 这个梯度会**原封不动地** 传递给 `B@A` 这一部分。
    4. 然后根据链式法则，梯度会继续传递，最终用来更新 **B** 和 **A** 。

这里的关键是：LoRA并没有改变 `W₀`，它只是在旁边加了一个“补丁”(`B@A`)。这个“补丁”自己承担了所有关于“大小”和“方向”的混合更新任务。

* * *

### 2\. DoRA的反向传播（精妙的分工）

现在来看DoRA。您的理解“使用两种算法分别更新两个矩阵”可以更精确地描述为：

**在反向传播时，将总梯度`dL/dW_final` 通过不同的计算路径，“智能地”分配给“大小组件”和“方向组件”去更新。**

它不是两种独立的算法，而是同一个反向传播过程中的**梯度分流** 。

我们来看看这个分流过程：

  - **前向传播** : `W_final = m (可训练) * normalize( V (冻结) + B @ A (可训练) )`

    - `m` 是大小向量。
    - `V` 是从 `W₀` 提取的初始方向，被冻结。
    - `B@A` 是用来微调方向的LoRA模块。
    - `normalize(...)` 是归一化操作，确保后面括号里的部分始终是单位向量（纯方向）。
  - **反向传播 (关键步骤)** :

    1. 同样，我们先得到总梯度 `dL/dW_final`。
    2. 现在，这个梯度来到了一个“十字路口”。它需要决定多少应该分给 `m`，多少应该分给 `B@A`。
    3. **流向“大小” (`m`) 的梯度**:
       - 这部分梯度是通过计算总梯度 `dL/dW_final` 在当前“方向” `normalize(V + B@A)` 上的**投影** 来得到的。
       - **直观解释** : 这相当于在问：“为了减小损失，我应该在**当前这个方向上** 增加还是减小权重的大小？” 计算结果会直接用来更新 `m`。
    4. **流向“方向” (`B@A`) 的梯度**:
       - 这部分梯度是总梯度 `dL/dW_final` 中**与当前方向垂直** 的分量。
       - **直观解释** : 这相当于在问：“为了减小损失，在**不改变当前权重整体大小** 的前提下，我应该把方向往哪个新方向偏转？”
       - 这个经过“提纯”的、只关于方向的梯度，会继续传递给 **B** 和 **A** ，用来更新它们。

### 总结与对比

| 特性 | LoRA | DoRA |
| --- | --- | --- |
| **可训练部分** | 两个小矩阵 `B`, `A` | 一个大小向量 `m` \+ 两个小矩阵 `B`, `A` |
| **反向传播** | 总梯度 `dL/dW_final` **直接** 流向 `B@A`。 | 总梯度 `dL/dW_final` 被**分解** ： |
| (更新信号混合了大小和方向) | 1\. **投影分量** 流向 `m` (只更新大小) |  |
|  | 2\. **垂直分量** 流向 `B@A` (只更新方向) |  |
| **合并** | 训练结束后，`W_final = W₀ + B@A`。 | 训练结束后，`W_final = m * normalize(V + B@A)`。 |
  
**所以，您的理解非常接近核心思想了！** DoRA并不是用两种不同的优化算法，而是在反向传播的数学路径上做了手脚，实现了梯度的“智能分流”。这确保了更新大小的梯度只去更新大小向量 `m`，而更新方向的梯度只去更新LoRA模块 `B@A`，从而解决了两者被耦合在一起的问题。
