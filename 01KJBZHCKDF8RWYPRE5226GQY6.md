# 论文目标

论文提出了分子描述符cddd的生成方法, 用作各种分子表示之间的中介

核心思路类似于自编码器 (输入 -> 编码器 -> 中介表示 -> 解码器 -> 输出=输入), 自编码器用来训练"中介表示", 使其对输入进行特征提取和降维.

本文的思路略有不同 (分子表示1 -> 编码器 -> 中介表示 -> 解码器 -> 分子表示2), 相当于通过不同语言间的翻译来训练"中介表示"

cddd就是针对分子式的中介表示 (向量)

# 训练数据

```
论文中描述训练数据构建方法的部分主要在 **2.3 Datasets and preprocessing**  这一节。

具体来说，训练数据构建过程如下：

1. **数据来源：** 训练数据来源于两个大型化学数据库：ZINC15 和 PubChem。

2. **合并与去重：** 将两个数据库合并，并去除重复的分子结构。

3. **RDKit 过滤：** 使用 RDKit 化学信息学工具包对合并后的数据集进行过滤，以确保数据质量和一致性。过滤条件包括：
    * 只保留有机分子。
    * 分子量在 12 到 600 之间。
    * 重原子数量大于 3。
    * 分配系数 logP 在 -7 到 5 之间。

4. **去除其他信息：** 去除分子的立体化学信息，去除盐，并且只保留最大的分子片段 (largest fragment)。 这意味着如果一个条目包含多个分子，例如盐和它的组成离子，则只保留最大的那个分子。

5. **提取分子属性：**  为每个分子提取九种分子属性，用于辅助训练和模型验证。这些属性包括：
    * logP（油水分配系数）
    * 最大和最小局部电荷
    * 价电子数
    * 氢键供体和受体数量
    * Balaban's J 值 (一种拓扑指数)
    * 摩尔折射率
    * 拓扑极性表面积

6. **去除无法处理的分子：**  去除 RDKit 无法处理的分子。

7. **最终数据集：** 经过以上步骤，最终的训练数据集包含大约 7200 万个化合物。

此外，论文还提到，用于评估模型的 QSAR 和虚拟筛选数据集中的所有化合物都已从训练数据集中移除，以避免数据泄露和过拟合，保证评估的公平性。

总而言之，论文中训练数据的构建方法强调了数据的多样性（来自两个数据库）、质量控制（RDKit 过滤和属性计算）以及评估的严谨性（移除评估数据）。
```
