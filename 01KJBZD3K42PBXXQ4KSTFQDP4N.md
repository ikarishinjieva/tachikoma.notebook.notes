# 创新

```
CODEAGENT 的创新并非局限于单一步骤，而是在代码生成的整个流程中，对多个步骤都进行了创新，并将这些创新点有机地结合在一起，最终实现了从独立代码单元生成到代码仓库级别生成的跨越。

具体来说，CODEAGENT 在以下几个步骤中都引入了创新：

1. **需求理解阶段**:  CODEAGENT 不仅仅关注用户提供的自然语言描述，还会分析整个代码仓库的信息，包括代码文件、文档、测试用例等，以便更准确地理解用户需求，并生成符合代码仓库整体风格和规范的代码。

2. **代码生成策略阶段**:  CODEAGENT  没有采用传统的单一策略，而是探索和实现了多种代理策略，例如规则型策略、ReAct 策略、Tool-Planning 策略等，并根据用户需求和代码仓库的具体情况选择最合适的策略来指导代码生成过程。

3. **代码实现阶段**:  CODEAGENT  深度集成了多种编程工具，例如信息检索工具、代码分析工具、代码测试工具等，并通过代理策略来协调 LLM 和工具之间的交互，从而实现更高效、更可靠的代码生成。

4. **代码验证阶段**:  CODEAGENT  使用代码仓库中预定义的测试用例来评估生成的代码，确保生成的代码不仅语法正确，而且功能符合预期。

CODEAGENT 将上述这些创新点有机地结合在一起，形成了一套完整的代码仓库级别代码生成方案。它不仅能够生成语法正确、功能符合预期的代码，还能够确保生成的代码与代码仓库的整体风格和规范保持一致，更易于维护和扩展。
``` 

# 不同的代码生成策略

可选的策略包括：

  - 规则型策略 (Rule-based): 按照预定义的规则顺序调用工具，例如先搜索文档，再查找代码示例，最后进行代码生成。
  - ReAct 策略: 让 LLM 动态地决定何时调用哪些工具，以及如何将工具的输出整合到代码生成过程中。
  - Tool-Planning 策略: 首先让 LLM 制定一个行动计划，明确需要调用哪些工具以及调用顺序，然后按照计划执行。
  - OpenAIFunc 策略: 利用 OpenAI 模型的函数调用能力，将预定义的工具函数暴露给 LLM，让其自主选择调用。
