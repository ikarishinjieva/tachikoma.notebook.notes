# 主要思路

CoRe 的主要目标是 训练模型对问题的解答思路

  - 使用Verifier来评估思路的好坏
  - 使用类似STaR的方法, 不断生成新的思路, 将好的思路放在训练数据集中
  - 用MCTS作为生成思路的方法

其中创新: 

  - 使用 困惑度 (PPL) 阈值, 来评估 低质量的数据. 其主要方法是: 
    - 将生成的推理路径的 PPL 与原始数据集中人类标注的真实推理路径的 PPL 进行比较。 如果生成的推理路径的 PPL 高于真实推理路径的 PPL，则说明该生成路径的质量可能不如人类的推理路径，因为它包含了模型难以理解或认为不自然的成分。  

  

# 详细信息

CoRe (合作推理) 的具体步骤如下：

  1. **准备阶段：** 选择一个预训练语言模型作为生成器 G，并选择两个预训练语言模型分别作为步骤验证器 Vstep 和路径验证器 Vpath。

  2. **合作训练阶段：**

a. **微调生成器 G：** 使用包含问题 (q)、真实推理路径 (p) 和真实答案 (gt) 的种子数据集 D = {(qi, pi, gti)} 对生成器 G 进行微调，训练目标是最小化语言模型损失 LLM，即最大化生成真实推理路径的概率。

b. **生成推理路径：** 使用微调后的生成器 G，对数据集 D 中的每个问题 q 生成多个推理路径 (rp) 和对应的答案 (a)，形成新的数据集 D+ = {(qi, rpi,j, ai,j)}，其中 j=1,...,M (M 为每个问题生成的推理路径数量)。

c. **训练验证器：** 使用数据集 D+ 和原始数据集 D 训练步骤验证器 Vstep 和路径验证器 Vpath。

     - Vstep 的训练目标是：1) 最小化语言模型损失 LLM，2) 最小化预测分数与真实标签之间的均方误差 LV S，其中分数用于评估推理步骤的质量，真实标签为答案是否正确 (a == gt)。 Vstep 通过对推理路径中的每个 token 进行评分来评估推理步骤的质量。
     - Vpath 的训练目标是：1) 最小化语言模型损失 LLM，2) 最小化预测分数与真实标签之间的均方误差 LV P，其中分数用于评估整个推理路径的质量。 Vpath 通过对整个推理路径的表示进行评分来评估路径的质量。
  3. **合作推理阶段 (用于零样本推理和自我思考数据生成)：**

a. **输入问题 q：** 给定一个问题 q。

b. **蒙特卡洛树搜索 (MCTS)：** 使用 MCTS 算法进行推理路径搜索。 在搜索过程中，生成器 G 负责扩展节点，生成新的推理步骤；验证器 Vstep 和 Vpath 负责评估生成的推理步骤和路径，并提供反馈，引导生成器探索更合理的路径。 具体来说，MCTS 包括以下步骤：

     - **选择：** 根据改进的 PUCT 公式（结合验证器的评分和节点访问次数），选择一个最有潜力的节点进行扩展。
     - **扩展：** 生成器 G 基于当前节点的状态生成新的推理步骤，并创建一个新的节点存储这些步骤。 Vstep 对新的推理路径进行评分。
     - **模拟：** 从当前节点开始，让生成器 G 完成整个推理路径，直到生成结束符或达到最大长度限制。 Vpath 对完整路径进行评分。
     - **回溯：** 将验证器提供的分数（结合 Vstep 的步骤分数和 Vpath 的路径分数）回溯到根节点，更新节点的奖励值和访问次数。

c. **输出多个推理路径和分数：** MCTS 算法会生成多个推理路径 (rp_j) 和对应的答案 (a_j)，以及每个路径的分数 s_j (由 Vstep 和 Vpath 的评分共同决定)，其中 j=1,...,K (K 为生成的推理路径数量)。

d. **加权投票 (仅用于零样本推理)：** 根据每个推理路径的分数 s_j 进行加权投票，得到最终答案。

  4. **自我思考阶段：**

a. **初始化：** 使用上一轮训练得到的模型权重初始化 G、Vstep 和 Vpath。

b. **生成新数据集 Dnew：** 使用合作推理阶段的步骤，在原始训练集 D 的问题上生成新的推理路径和答案，形成新的数据集 Dnew。

c. **过滤 Dnew：** 使用两个标准过滤低质量数据：

     - **分数阈值：** 过滤掉分数低于预设阈值的推理路径。
     - **困惑度 (PPL) 阈值：** 使用生成器 G 计算生成推理路径的 PPL，只保留 PPL 不高于原始数据集 D 中对应问题真实推理路径 PPL 的路径。 这是因为 PPL 值反映了模型对文本的理解程度，PPL 值越低，说明模型认为这段文本越自然，越符合语言规律。 通过与人类标注的真实推理路径进行比较，可以筛选出更符合人类逻辑的推理路径。

d. **合并数据集：** 将过滤后的高质量数据 Dnew 合并到原始训练集 D 中。

e. **迭代训练：** 使用新的训练集重新进行合作训练 (步骤 2)，更新 G、Vstep 和 Vpath 的权重。

f. **重复步骤 b-e：** 重复上述步骤，直到模型性能达到饱和。

通过以上步骤，CoRe 框架可以有效地训练生成器和验证器，使它们能够相互配合，共同提高模型的推理能力，并在零样本推理任务中取得更好的性能。
