# 主要思路

论文中提出使用 困惑度的拐点, 作为文本分块的边界

困惑度定义: 类似于KL散度, 评估 GT文本 和 LLM续写的文本 之间的概率差, 概率差越大, 认为LLM对于 GT文本 越困惑

其原理是: 如果 文本在某一句上发生了 困惑度的突然上升的拐点, 那么文本在此处大概率开始描述另外一个事物, 或者另外一种逻辑, 此处可作为分块点

# 其他信息

简单来说，PPL分块的逻辑是：一个逻辑连贯的文本块内部，句子间的困惑度应该比较低且平稳。而当遇到逻辑断点或新的主题时，困惑度会上升，因此可以将困惑度上升的点作为分块的边界。

  1. 论文中提到的两种PPL分块策略是指：

  - **直接PPL分块 (Direct PPL Chunking):** 根据预设的困惑度阈值直接进行分块。如果一个句子的困惑度与其前一句子的困惑度差异超过阈值，则将该句子作为新块的起始位置。这种方法简单直接，但可能导致分块长度不均匀。
  - **结合动态合并的PPL分块 (PPL Chunking with Dynamic Merging):** 首先以较低的阈值甚至零阈值进行初步分块，得到一系列“元分块”。然后再根据用户指定的长度需求，将相邻的元分块进行合并，直到满足长度要求。这种方法能够更好地控制分块长度，并保持逻辑一致性。

总而言之，两种策略的核心都是利用困惑度来识别逻辑边界，但后者通过动态合并机制，能够更灵活地控制最终分块的长度，从而更好地适应不同的应用场景。
