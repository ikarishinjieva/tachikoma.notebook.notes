# 背景

在 [20250221 - 使用ms-swift对Qwen2-VL进行微调 [26] - 让模型学习] 中, 已经将长度15的分子的准确率都达标. 准备工作移交

# 梳理近期的实验路径

在 [20250215 - 使用ms-swift对Qwen2-VL进行微调 [25] - 扩展长度9] 中, 使用 增加数据+课程训练 的方式, 将训练长度扩展到9时, ring的识别率无法上升到90%. 

于是想调整训练方案: 

  - 之前的训练方案 要点: 
    - 随机分子 + 对分子的变化 构成一次训练, 按长度来切分课程
    - 在训练到branch结构时, 后面使用了"规范化"的数据 (selfies+smiles幂等性, rdkit库+smiles幂等性), 来区分 "真的分支" 和 "将直链写成分支"
    - 注意: "对分子的变化" 是只包括变化后的分子
  - 调整后的训练方案 要点:
    - 强调"变化", 从之前的训练经验中, 提取了有效的变形措施: 

  -     -       - 表达式规范化, 图形规范化 (让模型对于一张图, 只输出唯一的表达式)
      - 镜像 (选取最后一个原子作为起点)
      - 选取某一个原子作为起点
      - 对局部子串进行轮换 (将第一个或最后一个原子, 插入到其他位置)
      - 在表达式中添加branch/ring操作符
      - 调整branch/ring操作符的位置
      - 调整branch/ring操作符的长度
      - 将branch/ring操作符后的原子往前调整
      - 选取两个原子互换位置
    - 测试时, 使用的验证数据是 "之前的训练方案" 的测试数据, 谨防与训练数据拟合

#### 整理 [20250221 - 使用ms-swift对Qwen2-VL进行微调 [26] - 让模型学习] 中的要点: 

  

每一步"变化", 都要带有"变化前"和"变化后"的数据.

要求训练数据都是 "规范数据" (selfies+smiles幂等性, rdkit库+smiles幂等性)

  

首先训练直链分子, 使用add_basic_atom_to_begin/add_basic_atom_to_end 来扩展长度

到了长度9时, 准确率不达标, 使用swap_adjacent_atoms来增强数据, 可以达标 (第6次训练)

  

然后训练ring分子, 使用add_ring_op从直链分子变化出ring分子

使用图像变化的增强数据, 并没有提升 (第16次训练)

在多次变化后, 有一些评估分子始终无法提升, 认为模型是无法识别这些分子的"大类"(从而无法谈及变化). 研究错误类型后发现是规范化导致了数据偏斜: 

  - (rdkit库+smiles幂等性) 会将 共价键高的原子放在前面 (尽量以C开头, 其次是N, 最后是O, 共价键从大到小, C=4, N=3, O=2), 所以在"规范化"后的直链中插入ring op, 环内都是共价键高的原子, 环外都是共价键低的原子 (第24次训练)  

  - 那么添加ring op时, 需要对分子的镜像也添加一次, 修正共价键排序带来的偏斜

  

对于仍然无法识别的分子"大类", 其图像结构是和(规范化后)训练数据的图像结构 形成 x轴/y轴/xy轴 的镜像. 其也是因为(rdkit库+smiles幂等性)导致 图像只有一种. 于是增加按轴镜像的图像增强 (第27次训练)

额外: 在"第28次训练 - 环内原子轮换"中, 混合训练比分布训练的效果好 (在其他步骤中也有类似的结论或相反的结论, 认为是对于某一些特征, 混合训练更容易提取特征)

对ring分子的训练有利的变化: rotate_atoms_in_ring (第28次训练), swap_adjacent_atoms_in_ring(第27次训练), change_position_of_concat_atom_of_ring(第27次训练)

  

最后训练branch分子, 使用add_branch_op从直链分子变化出branch分子

branch op指定的长度和ring op指定的长度差1, 模型会有误解, 需要进行混合训练

对branch分子训练有利的变化: move_branch_op_one_step (第31次训练), lower_branch_length (第31次训练)
