# 主要思路

DOTS主要用模型生成 "思考路径" , 思考路径由各个原子动作构成. 

模型生成"思考路径"时, 需要解释为什么这样选择.

# 其他信息

DOTS (Dynamically via Optimal reasoning Trajectories Search) 旨在通过动态选择最优推理路径来增强大型语言模型 (LLM) 的推理能力。其核心思想是根据每个问题的具体特征和所使用的 LLM 的能力，自适应地选择最合适的推理动作序列。

DOTS 的具体步骤如下：

**1\. 定义原子推理动作模块:**

首先，DOTS 定义了一系列原子推理动作模块，这些模块可以组合成不同的推理路径。这些模块被分为三层：

  - **分析层:** 用于分析和理解问题。
    - 问题重写 (Query Rewriting): 将问题改写成更清晰或更易于 LLM 理解的形式。
    - 问题分解 (Query Decomposition): 将复杂问题分解成多个更简单的子问题。
  - **解答层:** 用于执行实际的推理过程。
    - 思维链 (Chain of Thought, CoT): 使用自然语言逐步推理。
    - 程序辅助推理 (Program of Thought, PoT): 将问题转化为代码进行计算。
  - **验证层:** 用于检验推理结果的正确性。
    - 自我验证 (Self-Verification): LLM 自我生成验证问题并解答，以检验答案的正确性。
  - **空操作 (Empty):** 允许跳过分析层或验证层。

**2\. 搜索最优动作轨迹:**

对于每个训练样本，DOTS 使用一个基于探索和评估的搜索算法来寻找最优的推理动作轨迹。

  - **初始化:** 创建一个包含所有可能的推理路径组合的候选集合。
  - **迭代评估:** 对候选集合中的每条路径进行多轮评估 (论文中使用两轮)。 在每一轮中，使用目标 LLM 按照每条路径进行多次推理 (Neval=4)，并记录成功率（正确答案的比例）。
  - **筛选:** 在每一轮评估后，根据成功率和路径长度（优先选择成功率高且路径短的）筛选出表现最好的 N_k 个路径 (N1=8, N2=3)，并更新候选集合。
  - **最终选择:** 迭代结束后，选择成功率最高且路径最短的路径作为该问题的最优轨迹。

**3\. 解释生成:**

找到最优轨迹后，使用更强大的 LLM (例如 GPT-4o) 为该轨迹生成一个自然语言的解释，说明为什么这条路径是最佳选择。 这有助于提高规划器 LLM 的学习效率和模型的可解释性.

**4\. 微调 LLM 以规划最优推理轨迹:**

使用收集到的训练数据（问题、最优轨迹、解释）对规划器 LLM 进行微调，使其能够根据输入问题预测最优的推理动作轨迹。 DOTS 提供了两种微调策略：

  - **外部规划器:** 训练一个独立的、较小的 LLM 作为规划器，为目标 LLM 选择推理路径。适用于闭源或计算资源有限的场景。
  - **内部化规划器:** 直接在目标 LLM 上进行微调，使其同时具备推理和路径规划的能力。适用于开源且规模较小的 LLM。

**5\. 推理应用:**

在实际应用中，当接收到一个新的问题时：

  - **路径规划:** 规划器 LLM (外部或内部化) 预测最优的推理动作轨迹。
  - **推理执行:** 目标 LLM 按照预测的轨迹执行相应的推理动作，最终得到答案。

通过以上步骤，DOTS 实现了根据问题和 LLM 的特点动态选择最优推理路径，从而提高 LLM 的推理性能和效率。
