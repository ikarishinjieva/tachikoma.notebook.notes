# 评估

  - 迁移评估
    - DB评估
      - 包括数据库版本，字符集，迁移数据量大小等基本信息

    - 对象评估
      - 包括表、索引、序列、自定义函数、存储过程等对象的兼容性及改造成本评估

    - 应用迁移评估
      - 包括应用涉及的 SQL 语句兼容性、前后性能对比差异及改造成本评估

    - 工作量评估
      - 根据对 DB、对象、应用迁移的改造成本进行具体工作量的评估

    - 难易度评估
      - 根据以上改造成本和工作量综合分析评估难易度

  - 智能转换
    - 表结构自动转换

      - 将表不同的字段类型进行转换，支持多种数据格式。

    - SQL 语法转换

      - 将 Oracle 的 SQL 语句通过自研引擎进行自动 SQL 语法转换，无需人工参与

    - 存储过程等特殊对象转换

      - 将 Oracle PLSQL 语言定义的存储过程转换为 Java 代码+MySQL 的 SQL 语句 ，实现代码智能逻辑重构

  - 数据同步: 针对停机时间窗口，迁移数据量、应用服务等级可以采用不同的数据同步方式
    - 全量同步

    - 增量同步
  - 数据校验
    - 确认源和目标端的对象是否一致

    - 针对源端和目标端的表中数据量进行统计对比

    - 支持源端和目标端针每行数据通过相关算法进行逐行对比，确保数据的准确性

  - 安全回退
    - 支持异构数据库双向同步，基于该方式可以实现在不同场景下的数据回退方

案

    - 对于部分业务模块迁移

      - 可以采用双向复制的方式，此时源端和目标端同针对不同的模块时提供服务，当目标端

出现异常后，源端数据库可以快速接管，应用只需要修改连接即可以实现快速切换。

    - 对于全部业务模块迁移

      - 可以采用单向反向复制的方式，此时只有目标端对外提供服务，源端用于在异常情况下

的快速回退，当运行一段时间稳定后，再将源端下线。

    - (复制方向与业务模块数没有逻辑关系)
