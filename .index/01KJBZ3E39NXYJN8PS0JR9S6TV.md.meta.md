## 摘要
本笔记描述了 SQL 规则检查系统的训练数据重新设计方案，目标是为 MySQL、PostgreSQL、DB2 等多种数据库设计统一的规则描述语言。内容包括规则元信息结构（编号、分类、适用范围、检查流程等）以及规则代码自动生成流程，同时列出了当前多库规则实现中的不一致问题和待优化项。

## 关键概念
- 规则元信息: 描述规则库的结构化数据，包含编号、种类、适用范围、检查流程等字段
- 脱机检查: 对静态 SQL 语句进行规则检查，如 CREATE TABLE、ALTER TABLE 等
- 联机检查: 运行时从数据库获取元数据进行动态检查
- 统一描述语言: 为多种数据库设计的主流程描述框架，使生成代码逻辑更准确
- DDL/DML 分类: 规则按 SQL 类型分类，支持分类扩展便于归纳和查找
