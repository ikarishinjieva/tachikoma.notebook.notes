## 摘要
探索在 ChatGLM 微调中仅训练总步骤来审核 SQL 规则的方法，发现第三步 AST 判断存在错误。通过将 match 训练数据加入训练，利用其长输入短输出特性提高是/否判断的权重，成功修正总步骤的准确性。

## 关键概念
- 总步骤：完整的 SQL 规则审核流程，包含规则获取、AST 解析、规则检查、结论输出
- AST 检查规则：基于抽象语法树的 SQL 规则验证逻辑，用于判断 SQL 是否符合指定规则
- 训练权重：输出长度影响模型学习，短输出在独立训练数据中权重更高
- match 数据：单独的 AST 规则判断训练数据，输出仅为是/否
