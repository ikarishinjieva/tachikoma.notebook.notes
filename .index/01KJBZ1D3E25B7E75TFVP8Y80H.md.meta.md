## 摘要
记录 ChatGLM-6B 在 SQL 规则验证任务中因嵌套 IN 和换行符导致的判断错误，发现模型对"LIMIT 子句"概念理解不稳定。通过调整训练数据格式和内容，修正模型输出使其正确识别无 LIMIT 的 SQL 违反规则。

## 关键概念
- 嵌套 IN 查询: 多层 IN 嵌套会导致模型产生错误分析，输出"没有返回第 n 个 SELECT 语句的结果集"等幻觉内容
- 换行符影响: SQL 中的换行符会干扰模型对语义的理解，去掉换行后分析更准确
- 概念稳定性: 模型对"使用 LIMIT"等核心概念的理解不一致，需要针对性训练数据修正
- 训练数据调整: 通过增加正负样本对（有/无 LIMIT 的 SQL）来稳定模型对规则的理解
