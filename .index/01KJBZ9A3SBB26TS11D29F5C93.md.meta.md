## 标签
ChatDBA, 数据库故障排查，临时文件，SQL 优化，hash join, MySQL

## 摘要
记录国债生产环境数据库临时文件异常占用（300G→1.9T）的完整排查过程。通过慢日志和 processlist 定位到含 TRIM 函数的 SQL 导致全表扫描和 hash join 产生巨大中间结果集。优化方案为移除关联条件上的函数、添加有效过滤条件以减少结果集。

## 关键概念
- 内部临时表引擎：MySQL 使用 Memory 或 TempTable 存储临时表，Memory 存储效率较低
- hash join：哈希连接可能产生巨大的临时文件，尤其是关联条件使用函数时
- TRIM 函数影响：在关联条件上使用 TRIM 会导致索引失效，触发全表扫描
