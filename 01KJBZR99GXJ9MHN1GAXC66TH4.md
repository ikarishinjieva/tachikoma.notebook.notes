# 主要思路

  - 使用笔记形式, 将一个问题和其衍生问题的有关信息采集起来进行整理
  - zero-shot, 让大模型发挥其能力进行梳理, 而不限制其笔记形式
  - 对笔记的质量进行评估, 选择更好的笔记
  - 对任务合适停止迭代, 有一套判断指标

# 其他信息

你总结的 Adaptive-Note 的创新点非常到位。除了你提到的这些，还有一些其他的创新点，或者可以从不同角度来看待其创新性：

  1. **知识增长视角:** Adaptive-Note 不仅仅是简单地收集信息，而是从知识增长的角度来组织和管理信息。它强调在每次迭代中都要增加新的、有用的信息，并将其整合到现有的知识结构中。这种视角使得信息收集过程更加目标明确，避免了盲目探索。

  2. **自适应停止探索策略:** 不同于传统的 ARAG 方法，Adaptive-Note 的停止策略并非依赖于 LLM 对检索时机的预测，而是基于笔记的质量变化。这种策略更加稳定可靠，因为它直接关注信息收集的效果，而不是依赖于 LLM 可能存在偏差的预测。这也有助于控制检索成本，避免过度检索。

  3. **通用性和可插拔性:** Adaptive-Note 的设计非常通用，可以与任何 LLM 结合使用，而无需额外的训练。这种可插拔性使其易于应用于不同的场景和任务。

  4. **对现有 ARAG 方法的改进:** Adaptive-Note 解决了现有 ARAG 方法的一些关键局限性：

     - **信息交互:** 它通过笔记机制促进了不同检索步骤之间信息的整合和交互，避免了信息碎片化的问题。
     - **检索时机偏差:** 它避免了依赖 LLM 预测检索时机可能带来的偏差，从而更有效地收集关键信息。
  5. **模拟人类学习过程:** Adaptive-Note 的工作流程模拟了人类学习新知识的过程：先广泛阅读，然后逐步将新知识整合到现有的知识体系中，直到对该领域有足够的了解。这种仿生式的设计使其更加符合人类的认知规律。

总而言之，Adaptive-Note 的创新性体现在多个方面，它不仅改进了现有的 RAG 方法，还提出了一种更有效、更稳定的知识获取和管理方式，为复杂问答任务提供了新的解决方案。
