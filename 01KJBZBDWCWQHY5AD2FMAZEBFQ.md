# 创新点1 - 三元组超节点

使用 三元组超节点 来构建知识图谱

传统三元组举例: (John Travolta) - [主演] -> (Pulp Fiction)

三元组超节点举例: ((Seth MacFarlane)−[created]→(Family Guy))−[in]→(1999)

简单说: 三元组超节点就是包含了多个三元组的 (或者说 包含一个完整句子的) 节点

其构造更复杂(不好把握尺度, 且论文中没有提及如何构造), 但因为包含了更多信息, 在之后CoE的相似性查询中效果更好

# 创新点2 - CoE - 由LLM进行调度的图搜索

```
CoE 算法的具体步骤如下：

1. **规划 (Planning):** 
    * 算法首先根据用户查询制定一个探索知识图谱的计划。
    * 该计划定义了探索的步骤顺序，以及每一步要查找的节点或关系类型。

2. **知识图谱查找 (KG Lookups):**
    * 算法根据当前步骤的计划，利用向量相似性搜索和图数据库查询两种方式查找相关的节点或关系。
    * 向量相似性搜索用于快速定位与查询语义相关的节点，而图数据库查询则用于精确匹配特定类型的关系。

3. **评估 (Evaluation):**
    * 算法会评估当前检索到的信息是否满足查询需求。
    * 如果满足，则算法会根据检索到的信息生成最终答案。
    * 如果不满足，算法会根据评估结果调整探索计划，例如：
        * 扩展搜索范围，探索更多节点或关系。
        *  уточнить搜索条件，查找更精确的信息。

4. **循环 (Loop):**
    * 算法会重复执行步骤 2 和 3，直到找到满足查询需求的信息，或者达到预设的探索次数限制。

总而言之，CoE 算法通过循环执行 "规划-查找-评估" 的步骤，逐步缩小搜索范围，最终找到与用户查询最相关的知识图谱信息。
```
