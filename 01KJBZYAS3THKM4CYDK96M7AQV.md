# 精细化的数据筛选策略

  - 提示选择 (Prompt Selection)：聚焦于“难”和“有价值”的问题
    - 研究人员根据模型对一个提示（Prompt）生成回答的长度，来判断这个提示的难度
    - 他们只保留了那些能引发模型长篇大论的、更难的提示。
    - **使用这一半的“硬核”数据进行训练，模型在数学和科学任务上的性能反而提升了2%-4%** 。
  - 响应级别缩放 (Response-Level Scaling)：放大优质样本的价值
    - 对于每一个高价值的提示，不是只用一个专家模型的回答来训练，而是**生成四个不同的回答 (generating four responses for each prompt)** 。

# 反直觉的单阶段长文本RL 

  - 直接在目标最大长度（64K）上进行单阶段RL，效果要优于多阶段渐进式训练。
    - 核心论点：多阶段训练会导致“长文本能力遗忘”

      - 论文给出了一个关键的解释：

        1. **SFT阶段的基础** ：在进入RL阶段之前，模型已经通过监督微调（SFT）被训练过，使其具备了生成64K长度响应的能力。这相当于模型已经“见过”并学习了长篇内容的结构和模式。
        2. **RL阶段的“倒退”** ：如果在RL的初始阶段，强行将模型的最大输出长度限制在一个较短的范围内（如16K或32K），模型为了在这些短任务上获得更高的奖励，会倾向于生成更短、更简洁的回答。
        3. **不可逆的性能下降** ：这个过程会导致模型“**unlearn** ”（忘掉）它在SFT阶段学到的长文本生成能力。论文指出，这通常会导致一个**“显著且不可逆的性能下降” (significant and irreversible drop in performance)**。模型的平均输出长度会变短，这种“退化”在后续切换到更长文本的RL阶段时，很难被完全纠正回来，从而限制了模型最终能达到的性能上限。

# 领域定制的RL损失函数- 科学RL

  - “使用专门的、专家验证过的多项选择题进行RL，比使用混合质量或未经证实的数据进行训练，能带来显著更好的性能。”
  - 但论文中并没有说明为什么是 "多项选择题"

# 迭代式自蒸馏 (Iterative Distillation)

使用 sft -> RL -> 生成新的轨迹 (论文的任务是训练reasoning) -> 数据筛选 -> 新的sft -> 新的RL

说明: 

  - 论文的前提是: 训练reasoning解题, 所以中间的解题轨迹有很多 (人类不可穷举)
  - 所以一次迭代, 是让模型提高到新的水平, 来生成更高质量的轨迹. 再让模型基于此来进行新一轮的解题
  - 数据筛选: 是看轨迹是否能解决问题
