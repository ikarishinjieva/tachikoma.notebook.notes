# 论文目的

在 自动规划 的任务中, 有以下几个概念: 行动模式/规划/符号规划期/规划方案, 区分如下: 

```
行动模式 (Action Schema):  定义一个动作的抽象模板。它规定了动作的参数、前提条件和效果。 类似于编程中的函数定义。

规划 (Plan): 指的是一系列具体的动作序列，用于实现目标状态。 类似于编程中的函数调用序列。

符号规划器 (Symbolic Planner):  接收行动模式和问题描述作为输入，并搜索一个可行的规划。可以理解为一个搜索算法，在所有可能的动作序列中找到能够达到目标状态的序列。

规划方案 (Plan Candidate): 符号规划器找到的满足目标的可行规划。一个问题可能有多个规划方案。
``` 

本文要解决的问题是: 人为生成的行动模式 过于单一, 并且成本过高

# 有用的概念 - 共形预测

共性预测 用于 为预测结果提供可行度 评估的, 比如 年龄-身高 的预测中, 对"20岁的平均身高是170cm"这个说法的可信度是85%.

其优点是: 

  - 分布无关： 不需要对数据的分布做出任何假设。
  - 有限样本有效： 即使在小样本情况下也能提供有效的置信度保证。
  - 可解释性强： 置信区间/集合能够直观地反映预测结果的可信度。

过程举例: 

```
让我们通过一个简单的例子来说明共形预测的流程。

**场景：** 假设我们想要预测一个人的身高，已知一些人的年龄和身高数据。

**数据：**

| 年龄 | 身高 (cm) |
| --- | --- |
| 10 | 140 |
| 12 | 150 |
| 15 | 165 |
| 18 | 175 |
| 20 | 180 |

**流程：**

1. **校准集：**  我们将最后两个样本 (18, 175) 和 (20, 180) 作为校准集，其余三个样本作为训练集。

2. **模型训练：**  我们使用一个简单的线性回归模型在训练集上进行训练，得到一个身高预测模型。

3. **非共形度量计算：**  我们使用绝对误差作为非共形度量。  对于校准集中的每个样本，计算其预测身高与实际身高的绝对误差：

    * 样本 (18, 175):  假设模型预测身高为 170，则绝对误差为 |175 - 170| = 5
    * 样本 (20, 180):  假设模型预测身高为 178，则绝对误差为 |180 - 178| = 2

4. **新样本预测：**  现在有一个新的样本，年龄为 16 岁。  使用训练好的模型预测其身高，假设预测结果为 170 cm。

5. **新样本非共形度量计算：**  我们不知道新样本的实际身高，但可以假设一个身高值，例如 168 cm。 则新样本的非共形度量值为 |168 - 170| = 2。

6. **p值计算：** 将新样本的非共形度量值 (2) 与校准集中的非共形度量值 (5, 2) 进行比较。  新样本的非共形度量值在校准集中的排名为 2/3 (并列第二名，有两个样本的非共形度量值小于等于2，校准集样本数为2，所以p值为2/3。如果校准集样本数为n，则p值的分母为n+1)。  因此，新样本的 p 值为 2/3 ≈ 0.67。

7. **置信区间构建：**  假设显著性水平 α = 0.05。 由于 p 值 (0.67) 大于 α (0.05)，我们将预测身高 170 cm 包含在置信区间中。  我们可以根据不同的假设身高值计算不同的p值，最终得到一个置信区间，例如 [167, 173]，表示我们有 95% 的置信度认为这个 16 岁的人的实际身高落在 167 cm 到 173 cm 之间。

**总结：**

在这个例子中，我们使用共形预测方法为身高预测结果提供了一个置信区间。  即使我们不知道数据的真实分布，共形预测也能提供有效的置信度保证。  需要注意的是，非共形度量的选择和显著性水平的设定都会影响最终的置信区间。

希望这个例子能够帮助你更好地理解共形预测的过程.
``` 

# 论文的主要思路

为了让行动模式多样化, 采用如下步骤: 

  - **构建多样化的行动模式库：**利用多个LLM实例，并设置较高的温度参数，以生成多样化的行动模式候选方案，尽可能覆盖用户意图的各种可能解释。这也有利于提高找到可求解行动模式集的概率。

  - **语义一致性过滤：**利用句子编码器将行动描述和生成的行动模式转换为向量表示，计算它们之间的余弦相似度，并使用共形预测（Conformal Prediction）方法过滤掉低置信度的候选方案，确保生成的行动模式与任务描述在语义空间上高度一致。

  - **规划生成和排序：**将可求解的行动模式集输入经典符号规划器生成规划方案，并根据组成行动模式的累积语义相似度得分对生成的规划方案进行排序，优先选择与自然语言描述语义一致性最高的方案。

主要通过 LLM多次生成 + 过滤 来实现 行动模式的多样化. 行动模式用 PDDL 来作为形式化.

其中过滤阶段, 使用了 embedding到语义空间 + 共形预测 来进行初筛.

PDDL有一些通用规划器进行搜索

# 有用的概念 - PDDL

````
PDDL（Planning Domain Definition Language，规划领域定义语言）是一种用于描述规划问题的标准化语言。它提供了一种形式化的方式来表达规划任务中的各个方面，包括：

* **领域 (Domain):**  描述规划问题所属的 general environment，定义了可用的行动、谓词和对象类型。
* **问题 (Problem):** 描述一个具体的规划实例，指定了初始状态、目标状态和可用的对象。

**PDDL 的主要组成部分：**

* **对象 (Objects):** 规划问题中涉及的实体，例如机器人、积木、地点等。  每个对象都有一个类型 (Type)。
* **谓词 (Predicates):** 描述对象的状态或关系，例如 `(on blockA blockB)` 表示积木 A 在积木 B 上面,  `(at robot room1)` 表示机器人在房间1。
* **行动 (Actions):**  规划问题中可以执行的操作，每个行动都有参数、前提条件和效果。
    * **参数 (Parameters):** 行动作用的对象。
    * **前提条件 (Preconditions):** 行动能够执行的条件，用谓词逻辑表达式表示。
    * **效果 (Effects):** 行动执行后产生的结果，用谓词逻辑表达式表示，描述了世界状态的变化。
* **初始状态 (Initial State):**  规划问题的起始状态，用谓词的集合表示。
* **目标状态 (Goal State):**  规划问题想要达到的状态，用谓词逻辑表达式表示。

**一个简单的 PDDL 例子：**

```pddl
(define (domain blocksworld)
  (:requirements :strips :typing)
  (:types block)
  (:predicates (on ?x ?y - block) (clear ?x - block) (ontable ?x - block) (handempty))
  (:action pick-up
    :parameters (?x - block)
    :precondition (and (clear ?x) (ontable ?x) (handempty))
    :effect (and (not (ontable ?x)) (not (clear ?x)) (not (handempty)) (holding ?x)))
  (:action put-down
    :parameters (?x - block)
    :precondition (holding ?x)
    :effect (and (not (holding ?x)) (clear ?x) (handempty) (ontable ?x)))
  (:action stack
    :parameters (?x ?y - block)
    :precondition (and (holding ?x) (clear ?y))
    :effect (and (not (holding ?x)) (clear ?x) (handempty) (on ?x ?y) (not (clear ?y))))
  (:action unstack
    :parameters (?x ?y - block)
    :precondition (and (handempty) (clear ?x) (on ?x ?y))
    :effect (and (holding ?x) (not (clear ?x)) (not (handempty)) (clear ?y) (not (on ?x ?y)))))

(define (problem problem1)
  (:domain blocksworld)
  (:objects a b c - block)
  (:init (ontable a) (ontable b) (on c a) (clear b) (clear c) (handempty))
  (:goal (and (on a b) (on b c))))
```

**PDDL 的适用性：**

PDDL 虽然是一种表达能力很强的规划领域描述语言，但它并非适用于所有类型的规划任务。  它主要适用于**经典规划 (Classical Planning)** 问题，这类问题具有以下特点：

* **确定性 (Deterministic):**  行动的效果是确定的，没有随机性。
* **完全可观察 (Fully Observable):**  规划器可以完全了解当前的世界状态。
* **静态 (Static):**  除了规划器执行的行动之外，世界状态不会发生变化。
* **离散 (Discrete):**  世界状态和行动都是离散的。

对于不满足这些条件的规划问题，例如涉及随机性、不确定性、连续状态或动态环境的规划问题，PDDL 的表达能力就会受到限制。  在这种情况下，可能需要使用其他类型的规划模型和算法，例如概率规划、马尔可夫决策过程 (MDP) 等。

总而言之，PDDL 是一种强大的工具，可以用来描述各种经典规划问题，但它并非万能的。  选择合适的规划模型和算法取决于具体问题的特点。
```` 

# 有用的知识
