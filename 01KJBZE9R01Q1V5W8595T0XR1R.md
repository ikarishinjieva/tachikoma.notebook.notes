# 项目定位

OpenPrompt主要是 一个 prompt-learning的实现框架, 类似于 haystack 是一个RAG的实现框架

# 什么是prompt-learning

传统NLP任务, 比如分类, 如果直接用LLM处理, 是要求LLM直接输出分类结果. (此时LLM可用的只有自己的知识, 上下文信息太少, 所以能力有限, 需要依靠大量数据进行微调)

prompt-learning, 是构造一个提示词模板, 将NLP任务转换成一个LLM的预测任务, 并带有足够的上下文信息. 

比如: 

```
"The sentiment of this sentence is [MASK]."
``` 

要求LLM填写MASK. (提示词带有了足够的上下文信息)

prompt-learning 架构包括了 (提示词模板 + LLM + 训练), 通过训练让 (提示词模板 + LLM) 在NLP任务上表现提升. 

训练的方向有两种: 

  1. 仅对提示词模板进行调整
  2. 对LLM 在 提示词模板 下的性能进行调优

# 提示词模板 调整技术 - soft prompt

简述:

soft prompt 是在提示词中开辟了一块 "可训练块", 整个训练过程是对 "可训练块" 的向量表示进行优化. 

优化后的向量表示 保持在之后的embedding结果中, 不将其"解释"成 人类文字

举例: 

```
好的，让我们来看一个使用 soft prompt 进行情感分类的例子，包括其优化后的输出。

**任务：** 情感分类 (二分类，积极或消极)

**模型：** BERT

**Prompt 模板：** "The sentiment of this movie review is <soft> sentiment </soft>."

**Soft Prompt 初始化：**  将 "sentiment" 初始化为一个随机向量，维度与 BERT 的词向量维度相同。

**训练数据示例：**

* 输入: "The sentiment of this movie review is <soft> ... </soft> This movie is absolutely fantastic!"  标签: 积极
* 输入: "The sentiment of this movie review is <soft> ... </soft>  I was bored out of my mind."  标签: 消极

**训练过程：**

1. 将 prompt 和输入文本拼接在一起，输入 BERT 模型。
2. 固定 BERT 的参数，只更新 "sentiment" 这个 soft prompt 的向量表示。
3. 使用交叉熵损失函数计算模型预测的标签和真实标签之间的差距。
4. 根据损失函数的梯度，使用优化器 (例如 Adam) 更新 "sentiment" 的向量表示。

**优化后的 Soft Prompt 输出：**

经过训练后，"sentiment" 的向量表示会逐渐优化，例如：

* **积极情感**:  "sentiment" 的向量表示可能与 "positive", "amazing", "great" 等词语的词向量更接近。
* **消极情感**:  "sentiment" 的向量表示可能与 "negative", "terrible", "boring" 等词语的词向量更接近。

**预测过程：**

1. 将 prompt 和新的评论文本拼接在一起，输入 BERT 模型。
2. BERT 会根据优化后的 "sentiment" 向量表示，以及评论文本中的其他信息，预测 [MASK] 位置的词语概率分布。
3. 根据预测的词语概率分布，判断评论的情感倾向。

**总结：**

在这个例子中，我们通过训练 "sentiment" 这个 soft prompt 的向量表示，使其能够更好地表达情感信息，从而引导 BERT 模型进行情感分类，而不需要改变 BERT 本身的参数。
``` 

# 提示词模板 调整技术 - prefix prompt

简述: 相当于 在输入文本前 增加一个soft区域, 进行调优

举例: 

```
您对 Prefix Tuning 的理解非常准确！

**Prefix Tuning 的核心思想的确是在输入文本前增加一个可学习的 “前缀”，这个前缀可以看作是一个 “soft 区域”，它与输入文本一起被送入 PLM 进行处理。**

**举个例子：**

假设我们想要使用 Prefix Tuning 来进行机器翻译任务，将英文翻译成法语。

* **传统的 fine-tuning 方法：**  使用大量的英法平行语料，微调整个 PLM 的参数，使其能够进行翻译任务。
* **Prefix Tuning 方法：**
    * 在每个输入文本前添加一个可学习的前缀，例如：`<prefix> This is an example sentence .` 
    *  `<prefix>` 部分是一个长度固定的向量序列，初始化为随机值，并在训练过程中更新。
    *  将拼接后的文本 `<prefix> This is an example sentence .` 输入 PLM，让其预测对应的法语翻译。
    *  在训练过程中，**只更新 `<prefix>` 部分的向量表示**，而 PLM 的其他参数保持不变。

**Prefix Tuning 的优势：**

* **参数效率更高**:  相比于微调整个 PLM，Prefix Tuning 只需要调整少量的前缀参数，大大减少了计算成本和存储空间，尤其适用于超大规模的 PLM。
* **缓解灾难性遗忘**:  由于 PLM 的参数保持不变，Prefix Tuning 可以更好地保留 PLM 在预训练阶段学习到的知识，避免灾难性遗忘问题。
* **提高跨语言迁移能力**:  一些研究表明，Prefix Tuning 在跨语言迁移学习任务中表现出色，因为它能够学习到更通用的语言表示。

**总而言之，Prefix Tuning 是一种参数高效且有效的 prompt-learning 方法，它通过在输入文本前添加可学习的前缀，引导 PLM 进行特定任务，而无需改变 PLM 本身。**
```
