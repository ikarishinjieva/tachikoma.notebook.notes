# 主要思路

要解决的问题: 增加 查询重写 的效果

思路: 

  1. 进行多次重写, 形成候选答案
  2. 对于每次重写:
     1. 进行文档召回
     2. 对于召回的文档, 使用reranking模型进行排序, 记住排序的结果
  3. 对于多次重写, 评估 每个排序的结果 的优劣
     1. 优劣的评价 使用算法: Precision@K 或 MRR 

论文中还介绍了 在线反馈(实时反馈) 和 离线反馈(批量反馈), 常用学习算法: 

  - 在线反馈: 通常使用强化学习算法 (如 PPO)，因为这类算法适用于处理序列决策问题，并且可以根据环境的反馈逐步优化策略。
  - 离线反馈: 可以使用多种学习算法，例如：
    - 基于偏好对的算法 (如 DPO)。

    - 基于排序损失的算法

# 对排序结果的评价算法

## Precision@K 和 MRR 介绍

Precision@K 和 MRR 都是信息检索领域常用的评估指标，用于衡量检索系统或算法返回结果的准确性和相关性。

**1. Precision@K (P@K)**

* **定义:** Precision@K 指的是在检索系统返回的前 K 个结果中，相关文档所占的比例。   
* **计算公式:**  
```  
Precision@K = (检索到的相关文档数量) / K   
```  
* **例子:** 假设用户搜索 "机器学习算法"，检索系统返回了 10 个结果 (K=10)，其中 7 个是用户认为相关的，那么 Precision@10 = 7/10 = 0.7。  
* **优点:** 简单直观，易于理解和计算。  
* **缺点:** 只考虑了前 K 个结果，忽略了其他可能相关的结果。对于不同的检索任务，K 的选择也很重要。

**2. Mean Reciprocal Rank (MRR)**

* **定义:** MRR 用于评估第一个相关文档在检索结果中的排名位置。它计算第一个相关文档的排名的倒数，并对多个查询取平均值。  
* **计算公式:**  
```  
MRR = (1/第一个相关文档的排名) 的平均值  
```  
* **例子:** 假设用户进行了 3 次查询，第一个相关文档分别出现在第 1、3、2 位，那么 MRR = (1/1 + 1/3 + 1/2) / 3 = 0.61。  
* **优点:** 更加关注第一个相关文档的位置，对于那些只需要找到一个最相关结果的任务 (例如问答系统) 比较适用。  
* **缺点:** 只考虑了第一个相关文档的排名，忽略了其他相关文档的位置信息。

  
**总结**

* Precision@K 适用于评估检索系统返回结果的整体相关性，特别是关注前 K 个结果的情况。  
* MRR 适用于评估检索系统快速找到第一个相关文档的能力，特别是对于只需要一个最相关结果的任务。

在实际应用中，应该根据具体的检索任务和需求选择合适的评估指标。

# 学习算法

DPO: 需要多组数据, 进行 优-劣 对比, 通过对比进行学习

PPO: 定义策略函数, 通过训练策略函数的参数进行学习

DPO、PPO 和 KTO 都是用于训练机器学习模型的算法，但它们应用于不同的场景，并且核心思想也存在差异。以下是三种算法的详细对比：

**1\. DPO (Direct Preference Optimization)**

  - **类型** : 模仿学习 (Imitation Learning)
  - **适用场景** : 主要用于从**偏好数据** 中学习，例如用户对不同选项的偏好、排序任务等。
  - **核心思想** :
    - 输入：一组偏好对 (q, q'g, q'b)，其中 q'g 是相对于 q'b 更好的选择。
    - 目标：训练模型，使其对于任何输入 q，都能预测出 q'g 的得分高于 q'b。
    - 方法：通常使用基于对比学习的方法，例如将 q'g 和 q'b 分别输入模型，然后最小化两者得分之间的差距。
  - **优点** :
    - 可以直接从偏好数据中学习，不需要具体的奖励函数。
    - 训练效率较高，尤其适用于大规模数据集。
  - **缺点** :
    - 需要大量的偏好数据，数据的质量对模型性能影响较大。
    - 难以处理复杂的环境交互，因为其本质上是一种静态学习方法。

**2\. PPO (Proximal Policy Optimization)**

  - **类型** : 强化学习 (Reinforcement Learning)
  - **适用场景** : 适用于需要与环境进行交互，并通过试错来学习最优策略的场景，例如游戏 AI、机器人控制等。
  - **核心思想** :
    - 目标：学习一个策略函数 (policy function)，该函数能够根据当前状态选择最佳的动作，以最大化长期累积奖励。
    - 方法：通过迭代更新策略函数的参数，使得新策略相对于旧策略的改进幅度得到控制，从而保证训练的稳定性。
  - **优点** :
    - 能够处理复杂的环境交互，学习更灵活的策略。
    - 在很多领域都取得了很好的效果，例如游戏 AI、机器人控制等。
  - **缺点** :
    - 训练效率相对较低，需要大量的交互数据。
    - 超参数调节较为困难，对经验要求较高。

**3\. KTO (Knowledge-augmented Trajectory Optimization)**

  - **类型** : 模仿学习 (Imitation Learning)
  - **适用场景** : 主要用于从**示范数据** 中学习，例如学习人类专家的操作轨迹、从文本描述中学习策略等。
  - **核心思想** :
    - 输入：一组状态-动作对 (s, a) 或状态-动作-评价元组 (s, a, ρ)，其中 ρ 表示动作 a 在状态 s 下的好坏评价。
    - 目标：训练模型，使其能够模仿示范数据中的行为模式，生成高质量的动作序列。
    - 方法：通常使用基于序列模型的方法，例如循环神经网络 (RNN)，将状态作为输入，预测下一步的动作。
  - **优点** :
    - 能够从示范数据中学习，不需要具体的奖励函数。
    - 可以学习复杂的序列行为模式。
  - **缺点** :
    - 需要大量的示范数据，数据的质量对模型性能影响较大。
    - 泛化能力有限，难以处理未在示范数据中出现过的情况。

**总结**

  - DPO 适用于从偏好数据中学习，例如排序任务。
  - PPO 适用于需要与环境进行交互，并通过试错来学习最优策略的场景。
  - KTO 适用于从示范数据中学习，例如模仿人类专家的行为。

在 RaFe 中，DPO 和 KTO 可以用于离线反馈训练，因为它们可以利用预先收集的排序分数来构建偏好对或评价标签。而 PPO 更适用于在线反馈训练，因为它可以根据实时获取的排序分数来更新模型参数。

希望以上解释能够帮助您更好地理解 DPO、PPO 和 KTO 之间的区别。
