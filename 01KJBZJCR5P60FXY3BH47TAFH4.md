# 总结

人类总结: 

```
1. HippoRAG的创新点, 是如何从知识图谱中找到段落
2. 其构建了一个无模式的知识图谱
3. 在检索知识图谱时, 使用改进的PageRank算法进行沿着边的概率传播, 从而摆脱传统搜索过程中依赖于路径模式或者关键词的局限
4. 特异的节点具有更高的权重, 其影响面可以传播更远
5. HippoRAG适合于多跳推理, 主要是第3点中描述的PageRank可以摆脱路径/关键词的束缚, 仅通过"存在边"+概率传播 就可以发现 多跳的逻辑连接
``` 

其中: 无模式的图谱 + PageRank 在图谱中进行知识发现, 是个不错的思路

大模型完善: 

```
HippoRAG 的创新在于它提出了一种新的方法，利用知识图谱来桥接用户查询和相关段落，从而实现更精准和高效的检索，尤其是在需要知识整合的多跳推理场景下。其核心创新点可以总结如下：

1. **知识图谱引导的段落检索：** HippoRAG 的核心创新在于它不直接在段落层级进行检索，而是利用知识图谱作为中间桥梁。  通过将查询和段落都映射到知识图谱上，HippoRAG 可以更好地理解查询语义，并找到与查询语义相关的段落，而不是仅仅依赖于关键词匹配。

2. **无模式知识图谱构建：**  HippoRAG 构建了一个无模式的知识图谱。相比于传统的基于预定义 schema 的知识图谱，无模式知识图谱更加灵活，可以处理各种关系类型，并方便地整合新知识，无需预先定义 schema，从而更好地适应开放领域的知识检索。

3. **基于 Personalized PageRank (PPR) 的图搜索：**  在检索知识图谱时，HippoRAG 使用 PPR 算法进行沿着边的概率传播。相比于传统的基于路径模式或关键词的图搜索算法，PPR 算法具有以下优势：
    * **无需预定义路径：** PPR 算法不依赖于预定义的路径模式，可以根据查询中的种子节点（即查询中的关键概念）灵活地进行搜索，从而发现知识图谱中隐含的多跳连接。
    * **考虑节点间的关联性：** PPR 算法通过概率传播机制，能够有效地衡量节点与种子节点之间的关联性，并根据关联性对节点进行排序，从而更准确地找到与查询相关的节点。
    * **更高的效率：**  相比于遍历所有可能的路径，PPR 算法通过迭代传播概率的方式，能够更有效地进行多跳推理。

4. **节点特异性增强检索精准度：**  HippoRAG 引入了节点特异性（类似于 IDF）的概念，特异的节点（即出现段落数较少的节点）具有更高的权重。节点特异性会影响种子节点在 PPR 算法中的初始概率，特异性越高的节点，其初始概率越高，在概率传播过程中影响范围也更广，从而提高检索的精准度。

5. **适用于多跳推理场景：**  HippoRAG 尤其适用于多跳推理场景。这是因为它结合了 PPR 算法和无模式知识图谱。PPR 算法通过概率传播可以有效地探索图中的多跳连接，而无模式知识图谱则提供了丰富的语义关系。两者结合，使得 HippoRAG 能够基于图谱结构和概率传播来发现隐含的多跳逻辑连接，从而更有效地回答需要知识整合的多跳问题。

总而言之，HippoRAG 通过结合无模式知识图谱、PPR 算法和节点特异性等技术，实现了一种更灵活、高效、准确的知识检索方法，尤其适用于需要知识整合的多跳推理场景，显著提升了从知识图谱中找到相关段落的能力。
``` 

# 对PPR (PageRank改) 算法的描述

```
2. **PPR 算法的搜索步骤:** PPR 算法是一种基于随机游走的图搜索算法，它可以找到与给定种子节点相关的节点。以下是一个具体的例子：

    假设我们有以下知识图谱（节点和边）：

    * 节点：A, B, C, D, E
    * 边：A->B, A->C, B->C, C->D, D->E

    查询：A

    步骤：

    a. **初始化:** 将种子节点（A）的概率设为 1，其他节点的概率设为 0。

    b. **迭代传播:** 在每次迭代中，每个节点将其概率的一部分传递给它的邻居节点。传递的概率比例由边的权重和阻尼因子（damping factor）决定。阻尼因子通常设置为 0.85 左右。

        * 例如，如果阻尼因子为 0.85，A 将其概率的 0.85/2 传递给 B 和 C。

    c. **随机跳转:** 在每次迭代中，每个节点还有一定概率（1 - 阻尼因子）跳转到随机节点。

        * 例如，如果阻尼因子为 0.85，A 有 0.15 的概率跳转到随机节点（A, B, C, D, E 中的任意一个）。

    d. **重复 b 和 c:** 重复迭代传播和随机跳转步骤，直到节点的概率分布收敛。

    e. **排序:**  最终，根据节点的概率值进行排序，概率值越高的节点与种子节点的相关性越高。

    在这个例子中，最终的概率分布可能会类似于：A > B > C > D > E。这意味着 B 和 C 与 A 的相关性较高，而 D 和 E 的相关性较低。

``` 

# 对节点特异性的描述

```
3. **节点特异性:** 节点特异性 (Node Specificity) 类似于 IDF，用于衡量节点在语料库中的重要性。其计算公式为 `si = |Pi|^(-1)`，其中 `|Pi|` 是包含节点 `i` 的段落数量。`|Pi|` 越小，`si` 越大，表示节点 `i` 越独特，重要性越高。

    例如，如果“Stanford”出现在 10 个段落中，“Alzheimer's”出现在 100 个段落中，则“Stanford”的节点特异性高于“Alzheimer's”。在 PPR 算法中，节点特异性会影响种子节点的初始概率分布，从而影响最终的检索结果。

```
