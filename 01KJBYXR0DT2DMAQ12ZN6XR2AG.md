# 这是一场贩卖焦虑的PUA

大家把手机拿出来, 计算器调出来, 我们来算一笔帐, 关系到公司的未来

现在看一下周围的同事, 大家就理解为什么公司的策略很难执行, 需要大量的管理精力来贯彻策略

我来列举一下大家在想啥: 

  - 一个计算器怎么可能跟公司的未来有关
  - 不知道是啥, 我先观察观察, 看看大家咋做的
  - 万一大家都不响应, 就我拿出手机, 我有点丢人
  - 算账, 算什么账, 算的肯定不是我的工资, 不管
  - 上次这个人说了个决定, 结果没实现, 一看就执行力不行, 这次不能被坑了
  - 台上这TM谁啊, 他以为他是谁啊

我们今天先从个人发展开始PUA, 然后再来CPU一些管理的知识

# 个人发展

我们以DBA为例, 来说个人在职场的发展. 我2019年接手了华北的DBA团队, 对DBA的发展绝对有发言权

这里有没有出来工作只为了体验生活的同事, 那我们假设所有同事出来工作都是为了赚钱.

已经拿出了计算器的同事, 我们来算笔账, 没拿的就心算一下

现在我是一个初级DBA, 每天跑两个客户现场处理问题, 一年220个工作日, 全年无休

每年处理440个问题, 其中1/5的场景是新场景, 那一年积累88条经验 (安装问题/配置参数问题/性能问题诊断/巡检问题/... 都算经验)

准备三年换工作, 到时候就是中级DBA. 经验逐年遗忘, 每年遗忘30%, 问三年后, 还剩多少条经验

第一年结束: 88*0.7

第二年结束: (88*0.7+88)*0.7

第三年结束: ((88*0.7+88)*0.7+88)*0.7 = 134.904

问: 如果你是老板, 为一位具有134.904条经验的DBA愿意付多少钱 (如果我是老板, 得先给他买个医疗保险)

我们怎么提升自己的升级速度

一共有三个变量: 

  - 每年440个客户现场 (2个/天 * 220天/年)
  - 1/5的的场景是新场景
  - 每年遗忘30%

我们能提高哪一个?

客户现场的数量:

  - 现在开始锻炼身体, 吃保健品
  - 加速处理过程: 
    - 用标准流程
    - 用脚本辅助

每年遗忘量: 

  - 建立自己的知识库, 仔细整理知识, 可搜索的那种

新场景数量: 

  - 值得重点说

### 如何提高新场景数量

"重复的活我不干, 我是来学东西的"

这里面有两个误区: 

  - 活真的是重复的么
    - 在有脚本之前, 安装数据库是专门的一项工作, 工作量一天起. 
  - 老板欠你钱不

"大家有新场景都叫我一下"

  - 大家欠你钱不
  - 靠什么跟大家换: 多向大家提供帮助, 建立好自己的技术形象, 大家有问题找你就有用, 有技术就解决问题, 没技术就陪同人情

"公司应该向我提供培训"

  - "这些培训的场景, 我平常遇不到, 做了也没什么用"
    - 回到小学, 学习四则运算, 我们一定要拉到菜市场教么? 
    - 都要依靠 实验->知识->实践 的过程, 过不了门槛就没人带你玩
    - 无数次在现场, 当场景真的来的时候, 我看到DBA连该找什么文档都不知道
  - "公司的培训都是关于MySQL的, 不够丰富"
    - T型人才的定义, 初期要建立优势领域, 再将优势领域的方法论平铺扩大
    - 这不是个初级DBA该关心的事, 先温饱再思那啥
    - 学不动的时候, 桌面不平都是合理的原因

跟公司做双赢交易

  - 卖: 重复的活 -> 规范/工具, 向新人培训
  - 买: 让新人来接手重复工作
  - "那新人不就替代我了么"
    - 能优化工作的人 和 能工作的人 不在一个层次上
    - 流程，手法，工具，对于个人是用来突破固有领域的，不是拿来守住领域的

为什么年龄上去了, 级别没上去

  - 太"聪明"
    - 攀爬曲线 整体趋势往上, 不是每一步都在往上走, 有些步骤Leader会诱导你趟过去, 有些步骤得自己扛过去. 每一步都要求最优解, 往往走不下去
    - 很多人第一次扛失败, 心态就崩了, 之后成功的概率就不高了. 第一次的坚持性比较重要
    - 憨一点的风格, 多看一段时间 (3-6个月) 之后的总体趋势, 更容易突破
    - 管理的夸人法，是对个人能力的弱化，弱化了延迟满足的能力. 必要, 但不能总要

### 现在我们变成了一个中级DBA

温饱解决了, 下一步怎么走

很多同事会停在这个阶段, 达成工作和生活的"平衡", 开始熬年限. 为什么? 

  - 年龄到了
  - 收入解决了温饱, 生活丰富了起来
  - 开始有技术自信了, 在团队里获得了尊重和成就感
  - 往下一阶梯上升的方法, 已经超出了公司的培训体系, 自己摸索几次找不到路径

这种状态会维持到生活出现了突然的变故

(参加了同学会, 听到了同龄人的分享, 意外生了个娃, 春节回老家碰到了亲戚家的孩子, 中级的DBA很少会因为看了一两本书而受到刺激)

  - 突然不满目前的"平衡"状态, 舍不得怪自己, 得找个出气口 (家庭/公司 都有可能)
  - 重新走上提升的路程, 但提升的反馈有点慢
  - 更换个工作, 新的环境带来的新鲜感会维持18个月, 让自己有理由重新"振奋"起来. 18个月以后, 再次"平衡"

我们怎么安排好自己的"平衡"

  - 不可否认 我们需要分阶段达成平衡, 不能总鸡血, 也不能总趴窝
  - 踩踩油门, 松一松, 但不能 动能回收
  - 松油门可以维持 差不多6个月左右, 不会太危险  

    - 又有人开始卡bug了, 松6个月, 踩油门一天, 又开始新的6个月
    - 跟团队达成交易, 双方不会有误会, 但要有交易的资本

如何积累交易的资本/如何加速突破中级阶段

  - 首要要素: T型人才模型的 横向技能 建立
    - "诶呀, 公司是做MySQL的, 我还能怎么横向". 哪个公司没有主营业务
      - 经验: 在任意方向多做一点点, 就是这方面的"专家", 就有足够的信息杠杆. 大部分信息都会汇总到"专家"这里, 场景经验就都有了
    - "我就是个做技术的"
      - 技术人总有良好的元能力，学习能力/文档能力/实验能力，却只能把它放在技术上
    - 往技术方向和非技术方向 都尝试一下
      - 我们的价值, 是把事做成, 不是把技术活干完
    - 如何选择方向
      - 可以穷举: 为什么穷举是个好方法，因为大家看到算力不足就提前放弃, 能持续穷举的人很少
      - 面对可能的机会, 先参与, 搞清楚付成本的是谁, 然后再 衡量值得或不值得. 大部分人做反了
      - 管理/项目经理/售前/开发/测试/技术自媒体/深入技术/产品经理/保安保洁 方向 都可以尝试
        - 我为什么关掉了confluence的个人页面

  - 继续加速新技术吸取 (继续加速初级DBA的过程)
    - 如何提升关键因素: 客户现场的数量/每年遗忘量/新场景数
      - 为什么 沈波 建立了工单系统, 自己偷偷翻各个工单
      - 为什么 我刚接手华北的时候, 会一张一张复审华北的工单, 快速学习整个交付的知识
      - 核心思想: 不停跟公司作交易, 让自己立于小团队的信息中枢, 让其他人来采集信息, 我们来整理信息
    - 如何立于信息中枢
      - 靠谱是团队的准入证，进来以后才会有增量的收益

      - 用技术/信息换情感收入, 用情感支出换技术/信息
      - 快速扒取 高密度的信息点:
        - 谁是高密度的信息点, 如何去占用他的时间

  - 什么是职业边界
    - 我们设想我们自己开一家公司, 最有效率的情况, 就是让整个公司像一个人一样去工作
    - 在互联网上公开搜索JD, 按照这些JD上列举的条件, 招聘到刚刚好满足条件的人, 他们的工作仅限于这些条件上的规定, 那么公司是否能开起来
    - 每个职业都有棱有角的, 那么在图形上, 就会出现缺口和裂缝
      - 一个公司要发展, 要稳定发展, 整个图形就要牢固
      - 但这些缺口和裂缝多了, 结构就不稳定了
      - 所以在实际公司的运作中, 最好的情况就是每个职业边界都往外扩一点, 大家都重叠一部分
      - 当公司规模大了, 还会存在专门糊缝的职业, 管理/经理 就都是此类
    - 那么有价值的职业技能, 就是带个裙边的职业边界. 
      - 最有价值的职业技能, 就是各个方向都可以做一点, 适配公司各阶段的发展, 一直立在信息中枢上
    - 对于技术出身的人员, 掌握了信息采集/整理/处理的科学方法, 有足够的优势来扩展自己的职业边界, 充分发挥自己的优势获得更有利的地形
    - 拓展 职业边界是个没有短期收益的过程, 一定要架在平稳的职业节奏上
      - 间歇性踌躇满志，断层式努力症

  - 如何保障转岗的有效性
    - 只有在现有职位上 工作成果优秀的 才叫转岗, 其他的就只叫发配

### 现在我们变成了一个高级DBA

终于要开始考虑公司的战略与个人的关心, 为什么, 这难道不是总监以上的事情么

  - 收入 = 苦劳 + 功劳, 哪部分是有杠杆的? 
    - 对于初中级, 完成工作要求 + 创造少量额外价值, 就能保证收入
    - 对于高级, 年龄到了, 就得从苦劳转换到功劳
  - 什么是功劳:   

    - 对于公司来说, 就是在商业上赚钱
    - "我做的工作" -> 通过战略转换 -> 商业上的赚钱
  - 为什么不要完全听信公司给你的战略
    - 这是一个互相磨合/折磨的过程, 没有成熟的路径说 怎么怎么样 商业上一定能赚钱, 都必须要不停尝试 (老板也是第一次开公司)
    - 公司很难为每个人设计好 跟战略结合的过渡地带, 这个地带抓在自己手里会更有主动权
    - 对于达成一致的战略路径, 尝试的成功或者失败, 都是高价值的
    - 在这个战略路径上, 会不停更新迭代, 收集信息/参与决策/实施决策/技术回归 是高杠杆的价值.
    - 当参与到决策过程中时, 你看到的路径就会发生变化 (亲生的和领养的 区别)
  - 技术 or 管理
    - 这本身没有明确的界限, 当想把事情做成功, 单人的精力有限, 就要带个团队来完成事情. 缺什么就补什么, 这是技术的根本
    - 懂点管理, 才能拿捏住管理你的人. 不管你将来做不做管理.
      - 永远别用你的视角 去揣测 管理你的人, 全都是误解
    - 懂点管理, 就知道什么是最贵的资源, 就知道单人的力量实在是有限, 就知道什么对战略最重要
      - 公司会变来变去 -> 但永远不敢亏待资源中心
  - 容易走入的误区
    - 志大而不见机，多谋而少决，好兵而无权
      - 学会谈判: "我理解你要做什么, 我能做什么匹配你, 我还有更多的价值是你想要的, 你得多付给我什么回报"
    - 佛系: 到了高级的年龄, 研究的是临床佛学
      - 那是因为 是佛没骑在你头上拉屎
      - 尽量更"聪明"地赚钱: 很快就没条件 跟小伙子们拼体力
    - 随着你做的越来越好，身边都是洋溢的笑脸的十足的笨蛋
      - 向上寻求定位和突破
      - 下棋的对手，不是同事，是在一个社会性的工作体系
